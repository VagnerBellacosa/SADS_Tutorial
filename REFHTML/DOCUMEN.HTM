Documentações

O SADS/DD fornece documentações ativas e atualizadas de todos os sistemas, porque são obtidas diretamente dos cadastramentos efetuados no repositório de dados.

Estas documentações utilizam o REPRINTS e são emitidas com um banner contendo a empresa, área, nome do usuário, estação e usercode onde foi disparada a emissão.

Essas documentações, que podem ser emitidas em impressoras remotas ou na impressora do sistema central, são as seguintes:

Emissão de manual;

Fonte de library;

Relação de elementos;

Relação de programas;

Relação detalhada de entidades;

Desenho de tabelas;

Desenho de telas;

Atributos por sistema ou estrutura;

Atributos por sistema por código externo;

Estrutura de dados;

Estrutura de dados (datasets) por banco;

Descrição de elementos;

Cadastramento de programas;

Impressão de contexto de navegação;

Relação detalhada dos datasets de um banco.

A seguir, cada uma dessas documentações encontra-se explicada detalhadamente.

Emissão de manual

O SADS permite emitir um manual com informações detalhadas a respeito de um determinado sistema. Estas informações são obtidas dos cadastramentos dos elementos do sistema efetuados no repositório de dados do SADS.

Existem as seguintes possibilidades de emissão do manual de um sistema:

Emissão automática:

Emissão automática do manual do sistema;

Emissão automática do manual do usuário;

Emissão por comando.

A seguir, cada um desses tipos de manuais que podem ser impressos pelo SADS encontra-se explicado isoladamente.

Emissão automática

A emissão automática do manual de um determinado sistema é efetuada tomando como base uma estrutura predefinida, onde a seleção e a ordem dos capítulos a serem impressos já estão determinadas e não podem ser alteradas a gosto do usuário.

Porém, no momento em que se solicita a impressão automática do manual, pode-se escolher com qual das duas estruturas seguintes este será emitido:

.Manual do Sistema, que contém informações relativas à implementação física do sistema destinadas ao analista, tais como: arquivos, datasets, banco de dados, telas, jobs, programas on-line e batch etc;

.Manual do Usuário, que contém informações relativas à interface do sistema destinadas ao usuário final, tais como: desenho de telas, relatórios etc.

As estruturas de capítulos que serão impressas nos manuais de sistema e de usuário podem ser definidas para cada instalação nas tabelas SBPM7013 e SBPM7014, respectivamente.

Deste modo, quando uma emissão automática de manual for requisitada, seja ele do sistema ou do usuário, o SADS verificará se existe uma estrutura de capítulos definida para tal manual na tabela SBPM7013 ou na SBPM7014, dependendo do tipo do manual. Caso na instalação esta estrutura não tenha sido definida, o SADS imprimirá o manual segundo estrutura default especificada para o tipo de manual.

As estruturas default do SADS para o manual do sistema e para o manual do usuário estão descritas a seguir.

Nas tabelas SBPM7013 e SBPM7014 devem ser definidos os elementos - telas, arquivos, tabelas etc - que serão relatados em cada capítulo do manual e quais os relatórios - transações - a serem impressos referentemente a cada um destes elementos.

Para melhor esclarecimento de como devem ser cadastradas as estruturas destes manuais nas tabelas SBPM7013 e SBPM7014, deve-se consultar a Seção V, "Parametrizações do SADS", no item "Emissão de manual", neste manual.

Emissão automática do manual do sistema

O SADS emite o manual do sistema de acordo com uma estrutura própria da instalação que é definida na tabela SBPM7013 ou, caso esta não tenha sido cadastrada, uma estrutura default.

As informações do manual do sistema emitidas conforme a estrutura default do SADS são de interesse do analista de sistemas e são impressas conforme a seguinte estrutura:

Capítulo 1 : Descrição do sistema
Imprime a descrição do sistema.

Capítulo 2 : Funções
Imprime a descrição das funções cadastradas no sistema.

Capítulo 3 : Relação de Elementos
Imprime a relação de todos os elementos cadastrados e relacionados ao sistema, com seus códigos e
nomes.

Capítulo 4 : Resumo Quantitativo
Imprime as quantidades cadastradas por elemento no sistema.

Capítulo 5 : Resumo quantitativo de outros sistemas
Imprime as quantidades de elementos de outros sistemas que este sistema utiliza, por
tipo de elemento.

Capítulo 6 : Descrição de atributos
Imprime as descrições dos atributos cadastrados no sistema.

Capítulo 7 : Entidades
Imprime as descrições e estruturas de todas as entidades cadastradas no sistema.

Capítulo 8 : Programas on-line
Imprime a relação de programas on-line e os recursos (menu, datasets, telas, libraries
etc) cadastrados para cada um.

Capítulo 9 : Programas batch
Imprime a relação de programas batch e os recursos (job, datasets, relatórios, libraries
etc) cadastrados para cada um.

Capítulo 10 : Telas
Imprime os desenhos das telas cadastradas no sistema.

Capítulo 11 : Tabelas
Imprime os desenhos das tabelas cadastradas no sistema.

O SADS tem como default imprimir as telas e tabelas que estão em teste. Então, para que sejam impressas aquelas que estão em produção, deve-se informar "P" no campo ESTADO/AMBIENTE da tela de disparo da emissão.

Emissão automática do manual do usuário

O SADS emite o manual do usuário de acordo com uma estrutura própria da instalação que é definida na tabela SBPM7014 ou, caso esta não tenha sido cadastrada, uma estrutura default.

As informações do manual do usuário emitidas conforme a estrutura default do SADS são de interesse do usuário final do sistema e estão descritas nos tópicos a seguir:

Capítulo 1 : Descrição
Imprime a descrição do sistema.

Capítulo 2 : Funções
Imprime as descrições das funções cadastradas no sistema.

Capítulo 3 : Telas
Imprime para cada tela do sistema:

Desenho;

Descrição;

Estrutura de dados;

Descrição dos atributos de entrada - descrição dos atributos da tela.

Capítulo 4 : Tabelas
Imprime para cada tabela:

Desenho;

Estrutura de dados;

Descrição dos atributos da estrutura;

Estrutura de dados da chave secundária.

Capítulo 5 : Relatórios
Imprime as simulações dos relatórios por comando.

Essa simulação será por comando somente para aqueles relatórios que possuem os comandos de simulação cadastrados. Para os demais, a simulação será automática.

O SADS tem como default imprimir as telas e tabelas que estão em teste. Então, para que sejam impressas aquelas que estão em produção, deve-se informar "P" no campo ESTADO/AMBIENTE, na tela de disparo da emissão.

Emissão de manual por comando

Na emissão de manual por comando, a estrutura do manual a ser impresso é definida pelo usuário no momento em que a emissão é solicitada.

Para executar a emissão por comando, devem ser selecionados os elementos a serem impressos e, para cada elemento, uma ou mais transações.

Cada elemento selecionado irá definir um capítulo do manual e cada transação informada para o referido elemento constituirá um item - relatório - do capítulo.

Pode-se obter diferentes tipos de relatórios conforme os argumentos informados nos campos ELEMENTO e TRANSACOES. A explicação de como devem ser preenchidos estes campos está descrita no tópico a seguir.

Argumentos do relatório

Para que a emissão de manual seja disparada, alguns argumentos devem ser informados no menu de operação LISEMIMAN, quais sejam:

 Sistema - Código do sistema, no padrão SADS.

 Impressora - Código da impressora.

Pode assumir os seguintes valores:

 "<<Código da impressora remota>>" - Para impressão do manual na impressora remota informada (esta impressora deve estar cadastrada na rede de terminais);

 "PRINTER" - Para impressão do manual na impressora central, no formulário de 132 posições;

 "PRINxx" - Para impressão do manual na impressora central em formulário especial, onde "xx" é um formid definido na instalação.

Caso este campo não seja informado, será assumida a impressora default cadastrada para o usuário no Sistema de Segurança. Caso esta não exista cadastrada, então é obrigatório informar o código da impressora no momento do disparo da impressão.

As impressoras remotas podem ser cadastradas, com um formid correspondente, na tabela SBPM5026. Sempre que for disparada uma impressão pelo SADS, esta tabela é consultada e, caso a impressora esteja cadastrada, o relatório será impresso com o seu formid correspondente. A tabela SBPM5026 está descrita na Seção V, "Parametrizações do SADS", neste manual.

Será apresentada a tela de processamento para emissão do manual, onde os seguintes argumentos devem ser informados:

 SISTEMA - Código do sistema, no padrão SADS.

 USERCODE - Usercode de análise (opcional).

Só será obrigatório informar o usercode quando a transação 17 - Impressão de fonte de library - for selecionada na impressão de manual por comando.

 ESTADO/AMBIENTE - Estado das telas e tabelas ou ambiente das libraries a serem impressas (opcional).

Pode assumir os seguintes valores:

 "P" - Produção;

 "T" - Teste (default);

 "M" - Manutenção;

 "D" - Desenvolvimento.

Os valores "M" e "D" são válidos apenas para emissão por comando de telas e tabelas.

 TIPO DO MANUAL(AUTOM.) - Tipo do manual automático a ser impresso (opcional).

Este campo só deve ser informado quando o manual a ser impresso for automático - Manual do sistema ou Manual do usuário - e não for necessário, então, definir sua estrutura através dos campos ELEMENTO e TRANSACOES.

Pode assumir os seguintes valores:

 "S" (Manual do Sistema) - Para imprimir o manual do sistema de maneira automática, conforme estrutura predefinida na tabela SBPM7013, ou, caso esta não exista, conforme estrutura default deste manual definida pelo SADS.

 "U" (Manual do Usuário) - Para imprimir o manual do usuário de maneira automática, conforme estrutura predefinida na tabela SBPM7014, ou, caso esta não exista,conforme estrutura default deste manual definida pelo SADS.

 CAPA(S/N) - Indicador de impressão da capa do manual contendo um desenho com o código do sistema.

Pode assumir os seguintes valores:

 "S" - Para imprimir o manual com a capa;

 "N" - Para imprimir o manual sem a capa.

 INDICE(S/N) - Indicador de impressão de índice do manual.

O índice, caso seja selecionado, será impresso no final do manual e irá informar a estrutura de capítulos e páginas que foi impressa.

Pode assumir os seguintes valores:

 "S" - Para imprimir o índice;

 "N" - Para não imprimir o índice.

 CAPITULO REEMISSAO - Número do capítulo a ser reimpresso. (opcional)

Quando um capítulo for informado neste campo, o SADS imprimirá somente este capítulo.

Caso o manual já tenha sido impresso anteriormente e esta impressão tenha sido disparada através de uma emissão por comando, toda a estrutura do manual deverá ser informada novamente nos campos ELEMENTO e TRANSACOES, a fim de que o capítulo a ser reemitido fique com a numeração de páginas semelhante à anterior.

 IMPRESSORA - Código da impressora.

Pode assumir os seguintes valores:

 "<<Código da impressora remota>>" - Para imprimir o manual na impressora remota informada (esta impressora deve estar cadastrada na rede de terminais);

 "PRINTER" - Para imprimir o manual na impressora central, no formulário de 132
posições;

 "PRINxx" - Para imprimir o manual na impressora central em formulário especial, onde "xx" é um formid definido na instalação.

Caso este campo não seja informado, será assumida a impressora default cadastrada para o usuário no Sistema de Segurança. Caso esta não exista cadastrada, então é obrigatório informar o código da impressora no momento do disparo da impressão.

As impressoras remotas podem ser cadastradas com um formid correspondente, na tabela SBPM5026. Sempre que for disparada uma impressão pelo SADS, esta tabela é consultada e, caso a impressora esteja nela cadastrada, o relatório será impresso com o seu formid correspondente. A tabela SBPM5026 está descrita na Seção V, "Parametrizações do SADS", neste manual.

 ELEMENTO - Código do tipo do elemento a ser relatado, no padrão SADS (opcional).

Cada tipo de elemento definido irá compor um capítulo do manual.

Os códigos de tipos de elementos só devem ser informados quando a emissão do manual for por comando, ou seja, quando o campo TIPO DO MANUAL(AUTOM.) não for informado.

Os códigos de tipos de elementos válidos são:

 "sss" - Quando o capítulo a ser emitido referir-se ao sistema e forem especificadas transações que emitem relatórios diretamente relacionados ao sistema; ou

 "ssst" (código do tipo do elemento) - Quando o capítulo a ser emitido referir-se a um determinado tipo de elemento e forem especificadas transações que emitem relatórios referentes a todos os elementos deste tipo.

Para melhor entendimento de como será utilizado o campo ELEMENTO pelas transações, seguem os exemplos:

Informações do relatório

As informações a serem impressas dependem do tipo do manual selecionado, quais sejam:

Manual do sistema;

Manual do usuário;

Manual por comando.

A seguir, são apresentados os conteúdos de cada um destes manuais isoladamente.

Manual do sistema

O SADS emite o manual do sistema de acordo com uma estrutura própria da instalação que é definida na tabela SBPM7013 ou, caso esta não tenha sido cadastrada, uma estrutura default do manual do sistema.

A tabela SBPM7013 está descrita na Seção V, "Parametrizações do SADS", neste manual.

As informações do manual do sistema emitidas conforme a estrutura default do SADS são de interesse do analista de sistemas e estão descritas a seguir:

Capítulo 1 : Descrição
Imprime a descrição do sistema.

Capítulo 2 : Funções
Imprime as descrições das funções cadastradas no sistema.

Capítulo 3 : Relação de Elementos
Imprime a relação de todos os elementos - códigos e nomes - cadastrados e relacionados ao sistema.

Capítulo 4 : Resumo Quantitativo
Imprime as quantidades de elementos de cada tipo cadastrados no sistema.

Capítulo 5 : Resumo quantitativo de outros sistemas
Imprime as quantidades de elementos de outros sistemas que este sistema utiliza.

Capítulo 6 : Descrição de atributos
Imprime as descrições dos atributos cadastrados no sistema.

Capítulo 7 : Entidades
Imprime a descrição e a estrutura de cada entidade cadastrada no sistema.

Capítulo 8 : Programas on-line
Imprime a relação de programas on-line e os recursos (menu, datasets, telas, libraries
etc) cadastrados para cada um.

Capítulo 9 : Programas batch
Imprime a relação de programas batch e os recursos (job, datasets, relatórios, libraries
etc) cadastrados para cada um.

Capítulo 10 : Telas
Imprime os desenhos das telas cadastradas no sistema.

Capítulo 11 : Tabelas
Imprime os desenhos das tabelas cadastradas no sistema.

O SADS tem como default imprimir as telas e tabelas que estão em teste. Então, para imprimir aquelas que estão em produção, deve-se informar "P" no campo ESTADO/AMBIENTE da tela de disparo da emissão.

Manual do usuário

O SADS emite o manual do usuário de acordo com uma estrutura própria da instalação que é definida na tabela SBPM7014, ou, caso esta não tenha sido cadastrada, uma estrutura default do manual do usuário.

A tabela SBPM7014 está descrita na Seção V, "Parametrizações do SADS", deste manual.

As informações do manual do usuário emitidas conforme a estrutura default do SADS são de interesse do usuário final do sistema e estão descritas a seguir:

Capítulo 1 : Descrição
Imprime a descrição do sistema.

Capítulo 2 : Funções
Imprime as descrições das funções cadastradas no sistema.

Capítulo 3 : Telas
Imprime para cada tela:

Desenho;

Descrição;

Estrutura de dados;

Descrição dos atributos de entrada - descrição dos atributos da tela.

Capítulo 4 : Tabelas
Imprime para cada tabela:

Desenho ;

Estrutura de dados

Descrição dos atributos da estrutura;

Estrutura de dados da chave secundária.

Capítulo 5 : Relatórios
Imprime a simulação dos relatórios por comando.

A simulação será por comando somente para aqueles relatórios que possuem os comandos de simulação anteriormente cadastrados. Para os demais, a simulação será automática.

O SADS tem como default imprimir as telas e tabelas que estão em teste. Então, para imprimir aquelas que estão em produção, deve-se informar "P" no campo ESTADO/AMBIENTE, na tela de disparo de emissão.

Manual por comando

O SADS emite a documentação do manual por comando de acordo com a estrutura definida pelo usuário nos campos ELEMENTO e TRANSACOES.

Pode-se obter diferentes tipos de relatórios, conforme os argumentos informados nestes campos.

Os relatórios que podem ser impressos referentemente a cada elemento estão listados a seguir e alguns deles - exceto as simulações de relatórios - serão detalhadamente descritos nos próximos tópicos:

Simulação automática de relatório;

Fonte de library;

Simulação de relatório por comando;

Relação de elementos;

Estrutura de dados;

Atributos por sistema ou estrutura;

Desenho de telas;

Relação de programas;

Descrição de elementos;

Desenho de tabelas;

Relação detalhada de entidades;

Cadastramento de programas;

Atributos por sistema por código externo;

Estrutura de dataset por banco;

Relação detalhada de datasets por banco.

Fonte de library

O SADS oferece uma transação com a função de imprimir o código fonte da library de um elemento cadastrado no SADS.

Para imprimir o código fonte de uma library, é necessário que a mesma já esteja residente em um determinado usercode, independentemente de ter sido ou não gerada pelo SADS.

Na emissão de manual por comando, a impressão do código fonte de uma library está relacionada com a transação 15.

Argumentos do relatório

Para que o código fonte de uma library seja impresso, alguns argumentos devem ser informados no menu de operação LISFONLIB, quais sejam:

 Library - Código da library, no padrão SADS.

A padronização adotada pelo SADS para os códigos de libraries é a seguinte:

  "sssXXxxnn", onde:

 sss - Código alfabético que identifica o sistema;

 X - Constante do tipo do elemento

 X - Constante de arquivo físico (F ou L);

 xx - Identificador do elemento;

 nn - Sequencial identificador do elemento (opc.).

Caso a library seja referente a um programa on-line ou batch, sub-rotina ou DASDL de banco, a sua padronização adotada pelo SADS é a seguinte:

 "sssXFxxT", onde:

 sss - Código alfabético que identifica o sistema;

 X - Constante do tipo do elemento (N,P,S ou B, respectivamente);

 F - Constante de fonte

 xx - Identificador do elemento;

 T - Constante de DASDL de banco de dados em teste (opc.)

Caso a library seja referente a uma área de comunicação com sub-rotina, estrutura de work, tela, relatório, registro de uso geral, base de dados do sistema, dataset, parâmetros físicos de dataset, tabela ou arquivo, a padronização adotada pelo SADS é a seguinte:

 "sssXLxxnn", onde:

 sss - Código alfabético que identifica o sistema;

 X - Constante do tipo do elemento (S,P,T,,R,G,B, M ou A, respectivamente);

 L - Constante de Library;

 xx - Identificador do elemento;

 nn - Identificador numérico do elemento.
