Registro do dataset de uso geral

Conceituação

Registros do dataset de uso geral são registros de um dataset que podem ser utilizados para armazenar informações de controle dos diversos sistemas de uma instalação. No SADS, o dataset que tem esta função é chamado SBGB02.

Os sistemas podem ter visões distintas da área do SBGB02, sendo que cada uma destas visões deve ser cadastrada no SADS como um registro deste dataset.

Uma utilização criteriosa deste elemento pode evitar a construção de vários datasets para utilizações específicas pelos aplicativos. Um sistema de contabilidade, por exemplo, pode ter o registro de período fiscal em aberto armazenado no dataset de uso geral.

Padronização

O padrão de codificação adotado pelo SADS para a identificação de um determinado tipo de registro do dataset de uso geral é o seguinte:

"sssGxn"onde:

"sss" éo código alfabético que identifica o sistema;

"G" éa constante de registro do dataset de uso geral;

"xn" éo indicador alfanumérico do tipo de registro do dataset de uso geral (arquivo lógico).

Como exemplo da padronização adotada para tipo de registro do dataset de uso geral tem-se "ARHG01", onde:

"ARH" representa a Área de Recursos Humanos de uma empresa;

"G" é a constante do registro do dataset de uso geral;

"01" é o código identificador do tipo de registro ou do arquivo lógico.

Cada tipo de registro do dataset de uso geral, que deve ser entendido como um arquivo lógico, pode possuir vários registros, cada um com uma estrutura diferente de dados.

O padrão de codificação adotado pelo SADS para a identificação de um registro do dataset de uso geral é o seguinte:

"sssGxnmm"onde:

"sss"éo código alfabético que identifica o sistema;

"G"éa constante de registro de dataset de uso geral;

"xn"éo indicador alfanumérico do tipo do registro (arquivo lógico);

"mm"éo indicador do registro.

Como exemplo de um registro do dataset de uso geral, cadastrado para a mesma área do exemplo anterior, tem-se "ARHG0101", onde:

"ARH" representa a área de Recursos Humanos da mesma empresa;

"G" é a constante do registro do dataset de uso geral;

"01" é o indicador do tipo do registro ou do arquivo lógico;

"01" é o indicador do registro.

No SADS, cada registro do dataset de uso geral é formado pelos atributos/campos que compõem a sua estrutura de dados. A partir da estrutura de dados de um registro é gerada umalibrary para ser incorporada aos programas que o utilizam.

É importante ressaltar que, nos menus de operação e de escolha, o SADS considera o tipo de registro como sendo o próprio registro do dataset de uso geral e o registro, como sendo a sua estrutura.

Utilização

Os sistemas utilizam os datasets de uso geral para armazenar pequenos volumes de dados, relativamente estáticos, pouco versáteis, a fim de evitar a criação desnecessária de datasetsespecíficos.

Para o gerenciamento desses dados, geralmente é criado um SBGB02 para cada banco físico da instalação.

Os grandes volumes de dados devem ser armazenados em datasets próprios de cada sistema.

Sua estrutura de dados e sets de acesso são apresentados a seguir:

SBGB02 COMPACT DATA SET "USO GERAL"

  G0200-HEADER            GROUP          (Header do registro);
       G0200-HDRDATA      NUMBER (06)    (Data de última atualização do registro);
       G0200-HDRHORA      NUMBER (06)    (Hora de última atualização do registro);
       G0200-HDRESTACAO   NUMBER (03)    (Num. da estação que executou a última atualização do registro);
       G0200-HDRSTATGER   ALPHA  (01)    (Statusdo registro);
   0200-CHAVE             GROUP          (Chave de acesso ao reg.);
       G0200-SISTEMA      ALPHA  (02)    (Código do sistema - <ss;>);
       G0200-SUBSIST      ALPHA  (01)    (Código do subsistema - <s>);
       G0200-ARQUIVO      ALPHA  (02)    (Tipo reg - arquivo lóg.);
       G0200-REGISTRO     ALPHA  (02)    (Número do registro);

           G0200-COMPL)

A identificação do sistema e subsistema de um registro do dataset de uso geral do SADS - SBGB02 - é feita através dos campos da chave G0200-SISTEMA e G0200-SUBSIST.

Para um mesmo sistema, podem existir vários tipos de registros ou arquivos lógicos, que serão diferenciados pelo campo da chave G0200-ARQUIVO.

Cada arquivo lógico pode possuir diferentes registros, cada um deles armazenando um tipo diferente de informação, que serão identificados pelo campo da chave G0200-REGISTRO.Para cada um destes registros deverá haver um cadastramento de sua estrutura no repositório de dados do SADS.

As várias ocorrências de um mesmo registro são diferenciadas pelo campo G0200-COMPLEMENTO (G0200-COMPL e G0200-COMPL2).

Após fazer acesso ao dataset SBGB02, o conteúdo do campo G0200-AREAREG deverá ser movido primeiramente para o campo respectivo da librarydo registro, para que possa ser utilizado pelo aplicativo.

Da mesma forma, para compor o campo G0200-AREAREG, deve-se movimentar os dados primeiramente para o campo auxiliar da library e, depois deste, para o campo G0200-AREAREG.

Na primeira gravação da ocorrência do registro no dataset SBGB02, o campo G0200-TAMREG deverá ser composto com a quantidade de bytes que o campo G0200-AREAREG ocupará.

Tomando novamente como exemplo a Área de Recursos Humanos de uma empresa (ARH), temos as seguintes informações:

  1. Código do cargo de programador sênior;
  2. Faixa salarial de programador sênior;
  3. Matrícula do último programador sênior admitido;
  4. Código do cargo de gerente técnico;
  5. Faixa salarial de gerente técnico;
  6. Matrícula do último gerente técnico admitido;
  7. Período fiscal em aberto.

Analisando estas informações tem-se que, para o sistema ARH, os dois primeiros grupos de três (de 1 a 3 e de 4 a 6) contêm o mesmo tipo de informação e por isso mesmo podem ser armazenadas em um registro de mesma estrutura - ARHG0101.

Já a sétima informação possui conteúdo diferente e, por isso mesmo, deverá ser criado novo registro para armazená-la - ARHG0102;

As três primeiras informações (de 1 a 3) deverão ser armazenadas em um só registro, pois referem-se ao cargo de Programador Sênior, e as próximas três (de 4 a 6) também deverão ser armazenadas em um mesmo registro, pois referem-se estritamente ao cargo Gerente Técnico.

Seguindo este raciocínio, conclui-se que para cada tipo de informação deve-se criar um registro específico que irá armazená-la.

Manipulação de registros do dataset de uso geral

O SADS fornece funções que permitem a manipulação das ocorrências dos registros armazenados no dataset de uso geral de seu próprio banco - SBGB02 do SDDB01. Estas funções englobam inclusão, alteração, consulta e exclusão dos registros.

No SADS, o sistema responsável por executar estas funções é o SBA, cujos fontes, desenhos de telas e libraries são cedidos ao cliente quando da instalação do SADS.

Isto se deve ao fato de que o SADS visa permitir que todas estas funções possam ser executadas também para os registros do dataset de uso geral dos bancos dos aplicativos.

Porém, para isto, devem ser efetuados os seguintes procedimentos:

  • Os desenhos das telas do SBA - SBAT0100, SBAT0101 e SBAT0102 - devem ser duplicados com outros códigos. Cumpre ressaltar que, no momento da duplicação, as novas telas não devem ser convertidas para a versão 3.0 do SADS. As libraries das telas também devem ser duplicadas;
  • Os fontes dos programas cedidos - SBAN01 e SBAN11 - devem ser alterados para fazer acesso aos novos códigos de telas;
  • A tela do menu deve ser alterada para exibir o código do banco, cujo datasetde uso geral será manipulado;
  • O código do banco na library - SB/A/BL/00 - deverá também ser alterado para o novo código;
  • O host e os programas chamados devem ser compilados;
  • O binder deve ser efetuado;
  • O cadastramento no utility deve ser efetuado.

Cada ocorrência de registro pode ou não ter uma senha de acesso. Esta senha, caso tenha sido armazenada pelo próprio aplicativo no campo G0200-SENHA da ocorrência, será exigida pelo SBA nos acessos ao mesmo.

Cadastramento

O cadastramento de um registro do dataset de uso geral SBGB02 consiste na definição de sua estrutura no repositório de dados do SADS.

A partir de seu cadastramento poderá ser gerada uma library, que consiste em uma área auxiliar para utilização pelos programas e sub-rotinas dos aplicativos que farão acesso ao respectivo registro.

Um registro do dataset de uso geral (tipo = "G"), desde que sua estrutura não tenha sido definida, pode ser transformado em uma library de procedure (tipo = "P"). Também uma library de procedure, desde que não tenha módulos cadastrados, pode ser transformada em uma >library de registro de dataset de uso geral (tipo = "G").

A definição de um registro do dataset de uso geral pode ser feita separadamente ou gerada a partir de uma entidade já existente no repositório de dados.Para isso, como pré-requisito, é exigido que o seu sistema já esteja cadastrado no repositório de dados do SADS. Entretanto, se o registro do dataset de uso geral for gerado a partir de uma entidade, é necessário, também, que esta já tenha sido definida no repositório de dados, bem como sua estrutura e os atributos que a compõem.

Roteiro

Para cadastrar e tornar disponível um registro do dataset de uso geral, as seguintes funções devem ser executadas:

  • Cadastramento de tipo de registro do datasetde uso geral: pode ser feito através de sua criação a partir de entidade ou definição direta no repositório de dados do SADS;
  • Cadastramento de estrutura de registro do dataset de uso geral;
  • Geração de library de registro do dataset de uso geral;
  • Liberação de library de registro do dataset de uso geral para produção.
Cadastramento de tipo de registro do dataset de uso geral

O cadastramento de registro do dataset de uso geral no repositório de dados do SADS é necessário para que o mesmo possa ser utilizado como recurso pelos programas e sub-rotinas.

Criação de registro do dataset de uso geral a partir da entidade

Para gerar o registro do dataset de uso geral de um sistema a partir de alguma entidade, é necessário que a estrutura da mesma esteja cadastrada no repositório de dados do SADS.

A geração de registro do dataset de uso geral a partir de entidade não está associada a nenhuma tela de processamento. Portanto, para efetuá-la, basta transmitir os argumentos da tela "CRIREGENT", cujos campos estão explicados a seguir:

CAMPOS DA TELA

Registro - Código do registro do dataset de uso geral no padrão SADS (sssGxnmm).

O indicador numérico "mm" do registro do dataset de uso geral deve ser 01, se ainda não estiver cadastrado. Se o registro já estiver cadastrado, informe uma estrutura que não esteja cadastrada, mas que seja compatível com a quantidade informada no campo REGISTROS da definição.

Entidade - Código da entidade que servirá de base para a estrutura do registro do dataset de uso geral.

Definição de registro do dataset de uso geral

Utilizando-se da mesma tela de definição de registro do dataset de uso geral, podem ser definidas, também, as áreas de comunicação de sub-rotinas, libraries de procedure e áreas de trabalho. Por isso, existem nela alguns campos que não devem ser preenchidos no momento da definição do registro do dataset de uso geral. Os campos que compõem a tela de definição do registro estão explicados a seguir:

CAMPOS DA TELA

Elemento - Código do tipo do registro do dataset de uso geral no padrão SADS (sssGxn).

Geracao completa pelo SADS (S/N) - Controle de geração da library do dataset de uso geral, diferenciando libraries criadas no CANDE das geradas pelo SADS.

Pode assumir os seguintes valores:

"S" - Indicador de library gerada pelo SADS;

"N" - Indicador de library definida no repositório de dados do SADS, e criada manualmente no CANDE.

Este campo será utilizado quando uma compilação de programa ou sub-rotina que declare a library do datasetde uso geral como recurso, estiver com o campo COMPILAR MESMO QUE OS RECURSOS ESTEJAM DESATUALIZADOS preenchido com o valor "NAO". Neste caso, se a library do dataset de uso geral estiver com "S" será verificado o seu status de geração.

Nome - Descrição resumida do tipo de registro do dataset de uso geral.

O nome do tipo de registro tem o objetivo de identificar e explicar resumidamente a que se refere o registro do dataset de uso geral.

Registros- Quantidade de estruturas (layouts) que o registro terá (01 a 99).

Tipo - Tipo da library a ser gerada.

Para registros do datasetde uso geral deve-se informar "E" (estrutura).

Lib. de PROCEDURE p/ uso em:

Sub-rotina (S/N)- Não se aplica a dataset de uso geral.

On-line (S/N)- Não se aplica a dataset de uso geral.

Batch (S/N)- Não se aplica a dataset de uso geral.

Operacao- Operação efetuada sobre a definição do registro do dataset de uso geral.

As operações possíveis são as seguintes:

"INC" (Inclusão);

"ALT" (Alteração);

"EXC" (Exclusão);

"CON" (Consulta).

Cadastramento de estrutura de registro do dataset de uso geral

No SADS, cada registro do dataset de uso geral pode possuir mais de uma estrutura de dados. Estas estruturas são formadas por atributos e/ou campos.

A partir da estrutura de dados de um registro é gerada sua library, que posteriormente será incorporada aos programas que manipulam tal registro.

Para cadastrar a estrutura do registro do dataset de uso geral, é necessário que sua definição esteja cadastrada no repositório de dados do SADS.

CAMPOS DA TELA

ELEMENTO- Código do tipo do registro do dataset de uso geral no padrão SADS (sssGxn).

REGISTRO- Número da estrutura do registro.

Em caso de encadeamento, após a definição do registro, este campo virá preenchido com "01".

REDEFINE REGISTRO - Número do registro que terá sua estrutura redefinida pela estrutura em cadastramento.

Este campo somente deverá ser informado se a estrutura em cadastramento for redefinir algum registro. Caso sejam independentes, este campo não deverá ser informado.

NOME- Descrição resumida do registro do datasetde uso geral.

O nome do registro tem o objetivo de identificar e explicar resumidamente a que se refere o registro do dataset de uso geral.

Nos grupos de ocorrência da tela, deve-se informar os atributos e campos que compõem o registro.

SEQ- Número para posicionamento do atributo dentro da estrutura do registro do dataset de uso geral.

Este campo já vem informado, sendo o primeiro igual a "0010" e os demais, acrescidos de 10.

Para incluir um atributo entre dois outros já incluídos, deve-se preencher uma linha que estiver em branco e alterar o número sequencial desta linha para um número entre os já informados. O SADS reorganizará automaticamente a estrutura. Na próxima operação, os atributos já estarão na nova ordem.

ATRIBUTO- Mnemônico de atributo/campo.

SUF - Sufixo do atributo/campo.

Esta informação é utilizada somente para diferenciar um mesmo atributo ou campo que aparece mais de uma vez na mesma estrutura.

NIV- Nível de estruturação do atributo/campo, análogo ao nível do Cobol.

O nível inicial é o 10, se for necessário criar agrupamentos de atributos, deve-se usar múltiplos de 5 (15, 20 e assim por diante), de forma semelhante ao Cobol.

NDEF- Nível de definição do atributo/campo.

Os valores possíveis são os seguintes:

"S" (Sistema) - Para atributo/campo definido no sistema informado no campo SIS ou para atributo/campo do sistema do registro do dataset de uso geral em cadastramento;

"P" (Particular) - Para atributo/campo não definido no repositório de dados do SADS de utilização específica do registro do dataset de uso geral em cadastramento.

Não pode ser informado NDEF = "P" caso tenha sido parametrizado na tabela SBPM5000 que não se pode definir atributos/campos particulares nas estuturas SADS.

Maiores detalhes a respeito desta parametrização, deve-se consultar a Seção V, "Parametrizações do SADS", item "Atributo", subitem "Redefinição em estruturas".

"G" (Global) - Para atributo/campo já definido no repositório do SADS para utilização genérica dos sistemas aplicativos.

SIS - Código do sistema ao qual o atributo/campo pertence.

Só deve ser informado quando o campo NDEF estiver preenchido com "S" e o atributo/campo pertencer a um sistema diferente do sistema do registro do dataset de uso geral.

COMP- Indicador de atributo/campo numérico compactado.

Deve-se informar "S" para atributo/campo numérico compactado no registro;

FORM- Formato do atributo/campo na estrutura do registro.

Os valores possíveis são os seguintes:

"A" (Alfabético);

"N" (Numérico);

"D" (Display)- Alfanumérico;

"R" (Redefinição);

"B" (Boolean).

Este campo não pode ser informado caso tenha sido parametrizado na tabela SBPM5000 que os atributos/campos não podem ter seus formatos redefinidos nas estruturas nas quais são incluídos.

Maiores detalhes a respeito desta parametrização, deve-se consultar a Seção V, "Parametrizações do SADS", item "Atributo", subitem "Redefinição em estruturas".

TAM - Tamanho do atributo/campo.

Este campo não pode ser informado caso tenha sido parametrizado na tabela SBPM5000 que os atributos/campos não podem ter seus tamanhos redefinidos nas estruturas nas quais são incluídos.

Maiores detalhes a respeito desta parametrização, deve-se consultar a Seção V, "Parametrizações do SADS", item "Atributo", subitem "Redefinição em estruturas".

SIN - Indicadorde sinalização do atributo/campo.

Os valores possíveis são os seguintes:

"S" - Para atributo/campo com sinal;

"N" - Para atributo/campo sem sinal.

DEC - Quantidade de casas decimais.

Os campos FORM, TAM, SIN e DEC só devem ser informados para atributos e campos particulares ou quando, no registro em cadastramento, assumirem características diferentes das definidas para eles.

CLAS - Classificaço do atributo dentro da estrutura do registro de uso geral.

Pode-se assumir os seguintes valores:

"C" (Chave) - Atributo definido como chave do registro de uso geral.

"P" (Particular) - Atributo não definido no repositório de dados do SADS, a ser especificado na própria estrutura.

OCCURS - Número de ocorrências do atributo/campo para criação de grupos repetitivos.

O campo OCCURS é semelhante à cláusula "OCCURS" das estruturas do Cobol e é usado para definir tabelas ou grupos de ocorrências.

CONTINUA(S/N)- Indicador de continuidade de operação.

As possibilidades para o preenchimento deste campo são as seguintes:

"S" - Para prosseguir o cadastramento da mesma estrutura;

"N" - Para encerrar o cadastramento desta estrutura e, se houver, iniciar o da próxima.

OPERACAO - Operação efetuada em relação à estrutura do registro do dataset de uso geral.

As operações possíveis são as seguintes:

"INC" (Inclusão);

"ALT" (Alteração);

"EXC" (Exclusão);

"CON" (Consulta).

Geração de library de registro do dataset de uso geral

Cada tipo de registro do dataset de uso geral tem de possuir ao menos uma estrutura de dados com seus atributos e/ou campos. A partir desta estrutura de dados é gerada a library para ser incorporada aos programas que manipulam tal registro. Um tipo de registro pode possuir mais de uma estrutura e cada uma destas estruturas dará origem a uma library independente, mesmo que estes registros não sejam redefinições.

Para gerar as libraries de tipo de registro do dataset de uso geral, é necessário que suas estruturas já estejam cadastradas no repositório de dados do SADS.

A geração da libraryde tipo de registro do dataset de uso geral não está associada a nenhuma tela de processamento. Portanto, para efetuá-la, basta transmitir o menu de operação "GERLIBUSO", cujos campos estão explicados a seguir:

CAMPOS DA TELA

Registro- Código do tipo de registro do uso geral no padrão SADS (sssGxn).

Ambiente (P/T) - Ambiente em que a library será gerada.

Os valores possíveis são os seguintes:

"T" (Teste);

"P" (Produção).

Caso o ambiente não seja informado, será assumido o ambiente defaultdo usuário. Detalhes sobre o cadastramento do ambiente default do usuário podem ser obtidos na seção VI, "Sistema de Segurança", item "Usuários e habilitações".

Se uma library for gerada para o ambiente de teste, não é possível gerá-la novamente para o ambiente de produção. Neste caso, ela deverá ser liberada, conforme descrito no item "Liberação de library de registro do dataset de uso geral para produção", a seguir.

Usercode - Usercode de análise.

O usercode a ser informado deve estar habilitado ao usuário, como elemento, em sua lista de elementos "SEGUSER", no Sistema de Segurança. Detalhes sobre este cadastramento estão descritos na Seção VI, "Sistema de Segurança", item "Habilitações de usuários SADS".

Caso o usercode ão seja informado, será assumido o usercode default do usuário. Detalhes sobre o cadastramento do usercode default do usuário podem ser obtidos na seção VI, "Sistema de Segurança", item "Usuários e habilitações".

Para usercodes de análise são definidos os ambientes (usercode e disco) onde ficarão residentes suas libraries de teste e produção, sendo possível que as libraries de produção fiquem sem usercodes. Estas definições estão na tabela SBPM5002, cuja manutenção é feita pelo responsável pelo SADS na sua instalação.

O código da library é derivado do código da estrutura do registro.

Os prefixos dos campos de library de registro de dataset de uso geral têm a seguinte formação:

Terceira letra do sistema ("H", no exemplo) + a letra "G" (que identifica um registro do SBGB02) + o sequencial do registro ("01", no exemplo) + o número da estrutura do registro ("01" e "02", no exemplo) + "-" (constante) + o mnemônico do atributo ou campo.

Liberação de library de tipo de registro do dataset de uso geral para produção

É permitido liberar libraries de tipo de registro do dataset de uso geral do ambiente de teste para o ambiente de produção.

Para liberar uma library para produção, é necessário que esta tenha sido gerada inicialmente para o ambiente de teste.

Se a library não tiver sido gerada para o ambiente de teste, pode-se gerá-la diretamente para o ambiente de produção.

A liberação da library de tipo de registro do dataset de uso geral para produção não está associada a nenhuma tela de processamento. Portanto, para efetuá-la, basta transmitir o menu de operação "LIBREGUSO", cujos campos estão explicados a seguir:

CAMPOS DA TELA

Registro - Código do registro do dataset de uso geral no padrão SADS (sssGxn).

Usercode - Usercodede análise.

O usercode a ser informado deve estar habilitado ao usuário, como elemento, em sua lista de elementos "SEGUSER", no Sistema de Segurança. Detalhes sobre este cadastramento estão descritos na Seção VI, "Sistema de Segurança", item "Habilitações de usuários SADS".

Caso o usercode não seja informado, será assumido o usercode default do usuário. Detalhes sobre o cadastramento do usercode default do usuário podem ser obtidos na seção VI, "Sistema de Segurança", item "Usuários e habilitações".

Para usercodesde análise, são definidos os ambientes (usercode e disco) onde ficarão residentes suas libraries de teste e produção, sendo possível que as libraries de produção fiquem sem usercodes Estas definições estão na tabela SBPM5002, cuja manutenção é feita pelo responsável pelo SADS na instalação.

Após a liberação da library do registro do dataset de uso geral para produção, os programas que utilizem este registro podem ser compilados para produção.

Se na definição do seu usercode de análise, na tabela SBPM5002, estiver indicado que as librariesde produção ficarão residentes no usercode de objetos em produção, ao executar suas liberações, aquelas que estão em teste serão mantidas e versões idênticas serão criadas para o usercode de produção. Se as libraries de produção ficarem residentes sem usercode,ao executar suas liberações, aquelas que estão em teste serão removidas e outras idênticas serão criadas em *, ou seja, sem usercode, para atender aos dois ambientes (teste e produção).
