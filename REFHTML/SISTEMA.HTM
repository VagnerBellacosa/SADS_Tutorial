Sistema

Conceituação

Sistema é uma coleção de elementos discretos, interdependentes, que se interagem com o objetivo de atingir a uma causa ou meta comum. De maneira genérica, pode ser entendido como um conjunto de processos, dados e estruturas de dados.

Para o SADS, o sistema é o elemento de nível mais elevado, sob o qual estarão definidos todos os elementos dos modelos de dados e processos, sejam eles conceituais, lógicos ou físicos. Portanto o sistema é o primeiro elemento a ser definido no repositório de dados.

Padronização

O padrão de codificação adotado pelo SADS para a identificação de sistemas é o seguinte:

"sss" onde:

"sss" é o sistema.

Cumpre ressaltar que o primeiro caractere alfabético designado para a formação do código do sistema não pode ser "S", pois esta letra é reservada para identificar os sistemas do próprio SADS.

Apesar de não haver impedimentos para que as letras que comporão o código do sistema sejam escolhidas aleatoriamente, é conveniente que se crie uma estruturação adequada que facilite a organização e a administração das informações cadastradas no repositório de dados.

Desta forma, segue uma sugestão de estruturação para este código que, por possibilitar todas estas facilidades, julga-se a mais eficiente:

Suponha uma empresa com três grandes áreas, podendo-se denominar de:

"I", a sua área industrial;

"A", a sua área administrativa;

"F", a sua área financeira.

Supondo-se que na área industrial existem dois grandes grupos distintos de procedimentos, temos:

1º grande grupo: Programação e controle de oficina, representado pelas letras "IO" (Industrial + Oficina);

2º grande grupo: Especificações técnicas, representado pelas letras "IT" (Industrial + Técnicas).

Estes dois grupos de procedimentos podem ser entendidos como os dois grandes sistemas da área industrial. No entanto, não é necessário que eles sejam informatizados desta forma, podendo-se dividi-los em subsistemas menores.

Neste exemplo, o sistema de programação e controle de oficina será implementado a partir de três subsistemas, que serão efetivamente os sistemas desenvolvidos através do SADS.

Portanto, ao escolher o código do subsistema de programação de oficina, deve-se levar em conta que ele pertence ao sistema de programação e controle de oficina, que, por sua vez, pertence a área Industrial. Portanto, seu código deve ser "IOP" (Industrial + Oficina + Programação).

De forma análoga, pode-se codificar o subsistema de controle de fabricação como "IOC" (Industrial + Oficina + Controle) e o subsistema de estatística de oficina como "IOE" (Industrial + Oficina + Estatística).

Seguindo este raciocínio, pode-se concluir que na maioria das vezes são desenvolvidos subsistemas e que as três letras que formam o código do sistema no SADS significam: Área + Sistema + Subsistema, respectivamente.

Apesar de, no exemplo, as áreas estarem representadas por uma letra e os sistemas por duas, pode-se definir áreas com três letras, sem definir sistemas e/ou subsistemas, ou sistemas com uma letra, sem definir subsistemas.

Tomemos como exemplo a letra "G", cujo significado pode ser "Geral". Para definir a área Industrial, o código adotado deve ser "IGG" (Industrial + Geral + Geral) e o do sistema de Programação e Controle de Oficina "IOG" (Industrial + Oficina + Geral).

A letra usada para identificação da Hierarquia Funcional entre áreas, sistemas e subsistemas ("G", no nosso exemplo) não poderá ser utilizada para identificar subsistemas. Assim, a letra escolhida somente poderá ser utilizada para representar áreas.

Lembrando que esta não é uma regra absoluta de codificação de sistemas, diferentes formas de codificação poderão ser adotadas pela empresa. Portanto, antes de efetuar o cadastramento de um novo sistema, deve-se verificar com a Administração de Dados ou com o responsável pelo SADS o padrão a ser seguido.

Recomenda-se que, quando houver necessidade de dois sistemas se integrarem, o último caractere alfabético designado para a formação de seus códigos não seja o mesmo. Isto para evitar ambigüidade na identificação dos atributos destes sistemas que irão possuir este mesmo último caractere em seus mnemônicos. Ocorrendo isto, será necessário efetuar uma complementação na identificação dos atributos (atributo OF).

Utilização

Conforme descrito na conceituação de sistema, qualquer elemento do SADS - entidade, atributo, programa, dataset, arquivo, tela etc - só pode ser definido a partir do cadastramento de seu sistema no repositório de dados do SADS.

Quando um sistema utiliza elementos definidos em outros sistemas, no próprio código destes elementos está indicado em qual dos sistemas ele foi cadastrado. Porém, quando se quer referenciar atributos, campos e domínios cadastrados em outros sistemas, será necessário indicar o sistema sob o qual o elemento foi definido.

As estatísticas de evolução do repositório de dados são geradas por tipo de sistema e, também, por sistema. Estas estatísticas estão descritas na Seção III, "Funções de apoio", neste manual.

Definição de sistema.

Definição de sistema

CAMPOS DA TELA

SISTEMA - Código do sistema no padrão SADS (sss).

NOME - Descrição resumida do sistema.

O nome tem por objetivo identificar e explicar resumidamente a que o sistema se refere.

SITUAÇÃO DO SISTEMA - Situação atual do sistema.

Os valores possíveis são os seguintes:

"D" - Para sistema em desenvolvimento;

"P" - Para sistema em produção.

A situação do sistema é puramente documental, não implicando, pois, em nenhuma diferença prática no desenvolvimento do sistema. Esta situação não é alterada dinamicamente pelo SADS.

CLASS. QUANTO AO DESENVOLVIMENTO - Classificação quanto a forma como o sistema será desenvolvido.

Os valores possíveis são os seguintes:

"PP" (próprio e padronizado) - Para sistemas desenvolvidos para a própria empresa e
que obedecem aos padrões opcionais oferecidos pelo SADS.

"PN" (próprio não padronizado) - Para sistemas desenvolvidos para a própria empresa
e que não obedecem aos padrões opcionais oferecidos pelo SADS.

"TP" (terceiros e padronizado) - Para sistemas desenvolvidos por outra empresa e que
obedecem aos padrões opcionais oferecidos pelo SADS.

"TN" (terceiros não padronizado) - Para sistemas desenvolvidos por outra empresa e
que não obedecem aos padrões opcionais oferecidos pelo SADS.

A opção de classificação quanto ao desenvolvimento de um sistema é puramente documental, não implicando em nenhuma diferença prática no desenvolvimento do sistema.

CLASS. QUANTO AO USO - Classificação de acordo com o objetivo do sistema.

Os valores possíveis são os seguintes:

"E" (De uso geral da Empresa) - Subsistemas desenvolvidos para atender a uma área usuária que não seja o próprio CPD;

"I" (De uso restrito a área de informática) - Subsistemas desenvolvidos para atender necessidades próprias da área de informática;

"T" (Treinamento) - Subsistemas desenvolvidos em seções de treinamento;

"D" (Demonstração) - Subsistemas preparados para utilização em demonstrações para clientes ou usuários;

"S" (Controle) - Subsistemas criados unicamente para abrigar definições genéricas da empresa. Exemplo: Subsistema onde serão definidos os bancos de dados ou campos de controle;

"H" (Hierarquia funcional) - Subsistemas cujos códigos (sss) servem para identificar áreas ou sistemas, onde sistema é um agrupamento de subsistemas e área é um agupamento de sistemas. Para maiores informações, consulte o tópico "Padronização" descrito anteriormente, neste mesmo item.

A informação de classificação quanto ao uso é utilizada apenas para a composição das estatísticas de evolução de cadastramentos no repositório de dados do SADS.

INDICADORES:

COOPERATIVO (T/P/A) - Indica se o sistema é cooperativo (integrado ao produto "COOPERATOR" da MSA-INFOR) nos ambientes de teste, produção ou ambos.

Os valores possíveis são os seguintes:

"T" - O sistema é coooperativo no ambiente de teste;

"P" - O sistema é cooperativo no ambiente de produção;

"A" - O sistema é cooperativo em ambos os ambientes.

Se este campo não for informado, será assumido o valor default, isto é, que o sistema não é cooperativo, ou seja, não é integrado ao COOPERATOR.

UTILIZACAO SUB-ROTINAS GLOBAIS (S/N) - Indicador de utilização de sub-rotinas globais pelo sistema.

Pode assumir os seguintes valores:

"S" - Para indicar que o sistema utiliza sub-rotinas globais nos ambientes de teste e produção;

"N"- Para indicar que o sistema não utiliza sub-rotinas globais em ambos os ambientes;

"T" - Para indicar que o sistema utiliza sub-rotinas globais somente no ambiente de teste.

Este valor não pode ser informado pelo usuário, pois somente o programa responsável pela conversão de sistemas para utilização de sub-rotinas globais atualiza-o com o valor "T".

Cumpre ressaltar que existem algumas restrições no preenchimento do campo indicador de utilização de sub-rotinas globais pelo sistema, que estão vinculadas à parametrização de sub-rotinas globais na instalação, efetuada na tabela SBPM5000, quais sejam:

Só pode ser informado "S" no campo indicador de utilização de sub-rotinas globais pelo sistema, caso a utilização destas na instalação seja obrigatória ou livre (Tabela SBPM5000, CODIGO = "SUBGBL", PAR-3 = "O" ou "L"); e

Só pode ser informado "N" no campo indicador de utilização de sub-rotinas globais pelo sistema, caso a utilização destas na instalação seja inibida ou livre (Tabela SBPM5000, CODIGO = "SUBGBL", PAR-3 = "I" ou "L").

Uma vez definido que o sistema utilizará sub-rotinas globais, não haverá possibilidade de mudar esta opção. Em outras palavras, se o campo "UTILIZACAO SUB-ROTINAS GLOBAIS (S/N)" for igual a "T" ou "S", este não poderá ser alterado para "N".

DESCRICAO - Descrição detalhada do sistema.

Estão disponíveis 99 páginas para compor a descrição de um sistema.

CONTINUA - Indicador de continuidade da operação.

As possibilidades de preenchimento deste campo são as seguintes:

"S" - Nova tela será aberta para prosseguir descrevendo o sistema, caso esta página não seja suficiente;

"N" - Última página da descrição do sistema.

OPERACAO - Operação efetuada sobre o sistema.

Pode assumir os seguintes valores:

"INC" (Inclusão);

"ALT" (Alteração);

"EXC" (Exclusão);

"CON" (Consulta).
