Base de dados

Conceituação

A base de dados compreende os bancos de dados e os datasets que serão utilizados pelos programas e sub-rotinas do sistema.

Um sistema pode possuir mais de uma base de dados, dependendo da necessidade de utilização de seus hosts, programas batch e sub-rotinas.

Não é obrigatório que um sistema trabalhe com bancos de dados. Neste caso, a definição de uma base de dados também não é necessária.

Uma base de dados pode ser genérica para todo o sistema, possuindo até dois bancos de dados, ou específica por elemento(s) do sistema, com até dez bancos de dados. Desta forma, os elementos de um sistema - hosts, programas batch e sub-rotinas - podem utilizar diferentes bases de dados, que podem ser específicas para cada um deles ou específicas para um grupo deles.

O SADS faz o tratamento diferenciado das bases de dados específicas e genéricas, da seguinte maneira:

Para as bases de dados específicas, cujos códigos são identificados por "sssB00xx", onde "xx" é obrigatoriamente diferente de zeros:

É permitido cadastrar até dez bancos de dados.

Não é permitido determinar os datasets que são utilizados exclusivamente pelos programas batch e sub-rotinas. Caso sejam específicos, outra base de dados deverá ser criada para atendê-los.

As libraries destas bases de dados têm código "ss/s/BL/00/xx".

Os comandos de open dos bancos, internos à library, são gerados considerando o tipo de utilização dos mesmos no cadastramento da base de dados.

Para as bases de dados genéricas, cujos códigos são identificados por "sssB0000", onde "xx" é obrigatoriamente igual a zeros :

É permitido cadastrar apenas dois bancos de dados.

É permitido determinar os datasets que são utilizados exclusivamente pelos programas batch e sub-rotinas.

As libraries destas bases de dados têm código "ss/s/BL/00".

Os comandos de open dos bancos, internos à library, são gerados independentemente do tipo de utilização dos mesmos no cadastramento da base de dados.

No cadastramento de uma base de dados, somente um banco pode ser aberto para atualização de dados (update). Portanto, os demais somente podem ser abertos para consulta de dados (inquiry).

Padronização

O padrão de codificação adotado pelo SADS para base de dados é o seguinte:

"sssBOOxx" onde:

"sss" é o código alfabético que identifica o sistema;

"B" é a constante que identifica estruturas de bancos de dados;

"OO" é o identificador numérico do elemento base de dados e,

"xx" é o identificador alfanumérico da base de dados.

Para bases de dados com identificador alfanumérico igual a "00"- base de dados genérica do sistema- só é permitido cadastrar dois bancos de dados. Já para as bases de dados com identificador alfanumérico diferente de "00"- base de dados específica - é permitido cadastrar até dez bancos de dados diferentes.

No processo de conversão para a versão 4.0, todas as bases de dados já existentes no repositório de dados foram convertidas com o identificador alfanumérico da base de dados igual a "00", compondo o código no seguinte formato: "sssB0000".

Utilização

A definição da base de dados é utilizada na geração de sua library - ss/s/BL/00/xx, que poderá ser incorporada aos hosts, programas e sub-rotinas do sistema.

É nesta library que são geradas as declarações dos bancos de dados que estarão disponíveis para os hosts, programas batch e sub-rotinas do sistema e os comandos de Open e Close dos mesmos. São geradas, também, as declarações dos datasets para utilização nos programas on-line e os comamdos FREE para estes datasets.

Para que se torne possível a implantação parcial de um sistema no ambiente de produção, a base de dados deste pode ser composta por parte dos datasets do ambiente de desenvolvimento.

Cadastramento

A definição da base de dados só é possível a partir do cadastramento do sistema, dos bancos e dos datasets  no repositório de dados do SADS.

Além disso, os datasets devem estar relacionados aos bancos de dados que forem indicados na definição da base de dados do sistema.

Roteiro

Para cadastrar uma base de dados e torná-la disponível para utilização pelo sistema, as seguintes funções deverão ser executadas:

Definição de base de dados;

Cadastramento de estrutura de base de dados;

Geração de library de base de dados;

Geração de library de base de dados com seleção de datasets.

Definição de base de dados

Na definição da base de dados, deverão ser especificados seu código e nome.

NOME - Nome da base de dados.

Cadastramento de estrutura de base de dados

O cadastramento da estrutura da base de dados é efetuada através do Paginador de Entrada do SADS (STP05). Cada página a ser incluída corresponde ao cadastramento de um banco de dados e de todos os datasets deste mesmo banco que poderão ser utilizados pelos hosts, programas batch e sub-rotinas do sistema.

Cada uma destas páginas, além dos campos utilizados pelo paginador - PAGINA e PROXIMA PAGINA, possui os seguintes campos:

CAMPOS DA TELA

Base de Dados - Código alfabético que identifica a base de dados, no padrão SADS (sssB00xx).

Após o código, é apresentado o nome da base de dados.

Banco de Dados - Código do banco de dados, no padrão SADS (sssBxx).

Utilizacao (I/U) - Critério de utilização do banco de dados informado.

Os valores possíveis são os seguintes:

"I" (Inquiry) - Para banco de dados utilizado apenas para a operação de consulta;

"U" (Update) - Para banco de dados utilizado para atualização.

"INVOKE" de visoes (S/N) - Indicador de geração de invokes de diferentes visões de um mesmo set.

Pode assumir os seguintes valores:

"S" - Para que sejam gerados, na library de declaração de banco, os invokes de diferentes visões dos sets, caso existam cadastradas;

"N" - Para que seja gerado um único invoke do set.

Datasets para composicao da library da Base de Dados - Códigos dos datasets pertencentes ao banco que serão utilizados pelos hosts, programas batch e/ou sub-rotinas do sistema, no padrão SADS (sssBxx).

Os datasets deverão estar previamente cadastrados e relacionados aos respectivos bancos de dados.

Estes datasets irão compor a library da base de dados. Já a library poderá ser utilizada pelos seguintes elementos do sistema:

Hosts;

Programas on-line;

Neste caso, a base de dados não necessita ser especificada, pois será a mesma do host relacionado ao programa.

Programas batch.

Para os programas batch, somente as declarações dos bancos e seus comandos de OPEN e CLOSE serão utilizados a partir da library da base de dados. Os datasets terão suas declarações geradas diretamente nos programas.

Caso se esteja definindo uma base de dados genérica do sistema - Código da base igual a "sssB0000"- podem ser declarados datasets que não irão fazer parte da library da base e que serão utilizados apenas pelos programas batch e sub-rotinas.

Sub-rotinas.

Funcionamento de forma idêntica a dos programas batch.

Datasets para utilização exclusiva em programas batch e sub-rotinas - Códigos dos datasets pertencentes ao banco, informados no padrão SADS (sssBxx), que serão utilizados apenas por programas batch e sub-rotinas do sistema, ou seja, não serão utilizados em programas on-line.

Quando se estiver definindo a base de dados genérica do sistema ("sssB0000"), os datasets, quando forem utilizados exclusivamente em programas batch e sub-rotinas, devem obrigatoriamente ser declarados. Estes datasets não serão utilizados na composição da library da base de dados, porém serão utilizados na validação dos datasets definidos como recursos de programas batch e sub-rotinas.

A definição dos datasets para utilização exclusiva em programas batch e sub-rotinas só será permitida quando se estiver cadastrando uma base de dados genérica para todo o sistema - Base de dados com código "sssB0000". Caso contrário, se a base de dados for específica - Base de dados com código "sssB00xx", onde "xx" é obrigatoriamente diferente de "00" - deve-se então, se necessário, definir uma outra base de dados, que contenha tais datasets, que irá atender especificamente aos programas batch e/ou sub-rotinas do sistema.

Os datasets informados deverão estar previamente cadastrados e relacionados aos respectivos bancos de dados no repositório de dados.

OPERACAO - Operação a ser efetuada em relação ao cadastramento da base de dados do sistema.

As operações possíveis são as seguintes:

"INC" (Inclusão);

"ALT" (Alteração);

"EXC" (Exclusão);

"CON" (Consulta).

Geração de library de base de dados

A library de base de dados pode ser utilizada pelos hosts, programas e sub-rotinas do sistema.

É através dessa library que o host executa comandos de Open e Close do banco e que os datasets são declarados dentro dos programas on-line do sistema.

A geração da library da base de dados do sistema é imprescindível à compilação dos programas, caso o sistema utilize bancos de dados.

Para a geração de uma library de base de dados, é exigido apenas que a base de dados esteja definida no repositório de dados.

Para as bases de dados específicas - base de dados com código "sssB00xx", onde "xx" é obrigatoriamente diferente de zeros - sua library será gerada com o seguinte código: "ss/s/BL/00/xx". Porém, para as bases de dados genéricas dos sistemas - base de dados com código "sssB00xx", onde "xx" é obrigatoriamente igual a zeros - sua library será gerada com o seguinte código: "ss/s/BL/00".

Os comandos de open dos bancos, internos à library de base de dados específicas, são gerados considerando o tipo de utilização dos mesmos no cadastramento da base de dados.

Ao contrário das bases de dados específicas, internos à library de base de dados genéricas, os comandos de open dos bancos são gerados independentemente do tipo de utilização dos mesmos no cadastramento da base de dados.

A geração da library da base de dados não está associada a nenhuma tela de processamento. Para efetuá-la, basta transmitir o menu de operação "GERBAS", cujos campos estão explicados a seguir:

CAMPOS DA TELA

Base de dados - Código da base de dados no padrão SADS (sssB00xx).

Ambiente (P/T) - Ambiente onde será gerada a library da base de dados.

Os valores possíveis são os seguintes:

"P" - Para gerar a library no ambiente de produção;

"T" - Para gerar a library no ambiente de teste.

Caso o ambiente não seja informado, será assumido o ambiente default do usuário. Detalhes sobre o cadastramento do ambiente default do usuário podem ser obtidos na seção VI, "Sistema de Segurança", item "Usuários e habilitações".

Quando a library da base de dados for gerada para o ambiente de teste, será assumido o banco de teste relacionado ao banco de produção definido na base de dados.

Se os bancos de teste e produção forem diferentes, a library de teste não poderá ser liberada, requerendo, portanto, gerações específicas para teste e produção.

Para tornar possível a implantação parcial de um sistema no ambiente de produção, a base de dados pode ser composta por parte dos datasets do ambiente de desenvolvimento. Para isto, é necessário utilizar-se da geração de library de base de dados com seleção de dataset.

Usercode - Usercode de análise no qual está sendo desenvolvido o sistema.

O usercode a ser informado deve estar habilitado ao usuário, como elemento, em sua lista de elementos "SEGUSER", no Sistema de Segurança. Detalhes sobre este cadastramento estão descritos na Seção VI, "Sistema de Segurança", item "Habilitações de usuários SADS".

Caso o usercode não seja informado, será assumido o usercode default do usuário. Detalhes sobre o cadastramento do usercode default do usuário podem ser obtidos na seção VI, "Sistema de Segurança", item "Usuários e habilitações".

Para usercodes de análise são definidos os ambientes (usercode e disco) onde ficarão residentes as libraries de teste e produção, sendo possível que as libraries de produção fiquem sem usercodes. Estas definições estão na tabela SBPM5002, cuja manutenção é feita pelo responsável pelo SADS na sua instalação. Esta tabela encontra-se descrita na Seção V, "Parametrizações do SADS", neste manual.

Geração de library de base de dados com seleção de datasets

A library de base de dados, com seleção de datasets, pode ser utilizada em todos os hosts, programas e sub-rotinas do sistema.

É através dessa library que o host em produção executa comandos de Open e Close do banco e que os datasets são declarados dentro dos programas on-line do sistema em produção.

A geração da library de base de dados com seleção de datasets é utilizada para implantação parcial de um sistema em produção.

Para a geração de uma library de base de dados com seleção de datasets é exigido apenas que a base de dados esteja definida no repositório de dados.

Para efetuar a geração da library da base de dados com seleção de datasets, basta transmitir o menu de operação "SELDAT" e, após a transmissão deste menu, em telas paginadas, marcar um "X" nos datasets que fizerem parte da base de dados em produção.

Os campos do menu "SELDAT" estão explicados a seguir:

CAMPOS DA TELA

Base de dados - Código da base de dados no padrão SADS ("sssB00xx").

Usercode - Usercode de análise no qual está sendo desenvolvido o sistema.

O usercode a ser informado deve estar habilitado ao usuário, como elemento, em sua lista de elementos "SEGUSER", no Sistema de Segurança. Detalhes sobre este cadastramento estão descritos na Seção VI, "Sistema de Segurança", item "Habilitações de usuários SADS".

Caso o usercode não seja informado, será assumido o usercode default do usuário. Detalhes sobre o cadastramento do usercode default do usuário podem ser obtidos na seção VI, "Sistema de Segurança", item "Usuários e habilitações".

Para usercodes de análise são definidos os ambientes (usercode e disco) onde ficarão residentes as libraries de teste e produção, sendo possível que as libraries de produção fiquem sem usercodes. Estas definições estão na tabela SBPM5002, cuja manutenção é feita pelo responsável pelo SADS na instalação. Esta tabela encontra-se descrita na Seção V, "Parametrizações do SADS", neste manual.

Os campos da tela de processamento estão explicados a seguir:

Base de Dados - Código da base de dados cuja library será gerada.

Banco - Código do banco de dados informado no cadastramento da base de dados.

Tipo - Indicativo do tipo de utilização do banco: inquiry ou update.

Indicador de seleção de dataset - Deve ser marcado um "X" no campo que antecede cada dataset selecionado para compor a base de dados em produção.

CONFIRME(S/N) - Indicador de confirmação de operação.

Os valores possíveis são os seguintes:

"S" - Para confirmar a geração da library;

"N" - Para não efetuar a geração.

A geração da library é efetuada no encerramento do paginador, ao ser transmitido o valor "99" no campo PROXIMA PAGINA e "S" no campo CONFIRME.

A informação de quais datasets foram selecionados é armazenada e reapresentada na chamada subsequente desta transação.
