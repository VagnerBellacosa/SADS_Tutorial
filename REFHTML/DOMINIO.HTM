Domínio

Conceituação

Domínios são conjuntos de características - formato, padrão, sintaxe, tamanho, decimais, sinal, validades etc - que podem ser associados aos atributos e campos.

O domínio, geralmente, define as características gerais, os valores ou intervalos de valores, que um grupo de atributos/campos poderá assumir.

No repositório de dados, o domínio é identificado pelo seu sistema e mnemônico.

Os domínios podem ser definidos no ET-SADS, que é uma ferramenta UPPER-CASE voltada à aplicação da análise de dados e da análise estruturada e, posteriormente, transferidos para o repositório de dados do SADS. O ET-SADS trabalha na filosofia de estação de trabalho, requerendo uma plataforma IBM-PC ou compatível em ambiente MS-DOS.

As informações relativas à transferência de informações do ET-SADS para o SADS podem ser consultadas no "Guia de Referência do DFD" e no "Guia de Integração com o SADS", manuais que acompanham o ET-SADS.

Padronização

O mnemônico de domínio não possui nenhum padrão de codificação, devendo apenas respeitar o número de caracteres estabelecido - mínimo de 3 e máximo de 11.

Utilização

O domínio, após ter sido cadastrado no repositório de dados, pode ser referenciado no cadastramento de atributos/campos, para evitar recadastramentos idênticos e facilitar as atividades de manutenção.

Se um atributo ou campo for definido como possuidor de um domínio, aquele herdará deste suas características de formato, tamanho, decimais, indicador de sinal, padrão, sintaxe, validades e a tabela/sub-rotina para validação. Deste modo, a tarefa de definição destas características para este atributo ou campo será poupada.

Qualquer alteração efetuada no cadastramento do domínio será propagada para o cadastramento de todos os atributos ou campos a ele relacionados .

Não existe um limite de atributos ou campos que podem possuir o mesmo domínio. Porém, cada atributo ou campo só pode estar relacionado a um único domínio.

Cadastramento

O processo de definição de um domínio só é possível a partir do cadastramento de seu sistema no repositório de dados do SADS.

Apesar do domínio ser cadastrado sob um sistema específico, ele pode ser referenciado por atributos e campos de qualquer sistema.

Roteiro

Para cadastrar um domínio, as seguintes funções devem ser executadas:

Definição de domínio;

Cadastramento de validades de domínio, caso seja variável definida (Sintaxe = "VD").

Definição de domínio

CAMPOS DA TELA

SISTEMA - Sistema sob o qual o domínio está definido.

Todo domínio tem de estar, obrigatoriamente, relacionado a um sistema.

CODIGO - Mnemônico do domínio.

Domínios com mesmo código podem ser cadastrados em sistemas diferentes.

CODIGO EXTERNO - Código externo do domínio.
Este código pode ser informado no caso da instalação adotar um padrão de codificação diferente do padrão SADS ou quando houver integração entre o ET-SADS e o SADS. Neste caso, o código externo do domínio no SADS conterá o código do domínio no ET-SADS.

Exceto para a integração ET-SADS x SADS, o código externo é puramente documental, ou seja, somente será utilizado para identificar, nos relatórios e manuais, a qual elemento externo ao SADS o domínio se refere.

NOME - Descrição resumida do domínio.

O nome tem o objetivo de identificar e explicar, de maneira resumida, a que se refere o domínio.

FORMATO - Formato do domínio.

Indica o tipo do conteúdo do domínio.

Os formatos válidos são:

"A" (Alfabético);

"D" (Alfanumérico ou display);

"N" (Numérico).

Várias características do domínio estão relacionadas com o formato do mesmo. Um exemplo disto é que, só é possível cadastrar sinal e decimais para domínios com formato numérico.

Os testes padrões também devem estar compatibilizados com o formato do domínio. Por exemplo, os testes padrões para "CPF", "M10" e "DMA" estão vinculados a domínios numéricos e assim por diante.

PADRAO - Teste padrão ao qual os atributos e campos relacionados ao domínio serão submetidos para validação de seu conteúdo.

São os seguintes os testes padrões disponíveis:

"AMD" (Data) - Para data no formato ano, mês, dia;

Se o tamanho do atributo ou campo relacionado ao domínio for igual a seis, o teste do ano será feito com duas posições, ou seja, o ano poderá assumir os valores de "00" a "99". Caso seu tamanho seja oito, o teste do ano será feito com quatro posições e, neste caso, o ano poderá assumir os valores de "0000" a "9999".

"DMA" (Data) - Para data no formato dia, mês, ano;

Se o tamanho do atributo ou campo relacionado ao domínio for igual a seis, o teste do ano será feito com duas posições, ou seja, o ano poderá assumir os valores de "00" a "99". Caso seu tamanho seja oito, o teste do ano será feito com quatro posições e, neste caso, o ano poderá assumir os valores de "0000" a "9999".

"D/M" (Data) - Para data no formato dia/mês;

"M/D" (Data) - Para data no formato mês/dia;

"M/A" (Data) - Para data no formato mês/ano;

Se o tamanho do atributo ou campo relacionado ao domínio for igual a quatro, o teste do ano será feito com duas posições, ou seja, o ano poderá assumir os valores de "00" a "99". Caso seu tamanho seja seis, o teste do ano será feito com quatro posições e, neste caso, o ano poderá assumir os valores de "0000" a "9999".

"INV" (Data) - Para data dia, mês, ano com inversão para ano, mês, dia;

Se o tamanho do atributo ou campo relacionado ao domínio for igual a seis, o teste do ano será feito com duas posições, ou seja, o ano poderá assumir os valores de "00" a "99". Caso seu tamanho seja oito, o teste do ano será feito com quatro posições e, neste caso, o ano poderá assumir os valores de "0000" a "9999".

"CPF" (CPF) - Para CPF;

"CGC" (CGC) - Para CGC abrangendo CPF;

"ICG" (CGC) - Para CGC não abrangendo CPF;

"C10" (Mód. 10) - Para cálculo de dígito usando módulo 10;

"C11" (Mód. 11) - Para cálculo de dígito usando módulo 11;

"M10" (Mód. 10) - Para teste de dígito usando módulo 10;

"M11" (Mód. 11) - Para teste de dígito usando módulo 11;

"BRA" - Para teste do preenchimento de apenas um branco entre as palavras de um atributo ou campo relacionado ao domínio;

"CIV" (Estado civil) - Para teste de estado civil: "S" (Solteiro), "C" (Casado), "D" (Divorciado) e "O" (Outros);

"EST" (Estação) - Para teste de estação válida na rede;

Antes de efetuar a validação de um código de impressora na rede, é verificado se tal código está cadastrado na tabela SBPM5024 - Tabela de Impressoras em outra máquina. Caso positivo, tal código é aceito como válido.

Este teste também efetua corretamente a validação de impressoras ligadas ao sistema através de linhas que utilizam o protocolo TCP/IP.

"HOR" (Hora) - Para hora no formato HHMM ou HHMMSS;

Se o tamanho do atributo ou campo relacionado ao domínio for igual a quatro, será consistido horas e minutos, se o tamanho for igual a seis, será consistido horas, minutos e segundos.

"NEZ" - Para teste de atributo ou campo relacionado ao domínio, numérico e diferente de zero.

"PPO" - Para teste do preenchimento da primeira posição do atributo ou campo relacionado ao domínio, cujo conteúdo não pode ser branco;

"RPT" - Para teste de impressão via Reprints;

"SEX" (Sexo) - Para teste de sexo: "F" (Feminino) ou "M" (Masculino);

"S/N" ("S" ou "N") - Para teste de resposta: "S" (Sim) ou "N" (Não).

Estes padrões, quando definidos para o domínio, estabelecem para a consistência geral os testes aos quais serão submetidos os atributos e campos relacionados ao domínio.

Os testes de consistência de data, opcionalmente, aceitam que uma data seja totalmente preenchida com o algarismo 9.

Para fazer opção por este modo de funcionamento, o campo "DATA C/ 9's", da tabela SBPM5004, deve ser preenchido com "S", para um sistema específico ou para o sistema "SBC" quando se desejar configurar para todos os sistemas da instalação.

Maiores detalhes sobre esta parametrização podem ser pesquisadas neste Manual de Referência do SADS, na Seção V, "Parametrizações do SADS", item "Consistência", tópico "Datas preenchidas com noves (999999)".

SINTAXE - Sintaxe do domínio.

Determina qual a natureza da informação contida nos atributos e campos relacionados ao domínio.

Pode assumir os seguintes valores:

"VL" - Variável livre ou variável cujo conteúdo não possui valores válidos pré-estabelecidos;

"VD" - Variável definida ou variável cujo conteúdo está definido por um conjunto de validades pré-estabelecidas.

Neste caso, as validades do domínio devem ser cadastradas para complementar a definição do mesmo.

TAMANHO - Quantidade de caracteres (alfabéticos, alfanuméricos ou numéricos) que o domínio possui.

Se o domínio possuir formato numérico, seu tamanho compreenderá a soma do número de casas inteiras mais o número de casas decimais.

DECIMAIS - Quantidade de casas decimais do domínio.

Este campo poderá ser informado somente para domínios com formato numérico. O domínio que possuir casas decimais deve ter pelo menos uma casa inteira.

SINAL - Indicador de existência de sinal no domínio.

Pode assumir um dos valores:

"S" - Se o domínio possuir sinal (+ ou -);

"N" ou " " - Se o domínio não possuir sinal.

Um domínio poderá ser sinalizado somente se o seu formato for numérico.

TAB/SUB - Estrutura ou processo contra o qual os atributos e campos, relacionados ao domínio, serão validados.

Tanto a tabela quanto a sub-rotina informada para a validação dos atributos e campos relacionados ao domínio devem estar cadastradas no repositório de dados e ser informadas no padrão SADS (sssMnnnn e sssSxx, respectivamente).

Se, para um domínio, for informada uma tabela, significa que sempre quando um valor for atribuído a um campo ou atributo relacionado a ele a consistência geral irá testar se este valor é chave de algum item da tabela.

Da mesma forma, se uma sub-rotina for informada, a consistência irá chamá-la para testar a validade do conteúdo dos atributos e campos relacionados ao domínio. Neste caso, a sub-rotina deve obedecer aos padrões definidos pela consistência geral, que encontram-se especificados no item "Consistência geral", nesta seção.

OPERACAO - Operação efetuada em relação ao domínio.

As operações possíveis são as seguintes:

"INC" (Inclusão);

"ALT" (Alteração);

"EXC" (Exclusão);

"CON" (Consulta).

Cadastramento de validades de domínio

Validades são valores ou intervalos de valores que o domínio poderá assumir.

Exemplo de validades de domínios:

Maior que 10 e menor que 50;

Igual a 1, 2 ou 3;

Diferente de "MARIA";

Diferente de "A", "B" e "C".

Para cadastrar as validades de um domínio, é necessário que o mesmo já esteja definido no repositório de dados do SADS e que seja uma variável definida (SINTAXE = "VD").

As validades definidas para um domínio serão observadas sempre que um valor for atribuído, em tabelas ou telas de entrada de dados, aos atributos e campos relacionados a ele, onde a consistência geral irá testar se este valor se enquadra ou não entre valores válidos definidos para o domínio em questão.

As validades dos domínios são utilizadas pelo help on-line quando este é solicitado após a verificação de um erro pela consistência geral, sendo este erro relativo a um atributo ou campo relacionado a um domínio variável definida (SINTAXE = "VD").

Estas validades são utilizadas também pela sub-rotina SDDS04 que, ao ser disparada por programas aplicativos, decodifica as validades dos atributos e campos relacionados ao domínio para exibição em telas, relatórios ou composição de arquivos. Detalhes a respeito da sub-rotina SDDS04 podem ser consultados no item "Sub-rotina", nesta seção.

CAMPOS DA TELA

SISTEMA - Sistema no qual o domínio será definido, no padrão SADS (sss).

Todo domínio tem de estar, obrigatoriamente, relacionado a um sistema.

DOMINIO - Mnemônico do domínio possuidor das validades.

Para cadastramento das validades do domínio são apresentados os seguintes campos:

SEQ - Número para posicionamento das validades de domínio em que os testes serão executados.

O primeiro número é "0010" e os demais, acrescidos de 10.

Para incluir uma validade entre duas outras já existentes, é necessário preencher uma linha que esteja em branco e alterar o número sequencial desta linha para um número entre os já informados. O SADS reorganizará automaticamente a estrutura. Na próxima operação as validades já estarão na nova ordem.

E/O - Operador lógico utilizado para concatenar as expressões que definem as validades do domínio.

Os operadores lógicos possíveis são os seguintes:

"E" - Operador lógico E (and);

"O" - Operador lógico OU (or).

OPER - Operador relacional utilizado nas expressões que definem as validades do domínio.

Os operadores relacionais possíveis são os seguintes:

"EQ" (equal) - Igual a;

"NE" (not equal) - Diferente de;

"GE" (greater or equal) - Maior que ou igual a;

"GT" (greater than) - Maior que;

"LE" (less or equal) - Menor que ou igual a;

"LT" (less than) - Menor que.

OPERANDO - Valores que definem as validades possíveis para o domínio.

O operando deve ser informado alinhado de acordo com o formato do domínio:

Se o domínio for numérico, o operando também deve ser numérico e seu conteúdo
tem de ser ajustado à direita. Não é necessário preenchê-lo com zeros à esquerda.

Se o domínio for alfanumérico ou alfabético, o operando também deve ser
alfanumérico ou alfabético e seu conteúdo tem de ser ajustado à esquerda com brancos
à direita.

A seguir é apresentado um exemplo de validades concatenadas:

DOMÍNIO SEXO

Validades : EQ (igual) F

O (ou) EQ (igual) M

DESCRICAO - Descrição ou explicação resumida do operando.

Utilizado para descrever o valor informado no operando.

Retomando o exemplo apresentado anteriormente, poder-se-ia cadastrar as seguintes descrições:

DOMÍNIO SEXO

Validades : EQ (igual) F - Descrição : "Feminino"

O(ou)  EQ (igual) M - Descrição : "Masculino"

A descrição não é obrigatória, porém, quando possível, ela deve ser informada, pois será apresentada quando um help de um atributo ou campo relacionado ao domínio for solicitado.

CONTINUA (S/N) - Indicador de continuidade de operação, com as seguintes situações:

"S" - Nova tela será aberta para prosseguir a operação relacionada às validades do domínio;

"N" - Última tela para operação sobre as validades do domínio.

OPERACAO - Operação efetuada em relação às validades do domínio.

As operações possíveis são as seguintes:

"INC" (Inclusão);

"ALT" (Alteração);

"EXC" (Exclusão);

"CON" (Consulta).

Isto significa que os únicos valores disponíveis para os atributos e campos relacionados ao domínio MES são aqueles nele definidos. Por isto, sempre que um destes atributos ou campos aparecer em telas de entrada de dados e tabelas, a consistência geral irá validar se o valor informado para o mesmo é um destes pré-estabelecidos. Caso contrário, será acusado um erro, como o exemplo que se segue:

PRJMES (Mes do projeto) INVALIDO;

onde o atributo PRJMES possui o domínio MES.

Funções de apoio

Específicas de atributo

Duplicação de domínio

O SADS permite que seja definido um novo domínio com as mesmas características (formato, tamanho, decimais, indicador de sinal, padrão, sintaxe, tabela/sub-rotina, fórmula, validades e descrição) de um domínio já definido.

Para duplicar domínio, é necessário que o domínio que terá suas características duplicadas esteja cadastrado no repositório de dados do SADS e que o novo domínio não esteja cadastrado.

A duplicação de código de domínio não está associada a nenhuma tela de processamento. Para efetuá-la, basta transmitir o menu de operação "DUPDOM", cujos campos estão explicados a seguir:

CAMPOS DA TELA

 Original:

Sistema - Sistema do domínio a ser duplicado;

Dominio - Código do domínio a ser duplicado.

 Copia:

Sistema - Sistema para o qual será criado o novo domínio;

Domínio - Código do domínio que será criado.

Troca código de domínio

A troca do código de um domínio compreende, além da troca propriamente dita, a sua substituição em todos os relacionamentos com atributos/campos.

Para trocar o código de um domínio, é necessário que o domínio atual esteja cadastrado e que o novo domínio não esteja cadastrado.

A troca de código de domínio não está associada a nenhuma tela de processamento. Para efetuá-la, basta transmitir o menu de operação "TRODOM", cujos campos estão explicados a seguir:

CAMPOS DA TELA

 Atual:

Sistema - Sistema do domínio a ser trocado.

Dominio - Código do domínio a ser trocado.

 Novo:

Sistema - Sistema para o qual o domínio será trocado.

Domínio - Código para o qual o domínio será trocado.
