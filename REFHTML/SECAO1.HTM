Introdução

O SADS é um conjunto de ferramentas que atua em diversas fases da Engenharia da Informação e através dele pode-se desenvolver sistemas a partir de metodologias consagradas.

Sua arquitetura é composta por módulos integrados através de um repositório de dados que garante a padronização e a integridade das especificações do projeto. A interface com o usuário é orientada por menus e rotas de navegação estruturadas segundo a sequência dos passos para construção de um sistema.

Sua atuação tem início com o módulo de Planejamento e Controle de Projetos, que oferece os recursos necessários à administração da mão-de-obra alocada ao desenvolvimento e manutenção dos sistemas.

O modelo conceitual é elaborado através do módulo de Especificação do Modelo Funcional e de Dados, que é executado em estações de trabalho e possui um repositório de dados próprio, compatível e integrado ao do mainframe.

A prototipação evolutiva é obtida através do processamento interpretativo, que mantém independência entre os modelos lógico e físico, cujo processo de simulação é feito a partir das especificações lógicas e não necessita da criação de programas e de bancos de dados.

O SADS possui diversos módulos de geração automática, que trabalham a partir das especificações conceituais, lógicas e físicas cadastradas no seu repositório de dados.

Os aplicativos gerados possuem uma arquitetura aberta, apresentam alta performance de execução e baixo consumo de recursos de hardware e sua manutenção é facilitada pelo processamento interpretativo e pela padronização nos cadastramentos e gerações, o que exige um mínimo de reprogramações e recompilações.

A elaboração e atualização da documentação dos sistemas aplicativos se torna consequência do processo de seu desenvolvimento no SADS, não exigindo criação ou manutenção posterior.

Diagrama esquemático do SADS

Requisitos de hardware e software

Para utilização do SADS Versão 4.2, as seguintes configurações de hardware e software devem ser satisfeitas:

  • CPU:

    UNISYS - Série A.

  • Terminais:

    Vídeo - Compatíveis com a linha UNISYS TD;

    Impressora remota.

  • Sistema Operacional:

    MCP - Versão 42.3 até 48.1;

    DMS - Versão 44.2 ou superior;

    COMS TOTAL - Versão 44.2 ou superior.

  • Memória:

    Save = 140 kwords (840 Kbytes);

    Total = 530 Kwords (3180 Kbytes);

  • Áreas em disco:

    Software = 260.000 segmentos;

    Banco = 200.000 segmentos.

  • Uma (1) janela direta no COMS.

  • Se utilizar BNA, deve-se configurar MAXOUTPUTMESSAGESIZE = 2560 e MAXINPUTMESSAGESIZE = 1940.

    Padronização

    A padronização utilizada pelo SADS possibilita a organização do ambiente operacional, tornando-o mais seguro e de fácil gerenciamento e manutenção. Além disso, garante a uniformização dos procedimentos, durante as etapas do desenvolvimento de sistemas.

    O SADS padroniza:

  • Códigos de elementos do sistema;

  • Arquivos físicos derivados dos elementos;

  • Lógicas de programas;

  • Interface com usuário (navegação);

  • Estrutura física de sistema (conceito de host).
    •

    Codificação dos elementos do sistema

    O padrão de codificação do SADS para elementos é:

    "sss e xxxxxx" onde,

    "sss" - Código alfabético que identifica o sistema;

    "e" - Código alfabético que identifica o tipo do elemento;

    "xxxxxx" - Código alfanumérico que varia de acordo com o tipo do elemento.

    Navegação

    O SADS dispõe de um método amigável de navegação, orientado por menus, que facilita ainda mais a utilização do sistema.

    Este módulo segue o padrão SAA - Systems Application Architecture da IBM*, que, através de seu módulo CUA - Common User Access, define regras para interface do usuário com o software.

    As telas de navegação foram desenvolvidas de modo a proporcionar a visualização dos elementos agrupados, em sequência lógica de utilização, nas diversas fases dos projetos conceitual, lógico e físico.

    A navegação é realizada através de menus de escolha, menus de operação e telas de processamento, com opções de Ajuda on-line.

    Existem ainda os menus de opções, através dos quais os aplicativos são desenvolvidos, mas que não fazem parte do módulo de navegação do SADS.

    * IBM é marca registrada

    Menu de escolha

    Menu de escolha é aquele que define rotas de navegação, sendo que cada rota corresponde a um destino que pode ser outro menu de escolha ou um menu de operação. Portanto, um menu de escolha é utilizado apenas para navegação entre menus, não sendo possível solicitar operações específicas a partir dele.

  • Barra de Ação - A Barra de Ação é a primeira área da tela e é composta por:

    Título do Menu: formado por um código alfabético, composto de três, seis ou nove caracteres;

    Descrição do título;

    Data e hora em que o menu foi recebido no terminal;

    Campo ACAO, onde se pode executar um dos seguintes comandos:

  • "Inicio <nome do contexto> <título do menu>" - Comando para apresentar o menu inicial do contexto especificado, ou o menu especificado do referido contexto;

    A ação "Início" sem especificação do nome do contexto e título do menu, apresenta o menu inicial do contexto corrente.

    Caso um nome de contexto tenha sido especificado, este passará a ser o contexto corrente.

    A informação do título do menu exige que o nome do contexto tenha sido especificado.

  • "Anterior" - Comando para exibir o último menu apresentado, com os campos preenchidos

    Este comando não irá exibir as telas repetidas, caso uma mesma tela seja utilizada mais de uma vez. Apenas uma única amostragem das últimas telas processadas será exibida.

    Quando for informada a ação "anterior", e a tela anterior for uma tela de processamento, será apresentado o menu de operação que a tiver originado, pois telas de processamento não são gravadas no arquivo de controle da navegação.

  • "Origem" - Comando para retornar ao menu de nível imediatamente superior na hierarquia de menus;

    Quando for informada a ação "origem" em uma tela de processamento, será apresentado o menu de operação que tiver originado esta transação.

  • "Desvio <título do menu> <nome do contexto>" - Comando para desviar para o menu correspondente ao título informado do contexto corrente ou do especificado;

    Caso um nome de contexto tenha sido especificado, este passará a ser o contexto corrente.

    A informação do nome do contexto exige que o título do menu tenha sido especificado.

    As telas de processamento não possuem título, e por isto não é possível executar um desvio para elas.

  • "Restaura" ou "Refresh" - Comando para restaurar o menu corrente com todos os campos preenchidos, caso a transação tenha sido executada.

    Nesta ação, serão restaurados até o limite de 700 bytes dos campos preenchidos.

    Quando se executa a ação "restaura" ou "refresh" em uma tela de processamento com um campo de ação, será apresentado o menu de operação que a tiver originado.

  • "Fim" - Comando para encerrar a sessão do SADS, efetuando um logoff na estação;

  • "Comando <contexto> <título do menu> <argumentos da transação>" - Comando que corresponde à transmissão da tela menu com o título informado, definida no contexto determinado, com os argumentos especificados.

    O código do contexto pode não ser informado e, neste caso, o título do menu deve estar definido no contexto correntemente logado.

    Os argumentos da transação podem não ser informados, possibilitando o disparo de um menu de operação sem argumentos.

    Este título de tela deve ser correspondente a um menu de operação, pois somente de menus de operação pode-se solicitar uma transação qualquer.

    Exemplo: "C PEDIDOS:CADCLI ALT001"

  • "Contexto <nome do contexto> <título do menu>" - Comando para se efetuar desviar para o contexto especificado ou para o menu especificado do referido contexto ;

    A informação do título do menu exige que o nome do contexto tenha sido especificado.

    Caso não se saiba o nome do contexto que se deseja utilizar, pode-se solicitar a ação "D CONTEXTO" a fim de que seja apresentada a tela de seleção de contextos disponíveis (Tela STKT0004).

  • "Usuario" - Comando para fornecer as seguintes informações sobre o usuário e a sessão:

    Usuário: número, nome e código do usuário, sigla e área da empresa, padrões de impressora e de usercode;

    Sessão: número e nome da estação, data e horário do início da sessão e ambiente do logon.

    Para executar uma ação, deve-se informar a palavra ou apenas a primeira letra que a identifique. Porém, ao utilizar a ação CONTEXTO, deve-se informar pelo menos "CON" para diferenciá-la da ação COMANDO.

    Exemplo:

    DESVIO "DEFENT"

    DES "DEFENT"

    D "DEFENT"

    As ações ORIGEM,INICIO, DESVIO, RESTAURA (REFRESH),COMANDO, CONTEXTO e USUARIO podem também ser executadas de uma tela desformatada ou vazia.

    Quando a ação ORIGEM for informada a partir de uma tela desformatada, será apresentada a tela origem da última tela de navegacção enviada ao terminal.

    Também é possível efetuar outras operações no campo ACAO, além daquelas disponíveis na navegação do SADS, tais como: "?ON CANDE", "?WRU", outros trancodes etc.

    Deve-se cuidar para que o campo ACAO esteja vazio antes de transmitir qualquer informação do campo ESCOLHA.

    •

  • Corpo do Menu- O Corpo do Menu é a segunda área da tela, onde está definido o conjunto de rotas de navegação válidas para o menu e o campo ESCOLHA.

    Cada rota de navegação corresponde a um destino que pode ser outro menu de escolha ou um menu de operação, segundo uma ordem lógica de utilização, de forma a facilitar o seu uso.

    O campo ESCOLHA permite a navegação por esses menus e o seu preenchimento é feito utilizando-se as letras maiúsculas da palavra que representa a rota desejada.

    Exemplo: Para selecionar o menu de Utilitários, a partir da tela inicial do SADS, informe "UTI" no campo ESCOLHA.

    Também é possível fazer escolhas múltiplas nos menus de escolha. Para isto deve-se digitá-las separadas por um caracter especial no campo ESCOLHA. Por exemplo: no menu inicial do SADS, pode-se digitar "TEL DES" no campo escolha e será apresentado o menu "DESTEL".

  • Área de Mensagem - O último espaço da tela é a Área de Mensagem, que corresponde às duas linhas finais do vídeo. Deve-se observar sempre a base da tela, porque este é o local em que são apresentadas todas as mensagens que orientam o trabalho do analista.
    •

    Menu de operação

    Menu de operação é aquele que define as operações e os dados básicos a serem informados para solicitação de processamento das transações.

    A cada menu de operação pode estar associada nenhuma, uma ou mais de uma tela de processamento.

    Os menus de operação são divididos em três áreas: Barra de Ação, Corpo do Menu e Área de Mensagem.

    A diferença entre os menus de operação e os de escolha está no Corpo do Menu. Enquanto os menus de escolha exibem rotas de navegação, os de operação apresentam os campos específicos associados às transações a serem processadas. Menus de operação, portanto, são as únicas telas a partir das quais pode-se solicitar operações.

    Os campos opcionais são apresentados após os obrigatórios e contêm a explicação necessária à sua utilização.

    O acesso a uma tela de processamento é feito a partir do preenchimento dos campos do menu de operação a ela correspondente e da transmissão deste menu.

    Telas de processamento

    Telas de processamento são telas utilizadas pelos programas que processam as transações, permitindo a execução das operações de inclusão, alteração, exclusão, consulta, geração, liberação e outras relativas aos elementos constantes no repositório do SADS.

    Existem transações no SADS que são executadas a partir de um menu de operação sem telas de processamento associadas.

    Em uma tela de processamento, pode-se tomar duas decisões:

    1. Retornar ao menu de escolha origem do menu de operação que tiver originado a
      transação, sem executar o seu processamento.

      Para isso, a tela deve ser transmitida com o cursor no início do seu primeiro campo, após o código de transação.

      Se o terminal utilizado estiver programado para transmitir a tela inteira, deve-se utilizar a combinação de teclas "ETX" e "XMIT", para transmitir somente o trancode.

    2. Processar a operação solicitada.

      As telas de processamento são de quatro tipos:

      Atualização Simples - São telas com características básicas dos elementos, como definições ou descrições.

      Atualização Paginada - São telas que permitem atualização de itens em uma ou mais páginas, com facilidade de deslocamento entre elas, sem obedecer a uma sequência lógica.

      Os campos específicos para atualização paginada são:

      PAGINA - Informa o número da página corrente;

      PROXIMA PAGINA - Número da página desejada, quando for necessário ver uma tela fora de sequência. Este campo pode ainda assumir os seguintes valores:

      "- " ou "--" - Para voltar à página anterior;

      "UL" - Para chegar à última página;

      "99" - Para encerrar a paginação e efetivar o processamento da transação.

      Se este campo não for informado, será apresentada a próxima página na sequência que foi programada.

      O processamento dos dados preenchidos nas páginas somente será efetuado quando for transmitido "99" no campo PROXIMA PAGINA.

      Consulta Simples - São telas com características básicas dos elementos, como definições ou descrições.

      Consulta Paginada - Apresenta os itens em uma ou mais páginas, com a facilidade de deslocamento entre elas, sem obedecer a uma sequência lógica.

      Os campos específicos da consulta paginada são:

      PAGINA - Informa o número da página corrente.

      PROXIMA PAGINA - Este campo vem preenchido com o número da próxima página que será apresentada. Para consultar outra página, deve-se informar apenas o número da página desejada. Este campo pode ainda assumir os seguintes valores:

      "- " ou "--" - Para voltar à página anterior;

      "UL" - Para chegar à última página;

      "99" - Para encerrar a paginação.

      Se este campo não for informado, será apresentado a próxima página na sequência que foi programada.

      No rodapé da página será exibida a palavra "CONTINUA" ou "FIM", significando, respectivamente:

      "CONTINUA" - Existem mais páginas para serem exibidas;

      "FIM" - Não existem mais páginas para serem exibidas.

    Ajuda on-line

    Ajuda on-line é uma função que o SADS oferece, em tempo de execução, para esclarecimento de dúvidas ou erros. A Ajuda on-line está presente em todas as fases do desenvolvimento de sistemas, com a finalidade de agilizar o uso da ferramenta.

    Esta operação deve ser executada sempre que, na ocorrência de erro, forem necessárias explicações mais completas que as fornecidas pelas mensagens de erro, ou quando se tiver alguma dúvida quanto a um campo específico ou a uma determinada tela.

    A ajuda on-line é obtida de duas maneiras: pressionando a tecla "SPFY" ou disparando a transação "HELP". A tecla "SPFY" só pode ser utilizada para se obter explicação de um menu de navegação. Para se obter explicações mensagens de erro em tela de processamento, deve-se mudar de página no terminal e utilizar a transação "HELP".

    Para consultar um campo específico de uma tela, deve-se informar um caractere previamente definido na primeira posição deste campo e brancos nas demais posições. Este caractere deve ter sido especificado na tabela SBPM5004 para o próprio sistema ou para todos os sistemas da instalação. Detalhes a este respeito estão descritos na seção V, "Parametrizações do SADS", deste manual.

    Para a efetivação da ajuda a um campo específico, os hosts devem ter sido compilados e as consistências liberadas após a implantação da versão 3.2.661 do SADS.

    A transação "HELP" pode também fornecer explicação de uma tela sem que esta tenha sido chamada. Para isso, deve-se utilizar a transação seguida do código da tela, da seguinte maneira: "HELP sssTnnmm".

    As informações apresentadas pela Ajuda on-line diferem de acordo com o tipo de erro ocorrido e por quem foi detectado. Sabendo-se que, no SADS, os erros são detectados pelas consistências, temos que estas se decompõem em dois tipos: consistências gerais e consistências próprias das transações.

    Por sua vez, a consistência própria da transação trata erros específicos da transação que a consistência geral não consegue detectar. Estes erros específicos são detectados através de testes internos ao programa relacionado à transação.

    Com base na distinção entre consistência geral e consistência própria da transação, pode-se descrever de forma mais clara quais são as informações apresentadas pela Ajuda on-line:

  • Quando ocorre um erro detectado pela consistência geral, a Ajuda on-line exibe:

  • Dados do atributo ou campo com erro no preenchimento, definidos no momento de seu cadastramento: sistema, código, nome, código externo, domínio (se possuir, exibe também suas características), formato, tamanho, decimais, sinal, classificação, ocorrência, teste padrão, default, tabela/sub-rotina para validação do atributo, sintaxe, cabeçalho, máscara de edição, sinônimo (se possuir, exibe as características do atributo sinônimo), validades (caso a sintaxe seja variável definida), fórmula (caso a sintaxe seja variável gerada) e descrição;

  • Validades do atributo ou campo com erro no preenchimento, definidas nas validades da consistência.
    •

  • Sendo o erro detectado pela consistência própria da transação, a Ajuda on-line oferece uma descrição da mensagem de erro exibida pelo programa que tratou a transação.

  • Acionando a Ajuda on-line após receber um menu, obtém-se uma explicação detalhada do mesmo. Caso este seja um menu de operação, além de sua explicação, obtém-se também a explicação de sua tela de processamento.

    Procedimentos de habilitação

    Logon

    É imprescindível que o usuário, ao trabalhar com o SADS ou com aplicativos por ele gerados que utilizam a sua segurança, identifique-se ao sistema através da operação de logon.

    Para que um usuário possa efetuar logon ou a troca de sua senha, é necessário que o seu cadastramento tenha sido liberado pelo menos uma vez.

    Pode-se definir a string "SBT02.2501" no Notify Open text da Window, para que, assim que a mesma for aberta, seja encadeada a tela para o logon do usuário.

    Ao iniciar uma sessão de trabalho, requisitando um aplicativo que exija segurança, o usuário receberá antes dele, o menu de habilitação para uso, cujos campos devem ser preenchidos conforme indicado:

    Numero do Usuario - Número cadastrado pelo administrador de segurança e pelo qual o usuário pode ser reconhecido pelo sistema.

    Senha - Senha do usuário.

    A senha pode ter até seis caracteres alfanuméricos com pelo menos uma letra e um número, não podendo possuir caracteres especiais.

    Além disto, pode ter sido parametrizado no campo PAR-6, CODIGO = "SISSEG" da tabela SBPM5000, qual o tamanho mínimo de uma senha a ser definida. Se o valor deste campo não tiver sido informado, for "0" (zero) ou for inválido, não haverá a validação do número de caracteres mínimos da senha.

    As senhas são sempre criptografadas antes de sua gravação para evitar violações.

    Ambiente - Ambiente no qual se deseja trabalhar, caso existam habilitações cadastradas para ambos os ambientes - Teste (T) e Produção (P) e o usuário não possuir ambiente default.

    Este campo deve ser informado apenas se o usuário possuir habilitação para mais de um ambiente e não possuir um ambiente default.

    Empresa - Código da empresa para a qual o usuário está habilitado.

    Este campo deve ser informado apenas se o usuário possuir habilitação para mais de uma empresa e não possuir uma empresa default.

    Estabelecimento - Código do estabelecimento para o qual o usuário está habilitado.

    Este campo deve ser informado apenas se o usuário possuir habilitação para mais de um estabelecimento e não possuir estabelecimento default.

    Unidade - Código da unidade para a qual o usuário está habilitado.

    Este campo deve ser informado apenas se o usuário possuirr habilitação para mais de uma unidade e não possuir unidade default.

    Código do Usuario - Código cadastrado pelo administrador de segurança e pelo qual o usuário pode ser reconhecido pelo sistema.

    Para efetuar o logon, ou o número do usuário ou o código do usuário devem ser informados. Se ambos forem informados, prevalecerá o número.

    O preenchimento dos campos Ambiente, Empresa, Estabelecimento e Unidade é opcional, dependendo do tipo de habilitação especificado no momento do cadastramento e dependendo da existência de defaults cadastrados. Maiores detalhes sobre o preenchimento destes campos são elucidados a seguir através da exibição dos testes efetuados sobre os mesmos no momento do logon. Entretanto, a explicação de como cadastrar estes campos para cada usuário está descrita na Seção VI, "Sistema de Segurança", no item "Usuários e habilitações".

    O Sistema de Segurança do SADS permite que seja definido, no momento do cadastramento do usuário, se seu logon será integrado com o Marc. Caso positivo, fica estabelecido que não lhe será exigido o preenchimento da tela de habilitação anteriormente explicada, pois o logon será disparado automaticamente pelo próprio Sistema de Segurança assumindo os seguintes valores para os campos:

  • O código do usuário será igual ao usercode por ele informado no Marc ;

  • Através deste código é então identificado o número de usuário;

  • A senha não será exigida;

  • O ambiente e as unidades autorizadas são informações obtidas do cadastro do usuário, caso sejam específicas. Caso contrário, estas informações são obtidas dos defaults cadastrados para o usuário, que, no caso de integração com o Marc, têm obrigatoriedade de cadastramento.
    •

    Se o usuário possui logon integrado ao Marce deseja efetuar logon em um ambiente e unidades autorizadas diferentes daqueles assumidos, então ele deve utilizar o menu de operação "HAB" para disparar novo logon com os valores desejáveis. Neste caso, é necessário que o usuário informe a senha.

    A senha pode ter até seis caracteres alfanuméricos com pelo menos uma letra e um número, não podendo possuir caracteres especiais.

    Além disto, pode ter sido parametrizado no campo PAR-6, CODIGO = "SISSEG" da tabela SBPM5000, qual o tamanho mínimo de uma senha a ser definida. Se o valor deste campo não tiver sido informado, for "0" (zero) ou for inválido, não haver´ a validação do número de caracteres mínimos da senha.

    As senhas são sempre criptografadas antes de sua gravação para evitar violações.

    Uma vez executada com sucesso a habilitação para início da sessão SADS, será encadeado o aplicativo anteriormente requisitado. Se o usuário não houver requisitado anteriormente um aplicativo, será encadeada a resposta cadastrada para o usuário imediatamente após logon. Caso não exista resposta cadastrada, será encadeada a resposta definida na tabela SBPM6004. Maiores detalhes a respeito desta tabela podem ser obtidos através de consulta à seção V, "Parametrizações do SADS".

    Na primeira tentativa de logon, se não houver integração com o Marc, a troca da senha será exigida, a fim de que o cadastramento seja feito de acordo com a conveniência do usuário. Somente neste caso, deve-se informar, no momento da troca, a nova senha, uma vez que a antiga ainda não existe.

    Após o logon, a informação desses campos fica disponível para ser utilizada pelos programas. Maiores detalhes podem ser consultados na Seção VI, "Sistema de Segurança", item "Interface com aplicativos", tópico "Área global de comunicação".

    No momento do logon, o Sistema de Segurança efetua uma sequência de testes com o objetivo de identificar e validar as habilitações. São eles:

  • Teste de validade da senha: se a data de validade de senha estiver vencida, o usuário será automaticamente bloqueado e, consequentemente, impedido de efetuar logon.

  • Teste de usuário bloqueado ou em cadastramento: em ambos os casos, haverá impedimento para a execução do logon.

  • Teste de ambiente: se o usuário estiver cadastrado com habilitação para ambos os ambientes (Ambiente "A") e não possuir ambiente default cadastrado, deverá informar em qual dos ambientes pretende trabalhar: Teste (Ambiente "T") ou Produção (Ambiente "P"). Caso contrário, o preenchimento é opcional.

    O acesso é restrito para os sistemas compilados para o mesmo ambiente e para o qual o logon foi executado. Com o SADS, pode-se trabalhar tendo executado o logon em qualquer ambiente, com exceção para o sistema de tabelas que trata os itens do ambiente para o qual o logon foi efetuado.

  • Teste de limitação de horário de trabalho de usuário: caso haja restrições de horário de trabalho para o usuário, este não conseguirá executar logon, nem trabalhar, durante os mesmos.

  • Teste de empresa: a empresa informada deve ser compatível com a habilitação definida no cadatramento. Deste modo:

    Usuário habilitado "999" (todas as empresas): exige que o usuário informe em qual das empresas deseja trabalhar, caso ele não possua empresa default cadastrada;

    Usuário habilitado "nnn" (somente uma empresa específica): assume o logon na empresa "nnn", caso esta seja informada ou não, e não permite outra;

    Usuário habilitado "000": não permite a informação de empresa no momento do logon.

  • Teste de estabelecimento: o estabelecimento informado deve ser compatível com a habilitação definida no cadastramento. Deste modo:

    Usuário habilitado "9999" (todos os estabelecimentos): exige que o usuário informe em qual dos estabelecimentos deseja trabalhar, caso ele não possua estabelecimento default cadastrado;

    Usuário habilitado "nnnn" (somente um estabelecimento específico): assume o logon no estabelecimento "nnnn", caso este seja informado ou não, e não permite outro;

    Usuário habilitado "0000": não permite a informação de estabelecimento no momento do logon.

  • Teste de unidade: a unidade informada deve ser compatível com a habilitação definida no cadastramento. Deste modo:

    Usuário habilitado "999" (todas as unidades): exige que o usuário informe em qual das unidades deseja trabalhar, caso ele não possua unidade default cadastrada;

    Usuário habilitado "nnn" (somente uma unidade específica): assume o logon na unidade "nnn", caso esta seja informada ou não, e não permite outra;

    Usuário habilitado "000": não permite a informação de unidade no momento do logon.

  • Teste de estação: caso haja cadastramento de estações habilitadas, será permitido o uso de apenas uma delas.

    Caso não haja integração do logon com o Marc, além dos anteriores, os seguintes testes serão efetuados:

  • Teste de preenchimento da senha: a senha deve ser sempre informada, exceto quando ela ainda não existir. Neste caso, será encadeado o programa de troca de senha para que ela seja criada.

  • Teste de compatibilidade da senha: a senha informada deve ser correspondente ao número de usuário informado.

  • Teste de troca de senha: é exigida a troca da senha após aproximadamente 60 dias da última troca ou do cadastramento, de acordo com o tipo de cadastramento.
    •

    Na instalação, pode-se definir qual o tipo de bloqueio a ser efetuado quando for detectada mais de cinco tentativas inválidas de logon em uma estação: bloqueio do usuário, bloqueio do usuário e da estação ou bloqueio da estação.

    Caso esta parametrização não seja cadastrada, será assumido que haverá bloqueio da estação no momento em que houver tentativa de burlar a segurança - mais de cinco tentativas inválidas de logon na estação.

    Para se obter maiores detalhes sobre a especificação na tabela SBPM5000 do tipo de bloqueio a ser efetuado, deve-se consultar, neste manual, a Seção V, "Parametrizações do SADS", item "Segurança".

    Também através de parametrização efetuada na tabela SBPM5000, pode-se definir se será permitido ou não que um mesmo usuário efetue logon simultaneamente em mais de uma estação.

    Os detalhes sobre esta parametrização também são encontrados na Seção V, "Parametrizações do SADS", item "Segurança", deste manual.

    Reiniciando uma sessão

    Se for necessário reingressar no sistema com uma senha diferente da informada no início da sessão de trabalho, deve-se executar o desvio para o menu "HAB", onde os campos devem ser preenchidos conforme indicado:

    Operacao - "LOG", para efetuar logon.

    Numero do Usuario - Número de usuário com o qual o novo logon será efetuado.

    Senha - Senha com a qual o usuário deseja fazer o novo logon.

    Ambiente - Ambiente no qual deseja trabalhar, caso esteja habilitado a trabalhar em ambos os ambientes e não possua ambiente default cadastrado- Teste (T) e Produção (P).

    Empresa - Código da empresa para a qual o usuário está habilitado a trabalhar.

    Este campo deve ser informado caso exista habilitação para mais de uma empresa e não possua empresa default.

    Estabelecimento - Código do estabelecimento para o qual o usuário está habilitado a trabalhar.

    Este campo deve ser informado caso exista habilitação para mais de um estabelecimento e não possua estabelecimento default.

    Unidade - Código da unidade para a qual o usuário está habilitado a trabalhar.

    Este campo deve ser informado caso exista habilitação para mais de uma unidade e não possua unidade default.

    Codigo do Usuario - Código de usuário com o qual o novo logon será efetuado.

    Ou o código ou o número do usuário deve ser informado.

    Porém, se for necessário reingressar no sistema informando unidades (empresa, estabelecimento e unidade) diferentes daquelas especificadas no início da sessão de trabalho, permanecendo logado com o mesmo número e senha do usuário, deve-se executar o desvio para o menu "HAB" e preencher os seguintes campos:

    Operacao - "EMP", para trocar as unidades (empresa, estabelecimento e/ou unidade) do logon.

    Após a transmissão deste menu serão apresentados os dados do usuário logado na estação.

    Deve-se, então, proceder a alteração do código da empresa, estabelecimento e unidade, conforme desejado.

    Troca de senha

    Além do procedimento de logon, o menu de habilitação para uso possibilita a troca de senha, que é exigida após, aproximadamente, 60 dias do cadastramento ou da última troca realizada, se o tipo de cadastramento efetuado exigir a troca.

    Se a nova senha informada já existir, sua troca não é permitida e o possuidor da senha é bloqueado para que sua senha seja trocada, a fim de evitar uma violação na segurança.

    A troca de senha pode ser executada a qualquer momento, caso o cadastramento do usuário assim permita. Não é, pois, necessário esperar para fazê-la somente quando o sistema de segurança a exigir.

    Para que um usuário possa efetuar a troca de sua senha ou logon, é necessário que o seu cadastramento tenha sido liberado pelo menos uma vez.

    Após efetuada a troca de sua senha, a sessão de trabalho está pronta para ser iniciada.

    Para efetuar a troca de senha, é necessário executar o desvio para o menu "HAB" e informar os seguintes campos:

    Operacao - "TRO".

    Numero do Usuario - Número do usuário (opcional).

    Senha - Senha do usuário.

    Ambiente - Código do ambiente: "T" ou "P" (opcional).

    Empresa - Código da empresa (opcional).

    Estabelecimento - Código do estabelecimento (opcional).

    Unidade - Código da unidade (opcional).

    Codigo do Usuario - Codigo do usuário (opcional).

    Neste momento serão executados os mesmos testes efetuados no momento do logon.

    Será apresentada uma tela de processamento com os seguintes campos:

    Numero Usuario - Este campo vem preenchido com o número do usuário.

    Codigo Usuario - Este campo vem preenchido com o número do usuário.

    Senha Atual - Senha a ser substituída.

    Nova Senha - Nova senha a ser cadastrada.

    A senha pode ter até seis caracteres alfanuméricos com pelo menos uma letra e um número, não podendo possuir caracteres especiais.

    Além disto, pode ter sido parametrizado no campo PAR-6, CODIGO = "SISSEG" da tabela SBPM5000, qual o tamanho mínimo de uma senha a ser definida. Se o valor deste campo não tiver sido informado, for "0" (zero) ou for inválido, não haverá a validação do número de caracteres mínimos da senha.

    As senhas são sempre criptografadas antes de sua gravação para evitar violações.

    Nome do usuário.

    Esta tela de troca de senha será apresentada novamente para confirmação das senhas.

    Após a troca da senha, o SADS executará implicitamente um logon com a nova senha para o ambiente, empresa, estabelecimento e unidade informados no menu de troca de senha.

    Logoff

    O logoff encerra uma sessão SADS e impede que outros usuários que porventura queiram trabalhar nesta mesma estação aproveitem a sessão de trabalho já inicializada e que, por isso mesmo, já possui identificação.

    Para efetuar o logoff, uma das operações a seguir deve ser executada:

  • Transmissão de qualquer menu de escolha ou de operação, com o campo ACAO preenchido com "FIM" ou "F", ou;

  • Transmissão de qualquer menu de opções dos aplicativos, com o campo OPCAO = "99".

    Paginadores

    O SADS possui módulos que facilitam a programação dos sistemas que utilizam múltiplas páginas em aplicações on-line. São eles:

  • Paginador de entrada de dados e

  • Paginador de saída de dados.
    •

    Paginador de entrada de dados

    O paginador de entrada de dados facilita a programação de aplicações on-line de entrada de dados que utilizem múltiplas telas.

    Elimina a necessidade de codificação e economiza recursos de processamento e memória. Provê o armazenamento de diversas páginas de entrada de dados e flexibilidade de acesso às mesmas.

    Suas principais características são:

  • Facilidades de navegação entre as páginas;

  • Consistência efetuada a cada página transmitida, com possibilidade do programa aplicativo consistir as páginas armazenadas, de uma só vez, contra cadastro;

  • Manutenção da última transação paginada, em nível de estação;

  • Paginação de telas de layouts diferentes, com independência de seus formatos, sendo aplicável a transações de inclusão e alteração;

  • Possibilidade de restrição do número máximo de páginas a serem tratadas pelo usuário;

  • Utilização por programas escritos em diferentes linguagens.
    •

    Os campos específicos para a atualização paginada são:

    PAGINA - Número da página corrente;

    PROXIMA PAGINA - Número da página desejada, quando houver necessidade de se ver uma tela fora de sequência.

    Pode assumir os seguintes valores:

    "-"ou "--" - Para voltar à página anterior;

    "UL" - Para chegar à última página;

    "99" - Para encerrar a paginação e efetivar o processamento.

    Se este campo não for informado, será apresentado a próxima página na sequência que foi programada.

    O processamento dos dados preenchidos nas páginas somente será efetuado quando for transmitido "99" no campo PROXIMA PAGINA.

    Paginador de saída de dados

    O paginador de saída facilita a programação dos sistemas que utilizam consultas de múltiplas páginas em aplicações on-line.

    Elimina a necessidade de codificação e economiza recursos de processamento e memória. Provê o armazenamento de diversas páginas de consultas e flexibilidade de acesso às mesmas.

    Suas principais características são:

  • Preparação de consultas através de programas on-line, com arquivamento feito por sub-rotina e possibilidade de utilização de telas de layouts diferentes;

  • Paginação das consultas, independente de seus formatos;

  • Utilização por programas escritos em diferentes linguagens;

  • Possibilidade de impressão remota de páginas selecionadas;

  • Facilidades de navegação entre as páginas;

  • Manutenção da última transação paginada, em nível de estação.
    •

    Os campos específicos da consulta paginada são:

    PAGINA - Número da página corrente;

    PROXIMA PAGINA - Número da próxima página que será apresentada. Quando houver necessidade de se ver uma tela fora de sequência, deve-se informar o número da página desejada.

    Pode assumir os seguintes valores:

    "-"ou "--" - Para voltar à página anterior;

    "UL" - Para chegar à última página;

    "SS" - Para retornar ao menu, encerrando a transação, sem salvar (SS) os dados alterados. Neste caso não há então nenhuma atualização das informações apresentadas;

    "99" - Para encerrar a paginação.

    Se este campo não for informado, será apresentada a próxima página na sequência que foi programada.

    No rodapé da página será exibida a palavra "CONTINUA" ou "FIM", significando, respectivamente:

    "CONTINUA" - Existem mais páginas para serem exibidas;

    "FIM" - Não existem mais páginas para serem exibidas.
