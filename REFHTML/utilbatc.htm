Utilitários batch

O SADS oferece grande número de rotinas batch encarregadas por extrações ou cargas no repositório de dados, emissões de relatórios e funções de apoio genéricas.

Algumas delas possuem parâmetros que devem ser informados anteriormente à sua execução.

Estas rotinas podem ser disparadas diretamente no Cande através de seus jobs específicos. Porém, o SADS tornou possível, através do SPO - Sistema de Controle de Processamento, que elas sejam disparadas on-line, utilizando as escolhas disponíveis no menu de operação "ROTNAV" - Rotinas Batch - e utilizando também todas as facilidades que a navegação do SADS oferece.

Neste item serão explicados, pois, como pode ser efetuada a solicitação on-line de cada uma das rotinas batch do SADS e como a mesma é composta, para que, caso necessário, seja possível dispará-la via Cande.

Rotinas de extração

As rotinas de extração são responsáveis por gerar arquivos contendo informações cadastradas no repositório de dados do SADS, que poderão ser utilizados posteriormente para carga em outro ambiente ou computador.

A seguir, cada uma das rotinas de extração será detalhada.

Extração de telas e consistências (liberadas para produção)

O SADS permite que seja extraído o cadastramento de todas as telas e consistências liberadas para produção no repositório de dados, a fim de que, posteriormente, possa ser atualizada a máquina de produção.

Solicitação on-line

Menu de operação : TRATEL

Operação - "EXT" (Extração).

Tela de parâmetros:

Sistema - Código do sistema, no padrão SADS (sss).

Caso o sistema não seja informado, serão extraídos todos os sistemas.

Tipo - Tipo do elemento, quando se deseja extrair um elemento específico. (Opcional)

Pode assumir os seguintes valores:

"C" - Para extração de consistências de telas;

"Q" - Para extração de consistências de jobs; ou

"T" - Para extração de telas.

Identificador - Identificador do elemento específico a ser extraído.

Este identificador deve ser informado, quando se desejar extrair apenas um elemento (tela ou consistência) específico.

Extração de tabelas (via solicitação de transferência)

A transferência das tabelas selecionadas somente será concretizada através da execução de rotinas batch pelos jobs SSTJ01 e SSTJ02, de extração e carga de tabelas, respectivamente.

Quando a rotina de extração de tabelas via solicitação de transferência for executada, as consistências associadas às tabelas também serão extraídas do ambiente origem da solicitação de transferência.

O processo de extração consiste na geração de arquivos a partir das tabelas selecionadas (para as quais as solicitações foram efetuadas) e dos parâmetros informados no momento das solicitações. As solicitações são automaticamente excluídas após a execução do processo batch de extração do arquivo.

Extração de contexto

O SADS oferece um job para extração de todas as regras, rotas e títulos de um determinado contexto.

Solicitação on-line

Menu de operação : TRACON

Operação - "EXT" (Extração).

Tela de parâmetros:

Contexto - Nome do contexto para extração.

Operacao - Operação a ser efetuada, que pode ser:

"A" - Altera contexto já existente, que possui os códigos internos diferentes nos ambientes origem e destino;

"C" - Cria um contexto com código interno aleatório;

"O" - Altera contexto já existente, que possui os códigos internos iguais nos ambientes origem e destino;

"I" - Cria contexto com código interno informado em "Codigo interno".

Codigo interno - Código interno do contexto a ser criado, quando a operação solicitada for "I" (Inclusão de contexto). (Opcional)

Este código deve ser alfanumérico ("xx").

Titulo - Indicador de extração de títulos, que pode ser:

"S" - Para extração dos títulos;

"N" - Para não extrair os títulos.

Rota - Indicador de extração de rota, que pode ser:

"S" - Para extração das rotas;

"N" - Para não extrair as rotas.

Regra - Indicador de extração de regras, que pode ser:

"S" - Para extração das regras;

"N" - Para não extrair as regras.

Tela - Indicador de extração de telas, que pode ser:

"S" - Para extração das telas;

"N" - Para não extrair as telas.

Ambiente - Ambiente de extração das regras e telas, que pode ser:

"T" - Para extrair do ambiente de teste;

"P" - Para extrair do ambiente de produção.

Um contexto de navegação possui um código interno que o identifica. Para saber o código interno de um contexto, deve-se utilizar a operação "LIS" do menu CONCONNAV (Consultas relativas a contextos de navegação). "A" - Quando for carregar um contexto já existente no ambiente de destino, porém os códigos internos do contexto forem diferentes;

"C" - Quando for carregar um contexto novo no ambiente de destino e o código interno do contexto for criado aleatoriamente pelo programa de carga;

"I" - Quando for carregar um contexto novo no ambiente de destino e se desejar que o código interno seja aquele informado como parâmetro. Este código não pode ser um já existente no destino.

O SADS permite que sejam carregadas no repositório de dados todas as informações a respeito de um ou vários programas, que foram extraídas de um outro repositório de dados.

A versão do SADS no ambiente onde estas informações serão carregadas deve ser idêntico à versão daquele ambiente de onde elas foram extraídas. Isto para evitar incompatibilidades entre o conteúdo das informações nos dois ambientes.

Operação - "CAR" (Carga).

Tela de parâmetros:

Referencia - Data e hora de referência dos dados dos programas a serem carregados.

Substituição de atributos/caracteres em cadastramentos de programas

O SADS possui um job - SD/D/JB/19 - encarregado por trocar nomes de atributos, campos ou strings dos comandos de módulo COBOL, EDIT ou EXCEC.

Os atributos, campos ou strings a serem trocados, devem ser informados no arquivo SD/D/AF/79, onde cada registro conterá uma dupla de atributos origem e destino. O código do atributo a ser trocado deve ser informado primeiramente, entre barras (/) e, logo em seguida, o código do atributo que irá substituí-lo, também entre barras (/) - semelhante ao comando Replace do Cande.

Durante o processamento, será criado um arquivo de LOG de nome SD/D/AF/80. Este arquivo conterá, nas primeiras linhas, o conteúdo do arquivo SD/D/AF/79, ou seja, as substituições a serem processadas. Se houver algum erro de interpretação em alguma das linhas do arquivo SD/D/AF/79, a linha correspondente a este registro no arquivo SD/D/AF/80 virá preenchida com a letra "E" (Erro) na primeira coluna.

Depois da interpretação do arquivo SD/D/AF/79, o arquivo de LOG SD/D/AF/80 irá conter as alterações efetuadas nos cadastramentos dos programas do sistema em questão no repositório do SADS. Cada linha do arquivo corresponderá a uma substituição efetuada no repositório.

Podem ser feitas mais de uma alteração para um mesmo comando COBOL/EDIT/EXCEC. Quando não for possível fazer a alteração devido a um truncamento no comando, a linha correspondente no arquivo de LOG será também identificada com a letra "E" (Erro) na primeira coluna.

Prioridades para definição das palavras-chaves no corpo do menu

No momento da geração de uma página HTML, deverão ser identificados, no desenho do menu de escolha, onde se encontram as palavras-chaves definidas para tal menu. relatorio de usuarios em DEBITO

Em relação à geração de hyperlinks a partir da localização das palavras-chaves, tem-se ainda as seguintes considerações:

Isto irá ocorrer quando, pela análise do desenho, uma palavra-chave ocorrer em mais de uma frase (rota).     • As palavras-chaves que não forem encontradas serão ignoradas.
    • Serão considerados apenas os caracteres maiúsculos para localização das palavras-chaves.
