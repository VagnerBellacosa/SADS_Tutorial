Dataset

Conceituação

Datasets são coleções de registros de dados que podem ser manipulados exclusivamente através do sistema gerenciador de banco de dados DMSII. Nos datasets, os dados ficam armazenados de forma similar aos arquivos convencionais, destes diferenciando-se apenas pela organização e formas de acesso. O acesso ao dataset é feito somente através de extensões específicas do Cobol e BDMSCOBOL.

Um dataset poderá ser redefinido logicamente. A essa redefinição lógica é dado o nome de remap.

O remap pode ser útil na restrição do acesso às informações para um determinado grupo de usuários ou na reformatação dos registros do dataset. Essa redefinição evita alterações na lógica de programas que já utilizam o dataset.

Maiores informações relativas a dataset e remap poderão ser obtidas no "DMSII - Data and Structure Definition Language (DASDL) - Programming Reference Manual" da Unisys.

Padronização

O padrão de codificação para identificar datasets no repositório de dados do SADS é o seguinte:

"sssBxx" onde:

"sss" é o código alfabético que identifica o sistema;

"B" é a constante de dataset e

"xx" é I identificador alfanumérico do dataset.

O identificador alfanumérico diferencia um dataset de um remap. O dataset tem como identificador caracteres que variam de "00" a "50" e de "A0" a "ZZ", já o remap é identificado por números que variam de "51" a "99".

Como o padrão de codificação de dataset é semelhante ao de banco de dados, no momento da definição de um dataset ou remap no repositório de dados, o código informado só será válido se não tiver sido anteriormente utilizado para identificar um banco de dados.

O fonte DASDL, cujo código possui o padrão ss/s/BL/xx, contém os atributos e campos que compõem o dataset e a definição das chaves de acesso.

A seguir, é apresentado um exemplo de fonte DASDL gerado pelo SADS.

Para o dataset "SXYB01 - Lojas", temos o DASDL :

   SXYB01 DATA SET "LOJAS"
                   (
                    Y0100-HEADER                               GROUP                            (
                                Y0100-HDRDATA                  NUMBER (006)         ;
                                Y0100-HDRHORA                  NUMBER (006)    ;
                                Y0100-HDRESTACAO               NUMBER (003)    ;   )
                    Y0100-LOJCOD                               NUMBER (003)    ;
                    Y0100-LOJCGC                               NUMBER (014)    ;
                    Y0100-LOJNOM                               ALPHA  (030)     ;
                    Y0100-LOJTIP                               ALPHA  (001)     ;
                    Y0100-LOJEND                               ALPHA  (060)     ;
         );
   SXYB01A SET OF SXYB01
                  KEY   (Y0100-LOJCOD           ,
                         Y0100-LOJCGC           );
   SXYB01B SUBSET OF SXYB01
                     WHERE        Y0100-LOJTIP = "R"
                     KEY          Y0100-LOJCOD
                     DATA         Y0100-LOJNOM    ;

Os mnemônicos dos campos de DASDL têm a seguinte formação:

Terceira letra do sistema ("Y", no exemplo) + o sequencial do dataset ("01", no exemplo) + o número do registro ("00", no exemplo) + "-" (constante) + o mnemônico do atributo ou campo.

Utilização

O dataset, conforme descrito em sua conceituação, é utilizado para armazenar registros de dados manipulados pelo sistema gerenciador de banco de dados DMSII.

Um dataset pode ser definido como sendo um Restart Dataset, a fim de ser utilizado nas rotinas de recuperação DMS.

Um dataset, após ter sido cadastrado no repositório de dados do SADS, pode ser utilizado na definição do banco de dados. A partir do momento em que ele estiver relacionado a um banco de dados, o dataset poderá ser referenciado em definições de base de dados e, depois disso, como recurso de programas on-line, batch e sub-rotinas.

Cadastramento

O cadastramento de um dataset só será possível se o sistema e os atributos que compõem a sua estrutura estiverem definidos no repositório de dados.

Se o dataset for ser criado a partir de uma entidade, é necessário que a entidade já esteja totalmente cadastrada no repositório de dados do SADS (definição, estrutura e chaves secundárias, se houver). Também é necessário que tenha todos os seus atributos cadastrados, ou seja, entidades que possuam atributos com nível de definição "P" (provisório) não podem ser utilizadas para geração de datasets.

Roteiro

Um dataset poderá ser criado a partir de uma entidade ou definido diretamente no repositório de dados.

A geração de um dataset é feita através da seguinte função :

Criação de dataset a partir de entidade.

Para alterações do cadastramento gerado ou para inclusão de um dataset, temos as seguintes funções:

Definição de dataset;

Cadastramento de estrutura de registro de dataset;

Cadastramento de chaves de acesso (set/subset/access);

Para complementações do cadastramento, tem-se as seguintes funções :

Cadastramento de condições para subsets automáticos;

Cadastramento de parâmetros físicos de dataset e chaves de acesso.

Após o cadastramento do dataset, as seguintes funções poderão ser executadas:

Geração de fonte DASDL de dataset;

Geração de DASDL de parâmetros físicos de dataset.

Criação de dataset a partir de entidade

A criação de um dataset, de sua estrutura e chaves de acesso a partir de uma entidade só será possível se esta estiver cadastrada corretamente no repositório de dados do SADS.

Existe a possibilidade de uma entidade gerar mais de um dataset e de um dataset ser gerado a partir de mais de uma entidade. Neste último caso, cada entidade origina um tipo de registro do dataset. Para identificar os datasets ou tabelas gerados a partir de uma entidade, deve-se utilizar a referência cruzada de elementos "REFELEELE".

A entidade também deverá ter todos os seus atributos cadastrados no repositório de dados, isto é, não deve possuir nenhum atributo provisório (NDEF = "P").

Os atributos, opcionalmente, podem ser criados na estrutura do dataset ordenados de acordo com seus formatos, onde todos os atributos alfabéticos e alfanuméricos são criados antes dos atributos numéricos. A utilização desta ordenação de acordo com o formato pode ser especificada na tabela SBPM5000, cujos detalhes estão descritos na Seção V, "Parametrizações do SADS".

Esta operação não está associada a nenhuma tela de processamento. Para efetuá-la, basta transmitir o menu de operação "CRIDADENT", cujos campos estão explicados a seguir:

CAMPOS DA TELA

Entidade - Código da entidade de acordo com a padronização adotada pelo SADS (sssEaaa).

Dataset - Código do dataset a ser criado a partir da entidade (sssBxx).

Registro - Número do registro do dataset que está sendo criado (opcional).

É possível gerar um dataset ou um registro de dataset a partir de uma entidade. No segundo caso, mesmo que o dataset ainda não esteja cadastrado no repositório de dados, ele poderá ser gerado. No entanto, é desnecessário informar o número do registro, pois será gerado o registro de número "00".

Para criar um registro de formato variável para o dataset, deve-se informar o número do registro.Para gerar mais do que um registro, é necessário que na definição do dataset isto esteja indicado através do campo REGISTRO. É possível cadastrar um registro intercalado entre dois já existentes.

Tipo - Tipo de dataset a ser criado.

Os valores possíveis são os seguintes:

"S" (Standard);

"C" (Compact);

"R" (Random);

"D" (Direct).

Definição de dataset

A função de definição de dataset deve ser utilizada quando não houver uma entidade para, a partir dela, gerar o dataset e, também, para alterar definições geradas a partir de entidades.

CAMPOS DA TELA

DATASET - Código do dataset no padrão SADS (sssBxx).

NOME - Descrição resumida do dataset.

REMAP DE - Código do dataset que estiver sendo "remapeado".

Este campo somente deve ser informado nos casos de definição de remaps.

Remap é a visão lógica de um dataset já definido. A definição de remaps não exige definições prévias das estruturas dos registros, sets ou subsets do dataset a ser remapeado.

NOME PADRAO NA INSTALACAO - Código do dataset fora do padrão SADS.

Este código é utilizado quando a instalação possui um padrão específico para nomes de datasets, diferente do padrão adotado para criação através do SADS, ou para datasets que já existam na instalação e que não foram criados através do SADS.

Caso este campo seja informado, será utilizado em todas as gerações feitas pelo SADS, isto é, o nome padrão na instalação substituirá o código padrão do SADS tanto em fontes DASDL e de programas como nas libraries de parâmetros físicos e de declaração de bancos.

REGISTROS - Quantidade de registros a serem definidos.

Para datasets de formato fixo, deve-se informar "00".

Para datasets de formato variável, deve-se informar quantos tipos o dataset possui, independente do registro "00" (parte fixa que contém os campos chaves). Por exemplo: se o dataset possuir os registros "00", "01" e "02", deve-se informar "02" no campo REGISTROS.

TIPO DE DATASET - Tipo de organização do dataset.

Os valores possíveis são os seguintes:

"S" (Standard);

"C" (Compact);

"D" (Direct);

"R" (Random);

"T" (Restart Dataset).

O tipo "T" só é válido para os datasets de código "SBGB01", "SBGB03", "SBGB04". "SBGB05", "SBGB06" ou "SBGB07".

Para utilização de restart dataset com código diferente de "SBGB01", a tabela de opções de compilação, SBPM5018, deve ser utilizada, informando-se as opções "SADSRDS03" para uso do "SBGB03", "SADSRDS04" para uso do "SBGB04","SADSRDS05" para uso do "SBGB05","SADSRDS06" para uso do "SBGB06" ou "SADSRDS07" para uso do "SBGB07". Detalhes a este respeito estão descritos na seção V, "Parametrizações do SADS", neste manual.

INCORPORAR

NOMES EXTERNOS DE ATRIBUTOS - Indicador de utilização de código externo de atributo.

Pode assumir os seguintes valores:

"S" - Para que os atributos pertencentes à estrutura do dataset sejam referenciados pelos seus códigos externos.

Neste caso, os códigos externos dos atributos serão utilizados em todas as gerações feitas pelo SADS, isto é, os códigos externos substituirão os códigos padrões dos atributos nos fontes DASDL e de programas.

"N" - Para que os atributos pertencentes à estrutura do dataset não sejam referenciados pelos seus códigos externos e sim pelos seus mnemônicos (códigos).

ESTRUTURA DE HEADER PADRAO - Indicador de incorporação do header padrão da instalação no dataset.

Os valores possíveis são os seguintes:

"S" - Para incorporar o header padrão na geração do DASDL do dataset;

"N" - Para não incorporar o header padrão na geração do DASDL do dataset.

O dataset poderá ter um header específico, não ter header algum ou incorporar o header padrão da instalação ou do banco de dados, definido na tabela SBPM8015, cujos detalhes estão descritos na Seção V, "Parametrizações do SADS".

O valor default para incorporação de header padrão pode ser parametrizado na tabela SBPM5000, cujos detalhes estão descritos na Seção V, "Parametrizações do SADS".

Se a incorporação do header padrão for definida, no momento da geração do fonte DASDL do dataset, os campos definidos na tabela serão incluídos no início da estrutura do mesmo. Os campos de utilização genérica (NDEF="G") que fizerem parte do header, tanto o padrão quanto o específico, serão tratados nas gerações de programas executados através do SADS.

PARAMETROS FISICOS DEFAULTS - Indicador de incorporação de defaults de parâmetros físicos pré-definidos para os datasets da instalação.

Os valores possíveis são seguintes:

"S" - Para incorporar os parâmetros físicos definidos para todos os datasets da instalação na geração do DASDL do dataset;

"N" - Para não incorporar os parâmetros físicos de datasets da instalação na definição do dataset.

Os parâmetros físicos podem ser definidos especificamente para um dataset, exceto direct dataset, bem como para todos os datasets da instalação, na tabela SBPM5090. Detalhes a este respeito estão descritos na Seção V, "Parametrizações do SADS", neste manual.

Caso tenha sido definido a incorporação de defaults de parâmetros físicos para um dataset e se, no momento da geração de sua library de parâmetros físicos, alguma informação não estiver definida para o mesmo, esta informação será suprida pela contida na tabela de parâmetros físicos de datasets da instalação (Tabela SBPM5090).

LIBRARY PARA GERACAO DE DASDL - Código da library a ser incluída no fonte DASDL do dataset.

Pode-se declarar uma library que contenha definições de parâmetros físicos que não estejam sendo contemplados na versão corrente do SADS.

NOME EXTERNO ATRIBUTO RECORD TYPE - Nome externo que será gerado para o atributo tpregistro.

Cadastramento de estrutura de registro de dataset

Esta operação permite definir a estrutura dos registros do dataset, com as especificações dos atributos e campos que a compõem, seus relacionamentos e suas particularidades.

No contexto de dataset, entenda-se atributo como atributo propriamente dito, campo ou sinônimo.

CAMPOS DA TELA

DATASET - Código do dataset de acordo com a padronização adotada pelo SADS (sssBxx).

REGISTRO - Número que identifica o formato do registro (record type) nas estruturas do dataset.

NOME - Descrição resumida do dataset.

SEQ - Número sequencial que indica o posicionamento do atributo dentro da estrutura do dataset.

Este campo já vem informado, sendo o primeiro número igual a "0010" e os demais acrescidos de 10.

Para incluir um atributo entre dois já incluídos, deve-se preencher uma linha que estiver em branco e alterar o número sequencial desta linha para um número entre os já informados. O SADS reorganizará automaticamente a estrutura. Na próxima operação, os atributos já estarão na nova ordem.

ATRIBUTO - Mnemônico do atributo que fará parte da estrutura do dataset.

As estruturas dos datasets podem ser compostas por atributos ou campos. A tabela SBPM7002 e o parâmetro "EMPCOD" da tabela SBPM5000 definem estas possibilidades. Detalhes a este respeito estão descritos na Seção V, "Parametrizações do SADS".

A seguir, é apresentada uma lista contendo os atributos e campos genéricos do SADS, comumente utilizados na estrutura dos datasets, e cujo nível de definição é geral (NDEF = "G"):

EMPCOD - Código de empresa;

EMPNOM - Nome da empresa;

EMPSIGLA - Sigla da empresa;

ETBCOD - Código de estabelecimento;

ETBNOM - Nome do estabelecimento;

HDRDATA - Data para header de dataset;

HDRESTACAO - Número de estação para header de dataset;

HDRHORA - Hora para header de dataset;

HDRPROGRAMA - Código de programa para header de dataset;

HDRSTATGER - Status geral para header de dataset;

HDRUSUNUM - Código de usuário para header de dataset;

UNDCOD - Código de unidade;

UNDNOM - Nome de unidade.

NIVEL - Nível de estruturação do atributo, análogo ao nível de área de work dos programas Cobol.

O nível inicial é o 10. Se for necessário criar agrupamentos de atributos, deve-se usar múltiplos de 5, a partir de 15, até 45.

NDEF - Nível de definição do atributo.

Os valores possíveis são os seguintes:

"S" (Sistema) - Para atributo definido no sistema informado no campo SIS ou para atributo do mesmo sistema do dataset;

"G" (Geral) - Para atributo genérico do SADS;
Caso NDEF não seja informado, o atributo será interpretado como um item grupo na estrutura do dataset. Porém, paara os atributos que farão parte das chaves de acesso, o nível de definição deve obrigatoriamente ser informado.

SIS - Código do sistema do atributo.

Este campo deve ser informado somente se NDEF = "S". Se o atributo estiver definido no mesmo sistema do dataset em questão, não é necessário informá-lo.

OCCURS - Número de ocorrências do atributo para criação de grupos repetitivos dentro da estrutura.

Para criar um grupo repetitivo, é necessário definir um item grupo, pois os itens elementares não aceitam o preenchimento do campo OCCURS.

Se o dataset possuir um atributo multivalorado em sua estrutura, o fonte DASDL será gerado com a cláusula "occurs". O programa batch SD/U/PC/09 emite um relatório com a relação de todos os datasets da instalação que possuem atributos multivalorados em sua estrutura.

CLAUSULA - Tipo de relacionamento para o atributo.

Os valores possíveis são os seguintes:

"C" (Comentário) - Indica que o atributo funciona como uma redefinição do anterior.

Como o DMSII não permite redefinição, apenas será gerado um comentário no fonte DASDL do dataset. Caso este atributo esteja referenciado em outras estruturas, esta redefinição será observada nas movimentações geradas nos programas. Esta cláusula exige que o atributo anterior existente na estrutura do registro seja informado no campo COMPLEMENTO.

"D" (Cláusula Depending On)

No campo COMPLEMENTO, deve-se informar o atributo numérico pertencente à estrutura que está sendo definida e para o qual a cláusula depending on será gerada.

"E" (Nome externo) - Indica que o nome colocado no campo COMPLEMENTO deverá ser assumido na geração do DASDL das estruturas e dos programas, em detrimento do nome gerado no padrão SADS, facilitando, desta forma, a integração com bancos de dados já existentes;

"I" (Initialvalue) - Indica que o atributo terá o valor inicial informado no campo COMPLEMENTO;

"N" (Null) - Indica que o valor informado no campo COMPLEMENTO será utilizado para implementar a opção "NULL" do DMSII;

"R" (Required) - Indica que a informação do atributo será obrigatória na atualização do dataset;

"S" (Cláusula Stored Depending On ou Stored Optionaly) - Somente para compact dataset.

A cláusula Stored Depending On será gerada no fonte DASDL se um atributo númerico, pertencente à estrutura, for informado no campo COMPLEMENTO. Se nenhum atributo for informado no campo COMPLEMENTO, será gerada a cláusula Stored Optionaly.

"T" (Restart Dataset) - Indica que o atributo é um dos obrigatórios na estrutura do Restart Dataset.

Esta cláusula somente é válida para atributos pertencentes ao Restart Dataset.
Neste caso, no campo COMPLEMENTO deve ser informado qual o tipo do atributo especificado, que deve ser, nesta mesma sequência:

"TRANSACTION COUNT";

"RESTART TYPE";

"COMS-ID";

"COMS-PROGRAM" e

"COMS-LOCATOR".

"V" (Cláusula Size Varying Depending On ou Size Varying) - Somente para compact dataset.

A cláusula Size Varying Depending On será gerada no fonte DASDL se um atributo númerico, pertencente à estrutura, for informado no campo COMPLEMENTO. Se nenhum atributo for informado no campo COMPLEMENTO, será gerada a cláusula Size Varying.

Em uma estrutura de dataset, um atributo pode possuir várias cláusulas, bastando para isto repetir a informação do atributo, com uma outra cláusula.

COMPLEMENTO - Este campo tem sua utilização dependente da informação do campo CLAUSULA, conforme explicações anteriores.

CONTINUA (S/N) - Indicador de continuidade de operação.

Pode assumir os seguintes valores:

"S" - Para prosseguir na operação que está sendo efetuada;

"N" - Para encerrar a operação que está sendo efetuada.

Se uma tela não for suficiente para a informação de todos os atributos da estrutura do dataset, ao informar "S" no campo CONTINUA, uma nova tela será mostrada para informação de outros atributos.

Cadastramento de chaves de acesso (set/subset/access)

Set, subset e access são estruturas utilizadas para recuperação de registros em datasets, podendo também representar relacionamentos entre registros de datasets.

Set é utilizado para executar acessos a registros em datasets, em uma sequência lógica determinada. Cada set é um arquivo independente que contém uma coleção de índices ou ponteiros voltados para registros de datasets. Para cada registro do dataset há um equivalente em cada set.

Subset também é utilizado para executar acessos a registros em datasets, em alguma sequência lógica determinada. Porém, somente registros do dataset que satisfaçam a condição determinada para o subset terão nele um registro relacionado, ou seja, cada subset é, também, um arquivo independente que contém uma coleção de índices ou ponteiros voltados para alguns registros de um único dataset.

Os subsets podem ser automáticos ou manuais, sendo que para os automáticos existem condições agregadas denominadas "WHERE CONDITIONS".

Access é usado para recuperar registros em direct e random datasets. Estes tipos de datasets devem possuir um único access declarado, pois eles não são arquivos independentes. Portanto, o access define simplesmente como serão ordenados os registros dos datasets.

Para efetuar o cadastramento de set/subset/access, é necessário que o dataset e sua estrutura estejam cadastrados no repositório de dados do SADS.

CAMPOS DA TELA

CHAVE - Código alfanumérico que identifica a chave de acesso no padrão SADS (sssBxxa).

As chaves de acesso de um dataset são compostas pelo código do dataset mais uma letra que irá diferenciar as diversas chaves de acesso de um mesmo dataset.

NOME - Descrição resumida da chave de acesso.

PADRAO INST. - Código da chave de acesso fora do padrão SADS.

Este código é utilizado quando a instalação possui um padrão específico para nomes de estruturas DMSII, diferente do adotado pelo SADS, ou é utilizado para definir estruturas que já existam na instalação e que não foram criadas através do SADS.

Caso este campo seja informado, ele será utilizado em todas as gerações efetuadas pelo SADS, isto é, o nome padrão na instalação substituirá o código padrão do SADS tanto em fontes DASDL e de programas como nas libraries de parâmetros físicos e de declaração de bancos.

TIPO - Tipo de chave a ser utilizada.

Os valores possíveis são os seguintes:

"S" (Set);

"T" (Subset);

"A" (Access).

AUT/MAN. - Tipo do subset.

Os valores possíveis são seguintes:

"A" - Para subset automático;

"M" - Para subset manual.

Para subset automático, a condição para seleção dos registros a serem incluídos na chave é verificada automaticamente pelo DMSII. Portanto, é necessário que esta condição - WHERE CONDITION - seja definida no repositório de dados do SADS. Detalhes a este respeito estão descritos no tópico "Cadastramento de condições para subsets automático", neste mesmo item.

O subset deve ser declarado como manual quando a condição para seleção for verificada pelos programas aplicativos.

ORGAN. - Tipo de organização de sets e subsets.

Os valores possíveis são os seguintes:

"IS" (Index sequential);

"IR" (Index random);

"OL" (Ordered list).

O default assumido como tipo de organização da chave de acesso é "IS" (Index Sequential).

DUPLICATES - Indicador de registros duplicados na chave.

Pode assumir os seguintes valores:

"N" (No duplicates);

"K" (No duplicates keychangeok);

"D" (Duplicates);

"F" (Duplicates first);

"L" (Duplicates last).

VISAO DO SET - Código do set ao qual a visão se refere.

Caso esteja sendo cadastrada uma visão, deve-se então informar o código do set ao qual esta visão se refere. Neste caso o set deve ter sido anteriormente cadastrado e não deve ser informada a estrutura de tal visão, pois será assumida a mesma estrutura do set ao qual ela se refere.

INCORPORAR DEFAULTS NA GERACAO - Indicador de incorporação de defaults de parâmetros físicos para as estruturas de acesso na geração do DASDL do dataset.

Os valores possíveis são os seguintes:

"S" - Para incorporar os defaults padrão da instalação na geração da library de
parâmetros físicos do dataset;

"N" - Para não incorporar os defaults padrão da instalação na geração da library de
parâmetros físicos do dataset.

Os parâmetros físicos podem ser definidos para todas as chaves de acesso de todos os datasets da instalação na tabela SBPM5090. Detalhes a este respeito estão descritos na Seção V, "Parametrizações do SADS", neste manual.

Caso a incorporação de defaults da instalação tenha sido solicitada e se, no momento da geração da library de parâmetros físicos, alguma informação não estiver definida para a chave de acesso, esta informação será suprida pela contida na tabela de parâmetros físicos de datasets da instalação (Tabela SBPM5090).

LIBRARY PARA GERACAO DE DASDL - Código da library a ser incluída no fonte DASDL do dataset.

Pode-se incluir uma library que contenha definições de parâmetros físicos para estruturas de acesso DMSII que não estejam sendo contempladas pela versão corrente do SADS.

SEQ - Número sequencial para posicionamento do atributo na estrutura da chave.

Este campo já vem informado, sendo o primeiro número igual a "0010" e os demais, acrescidos de 10.

Para incluir um atributo entre dois já existentes, deve-se preencher uma linha que estiver em branco e alterar o número sequencial desta linha para um número entre os já informados. O SADS reorganizará automaticamente a estrutura. Na próxima operação, os atributos já estarão na nova ordem.

ATRIBUTO - Mnemônico do atributo que compõe a chave de acesso ou que será utilizado como campo de dado redundante na estrutura da chave (key data).

Este atributo deve, obrigatoriamente, fazer parte do registro "00" do dataset, caso o dataset possua formato variável.

Não é possível especificar um atributo com tamanho maior do que o permitido pelo DMSII para chaves.

CLASSIF. - Tipo de classificação dos registros utilizando o atributo especificado.

Para classificação em ordem descendente, deve-se informar "D".

Caso este campo não seja informado, será assumida uma ordem ascendente na classificação.

DADO(S) - Indicador de campo de dados na estrutura da chave.

Deve-se informar "S" para atributo que será armamazenado juntamente com a chave de acesso, de forma redundante, mas que não será chave dentro da estrutura.

Caso este campo não seja informado, o atributo será utilizado como chave de acesso.

A utilização deste recurso economiza tempo no acesso a informações que não são chaves, mas que são utilizadas com muita frequência.

CONTINUA (S/N) - Indicador de continuidade de operação.

Os valores possíveis são os seguintes:

"S" - Para prosseguir com a definição de chave de acesso;

"N" - Para encerrar a operação.

Cadastramento de condições para subsets automáticos

As where conditions - condições para gravação de subsets automáticos - podem ser definidas no repositório de dados do SADS para utilização posterior em gerações de DASDL de dataset.

Para defini-las, é necessário que o subset automático, para o qual as where conditions serão definidas, já esteja cadastrado no repositório de dados e que os atributos e campos que irão compô-las façam parte do registro "00", caso o dataset possua formato variável.

CAMPOS DA TELA

SUBSET - Código do subset automático, no padrão SADS (sssBxxa).

SEQ - Número sequencial que indica o posicionamento do item dentro da condição para o subset.

Este campo já vem informado, sendo o primeiro número igual a "0010" e os demais, acrescidos de 10.

Para incluir um item entre dois já incluídos, deve-se preencher uma linha que estiver em branco e alterar o número sequencial desta linha para um número entre os já informados. O SADS reorganizará automaticamente a estrutura. Na próxima operação, os itens já estarão na nova ordem.

E/OU - Conector lógico utilizado para comparação de operandos.

Os valores possíveis são os seguintes:

"E" - e (and);

"O" - ou (or).

ABRE PAR - Indicador de abre parêntese ("(") para agrupamento das expressões.

É possível definir mais de dois (2) níveis de parênteses em um mesmo agrupamento. Para isto, deve-se cadastrar uma nova sequência com TIPO "N".

ATRIBUTO - Mnemônico do atributo ou campo para o qual será definida uma condição.

O atributo ou campo deve pertencer à estrutura fixa do dataset (registro "00"), caso ele possua formato variável. O atributo ou campo pode também pertencer ao header padrão, caso o dataset o possua.

OPER - Operador relacional.

Os valores possíveis são os seguintes:

"NE" (Not equal) - Diferente de;

"EQ" (Equal) - Igual a;

"GT" (Greater than) - Maior que;

"GE" (Greater or equal) - Maior que ou igual a;

"LT" (Less than) - Menor que;

"LE" (Less or equal) - Menor que ou igual a.

TIPO - Tipo do operando a ser utilizado na expressão.

Este campo deverá ser preenchido de acordo com o conteúdo do campo OPERANDO.

Os valores possíveis são os seguintes:

"B" - Caso o atributo seja booleano;

"C" - Caso o operando seja um atributo ou campo;

"K" - Caso o operando seja uma constante;

"N" - Para mais de dois(2) níveis de parênteses em um mesmo agrupamento;

"-" - Para concatenação de constantes com mais de 11 (onze) caracteres.

O valor "-" deve ser informado no campo TIPO quando a constante a ser testada tiver mais de 11 posições. Este caractere ("-") indica que a constante continuará a ser definida na próxima linha.

Quando o conteúdo do tipo for "B", o operando deve ser "V" (verdadeiro) ou "F" (falso).

OPERANDO - Valor constante, numérico ou alfanumérico, ou mnemônico de atributo ou campo.

O operando deverá ser informado de acordo com o formato do atributo ou campo definido no campo ATRIBUTO, da seguinte forma:

Caso o atributo ou campo possua formato numérico e o operando seja uma constante, ela deve ser ajustada à direita. Não é necessário preencher com zeros à esquerda;

Caso o atributo ou campo possua formato alfabético ou alfanumérico e o operando seja uma constante, ela deve ser ajustada à esquerda.

Caso o teste seja feito considerando-se spaces, o número de brancos desejado deve ser informado entre aspas.

FECHA PAR - Indicador de fecha parêntese (")" ) para encerrar a condição iniciada por abre parêntese ("(" ).

É possível definir mais de dois (2) níveis de parênteses em um mesmo agrupamento. Para isto, deve-se cadastrar uma nova sequência com TIPO "N".

CONTINUA (S/N) - Indicador de continuidade de operação.

Os valores possíveis são os seguintes:

"S" - Para prosseguir na definição das condições para subset automático;

"N" - Para encerrar a operação.

Cadastramento de parâmetros físicos de dataset e chaves de acesso

Parâmetros físicos são características que o dataset e suas chaves de acesso assumirão no momento da compilação do DASDL do banco de dados.

Os parâmetros físicos podem ser definidos especificamente para um dataset, exceto direct dataset, bem como para todos os datasets da instalação, na tabela SBPM5090. Detalhes a este respeito estão descritos na Seção V, "Parametrizações do SADS", neste manual.

Caso tenha sido definido a incorporação de defaults de parâmetros físicos para um dataset, as seguintes situações irão ocorrer:

No momento do cadastramento de seus parâmetros físicos, os respectivos defaults cadastrados na tabela de parâmetros físicos de datasets da instalação (Tabela SBPM5090) já virão exibidos na tela, bastando confirmá-los ou alterá-los, se necessário;

No momento da geração de sua library de parâmetros físicos, se alguma informação não estiver definida para o mesmo, esta informação será suprida pela contida na tabela SBPM5090.

Para cadastrar parâmetros físicos para dataset e chaves de acesso, é necessário que os mesmos estejam definidos no repositório de dados do SADS.

CAMPOS DA TELA

Dataset/set/subset - Código do dataset ou da chave de acesso.

Nome - Nome do dataset ou da chave de acesso informada.

Banco - Código do banco de dados que possui o dataset para o qual serão especificados os parâmetros físicos (sssBxx).

Um dataset pode estar relacionado a bancos de dados diferentes. Para cada banco relacionado, o dataset poderá assumir parâmetros físicos diferentes. Deve-se atentar para a diferença existente entre bancos de teste e de produção.

Ambiente - Ambiente no qual serão válidos os parâmetros físicos do dataset ou chave de acesso, podendo ser teste ou produção.

Areas - Número de áreas a serem alocadas em disco.

O valor máximo permitido é 1000, com exceção das System areas.

Areasize - Tamanho de cada área, na unidade especificada.

Unidade - Unidade que especifica o tamanho de cada área.

As unidades possíveis são as seguintes:

"R" (Records) - Para que o tamanho seja expresso em número de registros;

"S" (Segments) - Para que o tamanho seja expresso em quantidade de segmentos;

"B" (Blocks) - Para que o tamanho seja expresso em número de blocos;

"Y" (bytes) - Para que o tamanho seja expresso em número de bytes.

O default do SADS para a unidade do campo AREASIZE é Blocks.

Blocksize - Tamanho do bloco, na unidade especificada.

Unidade - Unidade que especifica o tamanho de cada bloco.

As unidades possíveis são as seguintes:

"R" (Records) - Para que o tamanho seja expresso em número de registros;

"S" (Segments) - Para que o tamanho seja expresso em quantidade de segmentos;

"W" (Words) - Para que o tamanho seja expresso em número de palavras;

"Y" (bytes) - Para que o tamanho seja expresso em número de bytes.

O default do SADS para a unidade do campo BLOCKSIZE é Words.

AVERAGE RECORD - Tamanho médio de registros de tamanho variável em compact datasets.

O tamanho médio é definido em words.

BUFFERS - Número de buffers que o sistema irá alocar para a estrutura, mais o número de buffers adicionais que o sistema deverá alocar para cada programa que utilizá-la.

CHECKSUM - Indicador de utilização de checksum.

Os valores possíveis são os seguintes:

"S" - Para o uso de checksum;

"N" - Para eliminar o uso de checksum (default DMSII).

O checksum é utilizado para controlar erros de I/O, através da comparação de valores calculados na gravação e leitura de blocos de dados, podendo ser declarado em nível de estrutura.

DIGITCHECK - Indicador de utilização de digitcheck.

Os valores possíveis são os seguintes:

"S"- Para utilização do digitcheck;

"N"- Para eliminação do uso do digitcheck.

No caso de utilização do digitcheck, os itens number são examinados a fim de detectar valores hexadecimais inválidos na compatibilização entre máquinas.

Este parâmetro é válido somente para datasets.

MEMORYRESIDENT - Identificador de tipo de utilização da memória.

Pode assumir os sseguintes valores:

"ALL" - Para que os buffers usados permaneçam em memória até o último usuário da estrutura fechar o banco;

"COARSE" - Para que somente os coarse tables sejam mantidos em memória. Válido apenas para sets sequenciais indexados.

PACK - Nome do disco no qual será armazenado o elemento.

O disco default do DMSII é o Disk.

POPULATION - Tamanho esperado da estrutura em número de registros.

SECURITYGUARD - Nome do guardfile a ser utilizado.

REBLOCK - Indicador de utilização de tamanhos de blocos diferentes numa mesma estrutura.

Pode assumir os seguintes valores:

"S" - Para utilizar dois tamanhos diferentes de blocos numa mesma estrutura;

"N" - Para utilizar apenas um tamanho de bloco na estrutura.

REBLOCKFACTOR - Fator que dimensiona os blocos, quando estiverem sendo utilizados dois tamanhos diferentes.

O maior bloco, para acessos seriais, é calculado como o resultado da multiplicação do tamanho do bloco pelo valor do reblockfactor.

Esse campo deve ser informado apenas se o campo REBLOCK estiver preenchido com "S".

O valor máximo é 60 e o default é 1.

LOADFACTOR - Fator percentual que determina a alocação de entradas em tabelas com criação de novas tabelas quando necessário.

Parâmetro válido somente para sets e subsets.

O default do DMSII é 66%.

PRIME - Indica que o set é primário no acesso ao dataset.

O set primário é usado para reordenar o dataset durante a reorganização.

Só é válida a especificação de um set com o parâmetro PRIME, por dataset.

Parâmetro válido somente para sets.

TABLESIZE - Tamanho das tabelas usadas em sets e subsets (o tamanho das tabelas é definido em words).

Geração de fonte DASDL de dataset

Para que o DASDL de um dataset possa ser gerado, é necessário que o dataset, seus registros, suas chaves de acesso e as where conditions dos subsets automáticos já estejam cadastrados no repositório de dados do SADS.

Essa geração cria um fonte DASDL, no ambiente informado, cujo código possui o seguinte padrão: ss/s/BL/xx.

Exemplo: 

Dataset: XYZB02;
Fonte DASDL gerado: XY/Z/BL/02.

No fonte DASDL gerado, serão incluídos os nomes dos atributos do dataset como comentário e, caso o mesmo tenha sido definido como externo, seus atributos serão referenciados pelos seus códigos externos, ao invés de pelos seus códigos.

Se o dataset possuir um atributo multivalorado em sua estrutura, o fonte DASDL será gerado com a cláusula occurs. O programa batch SD/U/PC/09 emite um relatório com a relação de todos os datasets da instalação que possuem atributos multivalorados em sua estrutura.

Para a geração de fonte DASDL de todos os datasets de um sitema, pode-se utilizar a função de geração de família. Esta função está descrita na Seção III, "Utilitários on-line".

A geração de fonte DASDL de dataset não está associada a nenhuma tela de processamento. Para efetuá-la, basta transmitir o menu de operação "GERDAT", cujos campos estão explicados a seguir:

CAMPOS DA TELA

Dataset - Código do dataset no padrão SADS (sssBxx).

Ambiente - Ambiente onde será gerado o DASDL.

Os valores possíveis são os seguintes:

"T" (Teste);

"P" (Produção).

Caso o ambiente não seja informado, será assumido o ambiente default do usuário. Detalhes sobre o cadastramento do ambiente default do usuário podem ser obtidos na Seção VI, "Sistema de Segurança", item "Usuários e habilitações".

Usercode - Usercode de análise.

O usercode a ser informado deve estar habilitado ao usuário, como elemento, em sua lista de elementos "SEGUSER", no Sistema de Segurança. Detalhes sobre este cadastramento estão descritos na Seção VI, "Sistema de Segurança", item "Habilitações de usuários SADS".

Caso o usercode não seja informado, será assumido o usercode default do usuário. Detalhes sobre o cadastramento do usercode default do usuário podem ser obtidos na Seção VI, "Sistema de Segurança", item "Usuários e habilitações".

Para usercodes de análise, são definidos os ambientes (usercode e disco) onde ficarão residentes suas libraries de teste e produção, sendo possível que as libraries de produção fiquem sem usercodes. Estas definições estão na tabela SBPM5002, que se encontra descrita na Seção V, "Parametrizações do SADS".

Geração de DASDL de parâmetros físicos de dataset

O SADS permite gerar o fonte DASDL relativo aos parâmetros físicos de datasets e de suas chaves de acesso.

Essa geração cria um fonte DASDL, no ambiente informado, com o seguinte padrão:

"ss/s/BL/xx/sssBxx" onde:

"ss/s/BL/xx/" é derivado do código do dataset e

"sssBxx" é o código do banco de dados.

Exemplo: 

Dataset:                        XYZB01;
Banco de dados:                 XYZB99;
Fonte DASDL do dataset: XY/Z/BL/01;
Fonte DASDL de parâmetros
físicos do dataset no banco
de dados:                       XY/Z/BL/01/XYZB99.

No fonte DASDL do banco de dados, é gerado um comando include para este fonte DASDL de parâmetros físicos. Portanto, a geração dos parâmetros físicos de dataset é obrigatória e deve ser feita antes da compilação do banco, mesmo que não existam parâmetros definidos. Caso os parâmetros fisicos não sejam gerados, haverá erro na compilação do banco de dados.

Esta library não pode ser submetida a uma liberação para produção, uma vez que os bancos de teste e produção são diferentes.

A geração de DASDL de parâmetros físicos de dataset não está associada a nenhuma tela de processamento. Para efetuá-la, basta transmitir o menu de operação "GERPARFIS", cujos campos estão explicados a seguir:

CAMPOS DA TELA

Dataset - Código do dataset cuja estrutura será gerada (sssBxx).

Banco - Código do banco para o qual o DASDL será gerado.

Ambiente (P/T) - Ambiente para geração do DASDL.

Os valores possíveis são os seguintes:

"P" (Produção);

"T" (Teste).

Caso o ambiente não seja informado, será assumido o ambiente default do usuário. Detalhes sobre o cadastramento do ambiente default do usuário podem ser obtidos na Seção VI, "Sistema de Segurança", item "Usuários e habilitações".

Usercode - Usercode de análise.

O usercode a ser informado deve estar habilitado ao usuário, como elemento, em sua lista de elementos "SEGUSER", no Sistema de Segurança. Detalhes sobre este cadastramento estão descritos na Seção VI, "Sistema de Segurança", item "Habilitações de usuários SADS".

Caso o usercode não seja informado, será assumido o usercode default do usuário. Detalhes sobre o cadastramento do usercode default do usuário podem ser obtidos na Seção VI, "Sistema de Segurança", item "Usuários e habilitações".

Para usercodes de análise, são definidos os ambientes (usercode e disco) onde ficarão residentes suas libraries de teste e produção, sendo possível que as libraries de produção fiquem sem usercodes. Estas definições estão na tabela SBPM5002, que se encontra descrita na Seção V, "Parametrizações do SADS".

Para a geração de fonte DASDL de todos os parâmetros físicos de datasets de um banco de dados, pode-se utilizar a função de geração de família. Este procedimento está descrito na Seção III, "Utilitários on-line".
