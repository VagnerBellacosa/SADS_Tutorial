Seção IV

Esta seção contém a lista de atividades a serem executadas para determinados tipos de manutenção possíveis de realização durante o ciclo de vida dos sistemas aplicativos desenvolvidos através do SADS. As listas foram criadas para as atividades de manutenção consideradas mais complexas, por serem executadas em vários passos. Não existem nesta seção, portanto, listas de atividades para todas as operações de manutenção possíveis.

Cada manutenção é composta dos pré-requisitos para execução das atividades e das atividades propriamente ditas. Segue a cada atividade a identificação do menu de operação a partir do qual se pode executá-la, quando for o caso.

Inclusão de um novo dataset em um banco de dados já cadastrado

Neste item são apresentadas as atividades necessárias à definição de um dataset não cadastrado no repositório de dados e seu relacionamento aos bancos de dados através dos quais ele será utilizado.

Pré-requisitos

Os bancos de dados, aos quais o dataset será relacionado, bem como os atributos e campos que irão compor sua estrutura, têm de estar cadastrados no repositório de dados. Caso o dataset seja gerado a partir de uma entidade de dados, ela tem de estar com seu cadastramento completo no repositório de dados do SADS.

Atividades

1 - Cadastrar o dataset:

Para dataset gerado a partir de uma entidade já cadastrada:

Gerar o dataset a partir da entidade - Menu: CRIDATENT.

Para adaptar o cadastramento, se necessário:

Definição - Menu: DEFDAT;

Estrutura - Menu: ESTDAT;

Chaves de acesso - Menu: CHADAT;

Condições para subsets automáticos (se for o caso) - Menu: CONSUB.

Para dataset não gerado a partir de uma entidade já cadastrada:

Definição - Menu: DEFDAT;

Estrutura - Menu: ESTDAT;

Chaves de acesso - Menu: CHADAT;

Condições para subsets automáticos (se for o caso) - Menu: CONSUB.

2 - Gerar DASDL do dataset para teste - Menu: GERDAT;

3 - Incluir o dataset na relação de datasets que compõem o Banco de dados de teste - Menu: DEFBAN;

4 - Definir parâmetros físicos do dataset para o banco de dados de teste (se for o caso) - Menu: PARDAT;

5 - Gerar library de parâmetros físicos do dataset para teste - Menu: GERPARFIS;

6 - Gerar a DASDL do banco de dados para teste - Menu: GERDAS;

7- Compilar o banco de dados de teste - Esta atividade deve ser executada pelo administrador de banco de dados;

8 - Inicializar o dataset no banco de dados de teste - Esta atividade deve ser executada pelo administrador de banco de dados.

Cumpre ressaltar que, para relacionar o dataset a um banco de dados de produção, deve-se executar as seis últimas atividades apresentadas para o banco de dados de produção.

Inclusão de um novo dataset na base de dados on-line do sistema

Neste item são apresentadas as atividades necessárias à inclusão de um dataset na base de dados on-line de um sistema, ou seja, na base que dá origem à library de declaração de banco (ss/s/BL/00).

Pré-requisitos

É necessário que o dataset já tenha sido cadastrado no repositório de dados e que esteja relacionado a um banco de dados de produção.

Atividades

1 - Incluir o dataset na base de dados do sistema (atenção para incluir o dataset na base de dados on-line do sistema) - Menu: BASSIS;

2 - Gerar library de declaração de banco para teste - Menu: GERBAN;

3 - Gerar os hosts do sistema que utilizam o banco de dados no qual o dataset foi incluído - Menu: GERHOS;

4 - Compilar hosts para teste - Menu: COMHOS;

5 - Incluir o dataset como recurso dos programas que irão utilizá-lo - Menu: RECPRO;

6 - Alterar os programas que irão utilizar o novo dataset:

Para programas gerados totalmente pelo SADS:

Incluir o dataset como recurso do programa - Menu: RECPRO;

Alterar definição de processamento dos programas que tratarão o dataset - Menu: LOGPRO;

Para dataset tratado em módulos já cadastrados do programa:

Alterar os módulos que utilizarão o dataset - Menu: CADMOD.

Para dataset tratado em módulos não cadastrados do programa:

Selecionar os módulos que tratarão o dataset - Menu: SELMOD;

Incluir os novos módulos de processamento para tratamento do dataset - Menu: CADMOD;

Gerar os fontes dos programas - Menu: GERPRO.

Para programas não gerados totalmente pelo SADS:

Movimentar para usercode de análise todos os fontes de programas on-line relacionados ao host - Menu: COPFONMAN ou COPFAMFON para movimentar todos os fontes de um sistema;

Efetuar as manutenções necessárias nos programas que irão tratar o dataset - Esta atividade deve ser efetuada através de um editor de programas (Cande).

7 - Compilar todos os programas alterados para teste - Menu: COMPRO;

8 - Executar binder dos hosts para teste - Menu: BINPRO (como os hosts foram recém-compilados, deve-se executar o binder para todos os programas do host);

9 - Desabilitar/habilitar hosts no ambiente de teste - Pode ser feito através dos comandos "?Disable program <<sssnn >>" e "?Enable program <<sssnn>>" . Estes comandos só podem ser executados por usercodes autorizados;

10 - Testar os programas;

11- Liberar para produção:

Gerar a library de declaração de banco para produção - Menu: GERBAN;

Compilar hosts para produção - Menu: COMHOS;

Compilar todos os programas relacionados aos hosts para produção - Menu: COMPRO;

Executar binder dos hosts para produção- Menu: BINPRO;

Liberar os hosts e os programas para produção:

Verificar padrões de codificação dos programas (exceto dos hosts) - Menu: CONPADCOD;

Atualizar recursos de fonte de programa no repositório de dados - Menu: ATURECSDD;

Liberar fontes dos hosts e dos programas para biblioteca de fontes compactados - Menu: LIBFON;

Para ambientes de teste e produção em máquinas diferentes:

Copiar objetos dos hosts do usercode de produção de uma máquina para outra.

Desabilitar/habilitar hosts no ambiente de produção. Pode ser feito através dos comandos "?Disable program <<sssnn >>" e "?Enable program <<sssnn>>". Estes comandos só podem ser executados por usercodes autorizados.

2 - Alterar os programas que irão utilizar o novo dataset:

Para programas gerados totalmente pelo SADS:

Incluir o dataset como recurso dos programas - Menu: RECPRO;

Alterar o processamento dos programas que tratarão o dataset - Menu: LOGPRO:

Para dataset tratado em módulos já cadastrados do programa:

Alterar os módulos que utilizarão o dataset - Menu: CADMOD;

Para dataset tratado em módulos não cadastrados do programa:

Selecionar os módulos que tratarão o dataset - Menu: SELMOD;

Incluir os novos módulos de processamento para tratamento do dataset - Menu: CADMOD;

Gerar os fontes dos programas - Menu: GERPRO.

Para programas não gerados totalmente pelo SADS:

Movimentar para usercode de análise os fontes dos programas - Menu: COPFONMAN;

Efetuar as manutenções necessárias nos programs que irão tratar o dataset. Esta atividade deve ser efetuada através do editor de programas (Cande).

3 - Compilar os programas para teste - Menu: COMPRO;

4 - Testar os programas;

5 - Liberar para produção:

Compilar os programs para produção:

Verificar padrões de codificação dos programas - Menu: CONPADCOD;

Atualizar recursos de fontes de programas no repositório de dados - Menu: ATURECSDD;

Liberar fontes dos programas para biblioteca de fontes compactados - Menu: LIBFON;

Para ambientes de teste e produção em máquinas diferentes:

Copiar objetos dos programas do usercode de produção de uma máquina para outra.

Inclusão de um novo programa on-line para um host já existente

Este item contém as atividades necessárias à inclusão de um novo programa on-line em um sistema e este programa será relacionado a um host e a um menu já existentes.

Pré-requisitos

É necessário que o host ao qual o programa será relacionado e o menu que possuirá a opção que será tratada pelo programa já tenham sido cadastrados no repositório de dados.

Atividades

1 - Incluir opção para o programa na definição do menu a partir do qual ele será disparado - Menu: MEN;

2 - Gerar o host relacionado ao menu - Menu: GERHOS;

3 - Alterar desenho da tela - Menu de opção:

Se não houver necessidade de incluir novos argumentos no menu de opção:

Alterar desenho da tela do menu de opção, incluindo a opção correspondente - Menu: DESTEL;

Maiores detalhes sobre alterações em menus de opção podem ser consultados no item "Alteração de estrutura de menu de opção", nesta seção.

Gerar a library do menu para teste - Menu: GERTEL;

Liberar desenho da tela menu de opção para teste - Menu: LIBDESTEL.

Se houver necessidade de incluir novos argumentos no menu de opção:

Executar as atividades do item "Alteração de estrutura de menu de opção/operação" e as atividades descritas a seguir, que se referem ao novo programa.

4 - Compilar o host para teste - Menu: COMHOS;

5 - Efetuar o cadastramento da tela/relatório referente ao programa:

Para programa de emissão de relatório:

Definir o relatório - Menu: REL.

Maiores informações sobre definição de relatório podem ser consultados na Seção II, "Elementos do repositório de dados do SADS", no item "Relatório", deste manual.

Para programa que não emite relatório:

Definir a tela do programa - Menu: TEL.

Maiores informações sobre o cadastramento de telas podem ser consultadas na Seção II, "Elementos do repositório de dados do SADS", no item "Tela", deste manual.

6 - Fazer o cadastramento do programa - Menu: PRO;

Maiores informações sobre o cadastramento de programas podem ser consultados na Seção II, "Elementos do repositório de dados do SADS", no item "Programa", deste manual.

7 - Gerar o programa - Menu: GERPRO;

8 - Compilar o programa para teste - Menu: COMPRO;

9 - Fazer o binder do programa - Menu: BINPRO - (se o host foi recém-compilado, o binder será feito para todos os programas do host);

10 - Testar o programa;

11 - Para liberar o novo programa para produção:

Liberar o desenho da tela menu para produção - Menu: LIBDESTEL;

Liberar a library da tela menu para produção - Menu: LIBLIBTEL;

Compilar o host para produção - Menu: COMHOS;

Fazer as liberações relativas a tela/relatório do programa:

Para programa de emissão de relatório:

Liberar a library do relatório para produção - Menu: LIBREL.

Para programa que não emite relatório:

Liberar o desenho da tela para produção - Menu: LIBDESTEL;

Liberar a library da tela para produção - Menu: LIBLIBTEL.

Compilar o novo programa para produção - Menu: COMPRO;

Fazer o binder do programa - Menu: BINPRO - (como o host foi recém-compilado, o binder será feito para todos os programas do host);

Liberar o host e o programa para produção:

Verificar padrões de codificação (somente do programa) - Menu: CONPADCOD;

Atualizar recursos do programa no repositório de dados - Menu: ATURECSDD;

Liberar fontes do host e do novo programa para biblioteca de fontes compactados - Menu: LIBFON;

Para ambientes de teste e produção em máquinas diferentes:

Copiar objeto do host do usercode de produção de uma máquina para outra.

Desabilitar/habilitar o host no ambiente de produção. Esta operação pode ser feita através dos comandos "?Disable program" e "?Enable program". Estes comandos só podem ser executados por usercodes autorizados.

Inclusão de um novo menu de opção em um host em funcionamento

Este item contém as atividades necessárias à inclusão de um novo menu em um host que já esteja em funcionamento e, inclusive, apresenta as atividades de cadastramento de telas e programas relacionados às transações que serão disparadas a partir do menu. Apresenta, também, atividades de cadastramento de navegação interpretativa (padrão SAA) para o novo menu.

Pré-requisitos

Para a definição de um menu, é necessário que o host a que ele será relacionado já esteja cadastrado no repositório de dados e que já se tenha determinado o código dos programas que tratarão cada opção do menu.

Atividades

1 - Definir o menu no repositório de dados - Menu: MEN;

2 - Desenhar e cadastrar a tela - Menu: TEL:

Maiores informações sobre cadastramento de tela podem ser consultadas na Seção II, "Elementos do repositório de dados do SADS", no item "Tela", deste manual.

Para telas de atualização:

Gerar ou cadastrar estrutura de consistência da tela - Menu: GERCONTEL ou ESTCONTEL;

Cadastrar validações em nível de estrutura de consistência (se for o caso) - Menu: VALESTTEL;

Liberar consistência de tela para teste - Menu: LIBCONTEL;

Simular consistência da tela, caso necessário - Menu: SIMCONTEL.

3 - Cadastrar os elementos que ainda não estão cadastrados no repositório de dados e que serão utilizados pelo programa:

Relatórios - Menu: REL;

Datasets (executar atividades do item "Inclusão de um novo dataset na base de dados on-line do sistema", até a atividade "Gerar library de declaração de banco para teste", inclusive);

Registros de dataset de uso geral - Menu: USO;

Áreas de trabalho - Menu: ARE;

Sub-rotinas - Menu: SUB.

4 - Construir os programas definidos no novo menu:

Definir programa - Menu: a partir do DEFPRO;

Gerar fonte de programa - Menu: GERPRO;

Compilar programa para teste - Menu: COMPRO.

5 - Gerar host associado ao novo menu - Menu: GERHOS;

6 - Compilar o host para teste - Menu: COMHOS;

7 - Executar binder do host para teste - Menu: BINHOS - (como o host foi recém-compilado, o binder será feito para todos os programas do host);

8 - Cadastrar o trancode referente ao novo menu no COMS (ambiente de teste):

Gerar as especificações do COMS para o host (ambiente de teste) - Menu GERCOMHOS;

Executar a carga do arquivo gerado no Utility do COMS através do comando LOAD.

9 - Desabilitar/habilitar o host no ambiente de teste através dos comandos: "?Disable program <<sssnn >>" e "?Enable program <<sssnn>>" (somente usercodes habilitados podem fazê-lo);

10 - Testar programas;

11- Para definir uma navegação interpretativa (padrão SAA) para o novo menu:

Para opções do novo menu que não forem tratadas em contexto já existente:

Incluir contexto - Menu: CONNAV.

Para os menus de escolha:

Desenhar tela - Menu: DESTEL;

Definir tela - Menu: DEFTEL;

Liberar desenho da tela para teste - Menu: LIBDESTEL;

Cadastrar título da tela - Menu: TITNAV;

Cadastrar rotas de navegação - Menu: ROTNAV.

Para os menus de operação:

Desenhar tela - Menu: DESTEL;

Definir tela - Menu: DEFTEL;

Cadastrar estrutura da tela - Menu: ESTTEL;

Liberar desenho da tela para teste - Menu: LIBDESTEL.

Cadastrar título da tela - Menu: TITNAV;

Cadastrar regras de conversão de layout - Menu: REGNAV;

Liberar regras de conversão de layout para teste - Menu: LIBNAV.

Para cadastrar consistência para o menu, se necessário:

Gerar a estrutura da consistência - Menu: GERCONTEL;

Cadastrar validações em nível de estrutura - Menu: VALESTTEL;

Liberar consistência para teste - Menu: LIBCONTEL;

Simular consistência - Menu: SIMCONTEL.

Testar navegação;

12 - Para liberar para produção:

Liberar novas telas (do menu e de processamento) para produção:

Liberar desenho para produção - Menu: LIBDESTEL;

Liberar library para produção - Menu: LIBLIBTEL.

Liberar/gerar libraries dos demais elementos utilizados pelos programas para produção:

Relatórios - Menu: LIBREL;

Registros de dataset de uso geral - Menu: LIBUSO;

Áreas de trabalho - Menu: LIBARE;

Sub-rotinas - Menu: LIBLIBSUB.

Liberar os novos programas para produção:

Verificar padrões de codificação dos programas (exceto o host) - Menu: CONPADCOD;

Atualizar recursos do programa no repositório de dados - Menu: ATURECSDD;

Compilar programas para produção - Menu: COMPRO;

Liberar (compactar) fontes dos programas - Menu: LIBFON.

Liberar a navegação interpretativa, caso o sistema a utilize (padrão SAA):

Para os menus de escolha:

Liberar desenho da tela para produção - Menu: LIBDESTEL.

Para os menus de operação:

Liberar desenho da tela para produção - Menu: LIBDESTEL;

Liberar regras de conversão de layout para produção - Menu: LIBNAV.

Liberar o host para produção:

Compilá-lo para produção - Menu: COMHOS;

Executar binder para produção - Menu: BINHOS;

Liberar (compactar) fonte do host - Menu: LIBFON;

Para ambientes de teste e produção em máquinas diferentes:

Copiar objeto do host do usercode de produção de uma máquina para outra;

Extrair telas e consistências liberadas para produção - Menu: TRATEL;

Copiar de uma máquina para outra os arquivos contendo telas e consistências liberadas para produção;

Carregar telas e consistências na máquina de produção - Menu: TRATEL;

Se o sistema utilizar navegação interpretativa:

    • Extrair contexto de navegação - Menu: TRACON;
    • Copiar arquivos de uma máquina para outra;
    • Carregar contexto na máquina de produção - Menu: TRACON.

Alteração de estrutura de menu de opção

Este item contém as atividades necessárias à alteração de estrutura de menus de opção que estejam em funcionamento. São apresentadas, também, as atividades que devem ser executadas para alterar os menus de operação relacionados aos de opção. Ou seja, auxilia na inclusão de novos atributos e campos, bem como na exclusão e alteração de atributos e campos já existentes na estrutura do menu. As atividades apresentadas neste item, para menus de opção, são válidas também para submenus de opção. Neste item, a palavra atributo está sendo utilizada com um sentido mais amplo, referindo também a campos e sinônimos.

Pré-requisitos

Para a alteração da estrutura de menus, é necessário que os atributos que serão acrescentados às suas estruturas já estejam cadastrados no repositório de dados do SADS.

Atividades

1 - Alterar desenho da tela - Menu: DESTEL;

2 - Alterar estrutura da tela - Menu: ESTTEL;

3 - Gerar library da tela - Menu: GERTEL;

4 - Liberar desenho da tela para teste - Menu: LIBDESTEL;

5 - Alterar consistência geral:

Alterar estrutura da consistência - Menu: ESTCONTEL;

Alterar validações em nível de estrutura de consistência, se necessário - Menu: VALESTTEL;

Liberar consistência para teste - Menu: LIBCONTEL;

Simular consistência - Menu: SIMCONTEL.

6 - Adaptar os programas que tratam os novos atributos do menu:

Para programas totalmente gerados pelo SADS:

Fazer as alterações necessárias na definição do programa - Menu: PRO;

Gerar novamente o fonte - Menu: GERPRO.

Para programas não gerados totalmente pelo SADS:

Descompactar fonte do programa no usercode de análise - Menu: COPFONMAN;

Executar a manutenção, caso o programa tenha que tratar os novos campos do menu de opção.

7 - Compilar os programas para teste - Menu: COMPRO;

8 - Gerar novamente o host ou descompactar o host associado ao menu para usercode de análise - Menus: GERHOS ou COPFONMAN, respectivamente;

9 - Compilar o host para teste - Menu: COMHOS;

10 - Executar binder do host para teste - Menu: BINHOS;

11 - Desabilitar/habilitar o host no ambiente de teste - Pode ser feito através dos comandos "?Disable program <<sssnn >>" e "?Enable program <<sssnn>>" que somente podem ser executados por usercodes autorizados;

12 - Testar programas;

13 - Alterar menus de operação relacionados ao menu de opção:

Alterar desenho da tela - Menu: DESTEL;

Alterar estrutura da tela - Menu: ESTTEL;

Liberar desenho da tela para teste - Menu: LIBDESTEL;

Alterar regras de conversão de layout - Menu: REGNAV;

Liberar regras de conversão de layout para teste - Menu: LIBNAV.

14 - Alterar consistência geral para os menus de operação, caso seja necessário:

Alterar estrutura da consistência - Menu: ESTCONTEL;

Alterar validações em nível de estrutura de consistência, se necessário - Menu: VALESTTEL;

Liberar consistência para teste - Menu: LIBCONTEL;

Simular consistência - Menu: SIMCONTEL.

15 - Testar menus de operação alterados;

16 - Para liberar para produção:

Liberar o menu de opção alterado:

Liberar desenho da tela para produção - Menu: LIBDESTEL;

Liberar library da tela para produção - Menu: LIBLIBTEL.

Liberar os menus de operação alterados:

Liberar desenho da tela para produção - Menu: LIBDESTEL;

Liberar regras de conversão de layout para produção - Menu: LIBNAV.

Liberar programas relacionados ao menu de opção:

Compilá-los para produção - Menu: COMPRO;

Executar binder para produção - Menu: BINHOS;

Liberar fontes para biblioteca de fontes compactados - Menu: LIBFON.

Liberar objeto do host para usercode de objetos em produção - Menu: LIBOBJHOS.

Para ambientes de teste e de produção em máquinas diferentes:

Copiar de uma máquina para outra o objeto do host do usercode de produção;

Extrair telas, consistências e regras de conversão de layouts liberadas para produção - Menu: TRATEL;

Copiar de uma máquina para outra os arquivos extraídos com os elementos para produção;

Carregar os elementos na máquina de produção - Menu: TRATEL.

Observação: Caso a alteração seja executada em um menu de host secundário, deve-se executar estas mesmas atividades para o menu de opção do host primário que for relacionado ao menu secundário alterado.

Conversão de sistema para utilização de sub-rotinas globais

Neste item são apresentadas as atividades que devem ser executadas a fim de que um sistema antigo, que utiliza chamada convencional de sub-rotinas, possa chamá-las de acordo com o mecanismo de sub-rotinas globais disponibilizado pelo SADS.

A conversão destes sistemas para utilização de sub-rotinas globais pode ser feita de duas formas distintas, a saber:

Caso os programas sejam gerados completamente pelo SADS, pode-se alterar o campo indicador de utilização de sub-rotinas globais para "S" (menu "DEFSIS") e gerar novamente todos os programas do sistema, incluindo os hosts.

Neste caso, as atividades a serem efetuadas são as mesmas processadas para um sistema novo. Estas atividades estão descritas neste manual, na Seção II, "Elementos do repositório de dados do SADS", item "Sub-rotina", tópico "Criação de novos sistemas utilizando sub-rotinas globais".

Porém, esta forma de conversão possui o seguinte inconveniente: dado que o sistema é antigo, ele já deve estar em produção, efetuando, neste ambiente, as chamadas de sub-rotinas na forma convencional. Caso seja necessária uma manutenção no programa em ambiente de produção, o fonte recém-gerado não pode ser compilado para produção sem antes sofrer alterações, pois ele já possui as chamadas de sub-rotinas como globais, que é incompatível com o tratamento de sub-rotinas pelo sistema naquele ambiente.

Portanto, esta forma de conversão do sistema para utilização de sub-rotinas globais pode gerar problemas de incompatibilidade de programas nos dois ambientes.

A outra forma de conversão é através da execução de um Programa Conversor, que propicia a migração gradativa do sistema, sem o inconveniente da incompatibilidade de programas nos dois ambientes - teste e produção. Por este método de conversão, as duas possibilidades de utilização de sub-rotinas são incluídas nos fontes dos programas, através de diretivas de compilação, propiciando as seguintes situações: caso os programas sejam compilados para teste, as chamadas das sub-rotinas serão feitas pelo mecanismo de sub-rotinas globais; caso um mesmo programa seja compilado para produção, sem nenhuma alteração em seu fonte, as chamadas das sub-rotinas serão feitas de forma convencional.

Todas as atividades descritas a seguir são referentes a esta forma de conversão dos sistemas antigos - através da execução do Programa conversor - para utilização de sub-rotinas globais.

Pré-requisitos

A conversão de sistemas antigos para utilização de sub-rotinas globais somente será possível caso esteja parametrizado que a instalação pode fazer uso desta implementação. Esta parametrização é feita na tabela SBPM5000, CODIGO = "SUBGBL", PAR-3 = "L" (livre) ou "O" (obrigatório).

Atividades

1 - Certificar se o campo indicador de utilização de sub-rotinas globais pelo sistema - UTILIZACAO SUB-ROTINAS GLOBAIS (S/N) - é igual a "N" ou " " - Menu: DEFSIS;

2 - Disponibilizar todos os fontes dos programas on-line do sistema no usercode onde será executado o job de conversão:

Se os programas forem gerados completamente pelo SADS:

Gerar novamente seus fontes no usercode escolhido - Menu: GERPRO;

Se os programas não forem gerados completamente pelo SADS:

Descompactar seus fontes no usercode escolhido - Menu: COPFONMAN.

3 - Executar o programa de conversão - RUN SDR07, informando o código do sistema a ser convertido e se é desejável a conversão de suas libraries de procedure;

4 - Compilar e efetuar o binder do sistema para o ambiente de teste - Menus: COMPRO e BINPRO;

5 - Alterar o campo indicador de utilização de sub-rotinas globais pelo sistema - UTILIZACAO SUB-ROTINAS GLOBAIS (S/N) para "S", após testado o sistema - Menu: DEFSIS;

6 - Liberar cada uma das libraries de declaração de sub-rotinas do aplicativo para o ambiente de produção - Menu: LIBLIBGBL;

7 - Liberar para a produção todas as libraries de procedure alteradas pelo filtro conversor - Menu: LIBLIBPRO;

8 - Compilar e efetuar o binder do sistema para o ambiente de produção - Menus: COMPRO e BINPRO;

9 - Excluir, caso desejável, as linhas de chamadas convencionais de sub-rotinas dos fontes dos programas convertidos, através da execução do programa SDR08 - RUN SDR08.
