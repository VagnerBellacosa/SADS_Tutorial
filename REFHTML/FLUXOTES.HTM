Fluxo de testes

Existem basicamente dois tipos de testes efetuados pelo Sistema de Segurança do SADS para garantir a segurança no acesso aos dados da instalação: aqueles efetuados somente no momento do logon e os rotineiros, efetuados sempre que uma transação é requisitada.

No momento do logon, o Sistema de Segurança obtém os dados do usuário e valida a possibilidade deste iniciar uma sessão de trabalho no SADS. Os testes efetuados para essa validação são relacionados na Seção I, "Requisitos e conceitos para utilização do SADS", item "Procedimentos de habilitação".

Os testes rotineiros determinam se o usuário, que já se encontra logado, pode ou não ter acesso ao aplicativo e à transação requisitados, se o horário para acesso ao aplicativo lhe é permitido, se a transação está disponível para utilização e se o usuário pode trabalhar nesta transação com os valores dos campos cadastrados como elementos no SADS/SS.

Nos testes de segurança considerados rotineiros, sempre que um usuário solicita uma transação, o host do aplicativo verifica:

Se já foi efetuado o logon na estação (em caso negativo, ele será solicitado);

E, em caso positivo, se o usuário que efetuou o logon está habilitado a trabalhar com o aplicativo e a transação solicitados.

Satisfeitos estes testes, o host verifica se a transação requisitada está habilitada ou possui limitação de horário e se o usuário não possui limitação de horário para trabalhar com o aplicativo específico.

Uma vez efetuados satisfatoriamente todos estes testes, finalmente o host dispara a transação em questão, entregando o processamento para o programa chamado responsável pela transação disparada.

As validações de habilitação de usuário ao aplicativo e transação são, portanto, requisitadas pelo host deste e efetuadas pelas sub-rotinas de segurança. Sendo assim, os hosts dos aplicativos já são gerados pelo SADS com chamadas a estas sub-rotinas.

O Sistema de Segurança faz a validação da habilitação de aplicativos segundo os códigos de transação das telas de processamento ou de menus 01 a 09.

Para o SADS/SS, portanto, os códigos de transação (sssmmnn) dos menus de escolha ou de operação não podem ser utilizados para definir as habilitações dos usuários. Porém, apenas os códigos de menu (sssmm) dos menus de escolha e de operação devem ser habilitados, para que eles possam utilizar a navegação.

Já a validação de habilitação de elementos deve ser feita nos programas chamados dos aplicativos ou nas consistências de suas telas, uma vez que a sua necessidade é definida pelo analista responsável pelo sistema. Este deverá incluir nos seus programas, quando lhe convier, a chamada à sub-rotina SBTS03, que é a encarregada pela validação de elementos.

Maiores detalhes sobre habilitação de elementos são apresentados nesta seção, no item "Níveis de segurança disponíveis", tópico "Validação de elementos".

Existe também a validação de habilitação de programa por sub-rotina, que, da mesma forma que a habilitação de elemento, deve ser feita nas sub-rotinas dos aplicativos, uma vez que a sua necessidade também é definida pelo analista responsável pelo aplicativo.

Maiores detalhes sobre habilitação de programas para acesso a sub-rotina estão descritos nesta seção, item " Níveis de segurança disponíveis", tópico "Validação de programa para acesso a sub-rotina".
