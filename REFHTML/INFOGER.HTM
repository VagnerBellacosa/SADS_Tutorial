Informações gerenciais

O SADS possui um repositório de dados ativo, ou seja, que se mantém atualizado em qualquer fase do desenvolvimento dos sistemas.

Desta forma, para auxiliar no controle do desenvolvimento desses sistemas, o SADS oferece uma grande variedade de consultas gerenciais, quais sejam:

Programas em desenvolvimento;

Programas em produção;

Programas faltando consistir padrões de codificação;

Programas faltando atualizar recursos;

Estatísticas de programas;

Estatísticas do repositório de dados;

Situação do desenvolvimento de sistema;

Lista de sistemas;

Sub-rotinas da instalação.

Programas em desenvolvimento

Esta consulta é utilizada a fim de obter a relação dos programas em desenvolvimento, sua data de cadastramento e o número de compilações.

Programas em desenvolvimento são aqueles que ainda não foram compilados para a produção.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados".

Argumento da consulta

Para que esta consulta seja utilizada, o seguinte argumento deve ser informado no menu de operação PRODES:

 Sistema - Código do sistema dos programas a serem apresentados, no padrão SADS (sss).

Informações exibidas na consulta

Serão informados para cada um dos programas em desenvolvimento:

 PROGRAMA - Código do programa.

 NOME - Nome do programa.

 CADASTRAMENTO - Data de cadastramento do programa no repositório de dados.

 COMPILACOES - Número de compilações para o ambiente de teste, efetuadas desde o seu cadastramento.

Programas em produção

Esta consulta é utilizada a fim de obter a relação dos programas que estão em produção, a data de liberação do fonte do programa e o número de compilações para os ambientes de teste e de produção.

Programas em produção são aqueles que foram compilados para a produção pelo menos uma vez.

A data de liberação informada nesta consulta é a última data em que ocorreu a liberação do fonte do programa para o ambiente de fontes compactados. Para maiores detalhes sobre a liberação para a produção, deve-se consultar, nesta seção, o item "Liberação de fonte" ou "Liberação de família de fonte".

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados".

Argumentos da consulta

Para que esta consulta seja utilizada, alguns argumentos devem ser informados no menu de operação PROLIBPRO, quais sejam:

 Sistema - Código do sistema dos programas a serem apresentados, no padrão SADS (sss).

 Data - Data de liberação dos programas para os ambiente de fontes compactados (opcional).

Esta data, no formato DDMMAA (dia, mês e ano), determinará o início da pesquisa.

Informações exibidas na consulta

Serão informados para cada um dos programas em produção:

 PROGRAMA - Código do programa.

 NOME - Nome do programa.

 LIBERACAO - Data da última liberação do fonte do programa para o ambiente de fontes compactados.

 COMPILACOES:

 DES - Número de compilações para o ambiente de desenvolvimento;

 PROD - Número de compilações para o ambiente de produção.

Estatísticas de programas

Esta consulta é utilizada a fim de obter as estatísticas acumuladas dos programas cadastrados no ambiente SADS.

Argumento da consulta

Para que esta consulta seja utilizada,nenhum argumento precisa ser informado no menu de operação ESTPRO, basta transmitir este menu para confirmação da consulta.

Informações exibidas na consulta

 PROGRAMAS EM DESENVOLVIMENTO - Número de programas que ainda não foram compilados para o ambiente de produção.

 PROGRAMAS EM PRODUCAO - Número de programas que já foram compilados para o ambiente de produção.

 TOTAL - Número total de programas cadastrados no SADS.

 VERIFICADOS (PADROES) - Número de programas que estão com os padrões de codificação consistidos.

 RECURSOS ATUALIZADOS (DD) - Número de programas que estão com o cadastro dos recursos atualizados no dicionário de dados do SADS.

 MEDIA DE COMPILACOES:

 DESENVOLVIMENTO - Média de compilações no ambiente de desenvolvimento;

 PRODUCAO - Média de compilação no ambiente de produção.

Estatísticas do repositório de dados

Esta consulta é utilizada a fim de obter as estatísticas do repositório de dados por sistema, por tipo de sistema ou em todo o ambiente SADS.

Nestas estatísticas são informadas as quantidades de cada elemento armazenadas no repositório de dados do SADS e o percentual da sua evolução nos últimos três meses, tomando como base o mês informado como argumento da consulta.

A estatística do repositório de dados apresenta um indicador de ambiente para cada elemento, de acordo com o seguinte padrão:

 Se ao lado esquerdo do elemento aparecer a letra "D", o elemento existe apenas no ambiente de desenvolvimento; ou

 Se aparecer a letra "P", o elemento também existe em produção.

No rodapé de cada página da consulta é apresentada uma legenda para facilitar a compreensão deste indicador.

Através do menu ROTAPO - operação "ATU" - ou da execução do job SD/D/JB/45, podem ser extraídas as quantidades de cada elemento cadastradas nos datasets do SADS e armazenadas estas informações no mês corrente. Portando, para obter os dados atualizados desta estatística, deve-se executar, ao final de cada mês, a operação "ATU" do menu ROTAPO ou o job SD/D/JB/45.

Uma explicação do job SD/D/JB/45 está descrita a seguir:

Nome Externo: SD/D/JB/45;

Descrição: Armazena a situação do repositório de dados no mês corrente;

Periodicidade: Mensal;

Entradas: Todos os datasets do SADS/DD;

Saídas: SBGB02 (Dataset de Uso Geral);

Recuperação: No caso de falha, este job pode ser disparado novamente;

Parâmetros : Primeira posição em branco e, a seguir, a data no formato AAMM, se esta for diferente da data corrente.

Retenção arq.: Não há.

A tela de processamento apresentada é do tipo consulta paginada e está descrita, na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados".

Argumentos da consulta

Para que esta consulta seja utilizada, alguns argumentos devem ser informados no menu de operação ESTREP, quais sejam:

 Sistema ou Classificacao - Argumento para selecionar os sistemas (opcional).

Pode assumir os seguintes valores:

 "sss" - Para consultar a estatística de um sistema específico;

 "<<Classificação quanto ao uso>>" - Para consultar a estatística de todos os sistemas de
uma determinada classificação quanto ao uso.

As classificações quanto ao uso possíveis são as seguintes:

 "E" - Sistemas de uso geral da empresa;

 "I" - Sistemas de uso da área de informática;

 "T" - Sistemas de uso da área de treinamento;

 "D" - Sistemas para demonstração;

 "S" - Sistemas cuja sigla serve apenas para controle;

 "H" - Sistemas de hierarquia funcional;

 "C" - Sistemas sem classificação.

 Periodo - Período de referência da consulta no formato AAMM (ano e mês) (opcional).

Serão exibidas as estatísticas dos três meses anteriores ao mês informado no campo Periodo, incluindo o mês informado.

Caso o período não seja informado, serão exibidas as estatísticas dos três meses anteriores ao mês corrente, incluindo o mês corrente.

Informações exibidas na consulta

Serão apresentados todos os tipos de elementos cadastrados no SADS para os sistemas selecionados, as quantidade de cada um e o percentual da evolução dos sistemas mês a mês.

Situação do desenvolvimento de sistema

Esta consulta é utilizada a fim de obter a situação do desenvolvimento de um sistema.

Através desta consulta, serão apresentados os códigos dos elementos, os nomes e as situações dos cadastros destes elementos.

Nesta consulta só serão informados os elementos com algum tipo de irregularidade em seu cadastro.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados".

Argumento da consulta

Para que esta consulta seja utilizada, no menu de operação SIT deve ser informado o seguinte argumento:

 Sistema - Código do sistema a ser consultado, no padrão SADS (sss).

Informações exibidas na consulta

 SISTEMA - Código e nome do sistema a ser consultado.

 Serão informados para cada um dos elementos a ser apresentado:

 <<elemento>> - Código do elemento.

 NOME - Nome do elemento.

 SITUACAO - Descrição da situação do cadastro do elemento.

Lista de sistemas

Esta consulta é utilizada a fim de obter a lista de sistemas cadastrados no SADS.

Esta lista de sistemas pode ser selecionada conforme a situação, classificação quanto ao desenvolvimento e classificação quanto ao uso.

Estas classificações estão detalhadamente descritas na Seção II, "Elementos do repositório de dados do SADS", no item "Sistema".

Argumentos da consulta

Para que esta consulta seja utilizada, alguns argumentos devem ser informados no menu de operação LISSIS, quais sejam:

 Situacao do sistema - Situação do sistema (opcional).

Pode assumir os seguintes valores:

 "D" - Sistemas em desenvolvimento;

 "P" - Sistemas em produção.

Caso este campo não seja informado, serão selecionados os sistemas cadastrados em desenvolvimento e em produção.

 Classificação:

 Quanto ao Desenvolvimento - Classificação dos sistemas a serem consultados quanto
ao desenvolvimento (opcional).

Pode assumir os seguintes valores:

"PP" - Sistemas próprios padronizados;

"PN" - Sistemas próprios não padronizados;

"TP" - Sistemas de terceiros padronizados;

"TN" - Sistemas de terceiros não padronizados.

Caso este campo não seja informado, não será considerada a classificação quanto ao desenvolvimento no momento de se selecionar os sistemas a serem apresentados.

Quanto ao uso - Classificação quanto ao uso dos sistemas a serem consultados (opcional).

Pode assumir os seguintes valores:

"E" - Sistemas de uso geral da empresa;

"I" - Sistemas de uso área de informática;

"T" - Sistemas de uso do treinamento;

"D" - Sistemas para demonstração;

"S" - Sistemas cuja sigla serve apenas para controle;

"H" - Sistemas de hierarquia funcional.

Caso este campo não seja informado, não será considerada a classificação quanto ao uso no momento de se selecionar os sistemas a serem apresentados.

Informações exibidas na consulta

Serão informados para cada um dos sistemas a serem apresentados:

 SIS - Código do sistema.

 NOME - Nome do sistema.

 CLASSIFICACAO:

 Classificação do sistema quanto à situação;

 Classificação do sistema quanto ao desenvolvimento;

 Classificação do sistema quanto ao uso.

Sub-rotinas da instalação

Esta consulta é utilizada para se obter a relação das sub-rotinas cadastradas no repositório de dados do SADS.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados".

Argumento da consulta

Para que esta consulta seja utilizada, um argumento deve ser informado no menu de operação SUBINS, qual seja:

 Sub-rotina - Código da sub-rotina, no padrão SADS, a partir da qual serão apresentadas as sub-rotinas cadastradas no repositório de dados do SADS, ou brancos para obter a relação completa.

Informações exibidas na consulta

Serão informados para cada uma das sub-rotinas:

 Codigo - Código da sub-rotina.

 Nome - Nome da sub-rotina.
