Arquivo

Conceituação

Arquivo é um local onde são armazenadas, geralmente de forma temporária, algumas informações que serão processadas pelo sistema.

Estas informações são armazenadas em registros que já possuem uma estrutura pré-definida. Cada registro representa um conjunto de dados através dos quais os programas adquirem ou gravam informações no arquivo. Para o SADS, um mesmo arquivo pode ter registros com estrutura de dados diferentes, porém com o mesmo tamanho.

Para definir completamente um arquivo, é necessário efetuar a definição de suas especificações e o cadastramento das estruturas de seus registros.

A partir das informações cadastradas para o arquivo é gerada uma library para ser incorporada aos programas que fizerem acesso a ele ou que o atualizarem. No momento da geração da library do arquivo é efetuada a consistência contra o cadastramento do mesmo.

Existe um tipo especial de arquivo, os port files, que podem ser definidos no SADS para efetivar a comunicação entre programas.

Os port files são arquivos lógicos e podem ser utilizados de diversas maneiras, a saber:

 Em programa batch: deve-se declarar o arquivo port como recurso do programa;

 Em programa host: deve-se declarar o arquivo port na definição do host;

 Em programa chamado: deve-se declarar o arquivo port como recurso do programa;

 Simultaneamente, no programa chamado e no host: deve-se declarar o arquivo tanto no host quanto no programa. Desta forma, as áreas reservadas para suas declarações no programa chamado serão geradas como globais.

A declaração de um port file, em programas e hosts, resulta na geração de comandos copy das libraries do arquivo. O seu tratamento deve ser implementado através de módulos ou de libraries de procedure pelo próprio analista responsável.

Detalhes a respeito da utilização de port files estão descritos no "I/O SUBSYSTEM MANUAL", da UNISYS.

Padronização

O código padrão adotado pelo SADS para identificação do arquivo é a seguinte:

"sssAnn" onde

"sss" é o código afabético que identifica o sistema;

"A" é a constante de arquivo e

"xn" é o sequencial acima de 50 ou variável de A1 a Z9.

Utilização

Os arquivos são utilizados para armazenar informações, normalmente temporárias, que serão processadas pelo sistema.

Exemplos:

 Arquivo com dados extraídos de um dataset, para posterior emissão de relatório;

 Arquivo com dados a serem carregados em um dataset, para integração de sistemas.

Apesar dos arquivos serem definidos em um único sistema, eles podem funcionar como um elemento de integração entre dois ou mais sistemas.

No SADS, o arquivo pode ser utilizado como recurso de programas on-line, batch e sub-rotinas, e também na definição de hosts.

Cadastramento

Para cadastrar um arquivo, é necessário que o seu sistema já esteja cadastrado no repositório de dados do SADS.

Roteiro

Um arquivo poderá ser criado a partir de um dataset ou a partir de uma entidade através das seguintes funções:

 Criação de arquivo a partir de dataset;

 Criação de arquivo a partir de entidade;

Para alterações do cadastramento gerado ou para a inclusão de um arquivo diretamente no repositório de dados, temos as seguintes funções:

 Definição de arquivo;

 Cadastramento de estrutura de arquivo;

Para complementações do cadastramento, temos as seguintes funções:

 Geração da library de arquivo;

 Liberação da library de arquivo para produção.

Criação de arquivo a partir de dataset

O SADS permite criar um arquivo, suas especificações e as estruturas de seus registros, a partir de um dataset. Neste caso, o arquivo será a imagem desta estrutura DMSII e servirá, por exemplo, para armazenar seus dados temporariamente, para posterior operação de carga em outro dataset ou emissão de relatório.

Para criar um arquivo a partir de um dataset, é necessário que o dataset já esteja definido completamente no repositório de dados.

O tamanho do registro do arquivo é calculado a partir do tamanho dos campos contidos na estrutura do dataset.

A geração de arquivo a partir de dataset não está relacionada a nenhuma tela de processamento. Para efetuá-la, basta transmitir os argumentos do menu de operação "CRIARQDAT", cujos campos são explicados a seguir:

CAMPOS DA TELA

 Arquivo - Código do arquivo no padrão SADS (sssAxx).

 Dataset - Código do dataset no padrão SADS (sssBxx).

Criação de arquivo a partir de entidade

O SADS permite criar um arquivo, suas especificações e a estrutura de registro, a partir de uma entidade.

Para criar um arquivo a partir de uma entidade, é necessário que a entidade já esteja definida completamente no repositório de dados.

O tamanho do registro do arquivo é calculado a partir do tamanho dos campos contidos na estrutura da entidade.

A geração de arquivo a partir de entidade não está relacionada a nenhuma tela de processamento. Para efetuá-la, basta transmitir os argumentos do menu de operação "CRIARQENT", cujos campos são explicados a seguir:

CAMPOS DA TELA

 Entidade - Código da entidade no padrão SADS (sssExxx).

 Arquivo - Código do arquivo no padrão SADS (sssAxx).

Definição de arquivo

CAMPOS DA TELA

 ARQUIVO - Código do arquivo no padrão SADS (sssAxx).

 NOME - Descrição resumida do arquivo.

O nome tem o objetivo de identificar e explicar resumidamente a que se refere o arquivo.

 TIPOS DE REGISTROS - Número de tipos de registros diferentes a serem armazenados no arquivo.

Na library do arquivo, onde é gerada sua FD, será gerado um nível "01" para cada um dos tipos de registros cadastrados para o arquivo.

 TIPO DE ARQUIVO - Classificação do arquivo quanto ao tempo de retenção.

Os valores possíveis são os seguintes:

"W" - Para arquivo temporário (work);

"F" - Para arquivo fixo ou permanente.

 DISPOSITIVO FISICO - Dispositivo físico no qual residirá o arquivo.

Os valores possíveis são os seguintes:

"F" (Fita);

"D" (Disco);

"C" (Cartão parâmetro);

"P" (Port file).

 ORGANIZACAO - Parâmetro a partir do qual serão gerados os atributos ORGANIZATION e ACCESS MODE do arquivo.

Os valores possíveis são os seguintes:

"S" (Sequencial) - Arquivo passível somente de acessos sequenciais.

Para arquivos com ORGANIZACAO = "S" não serão gerados, na library, os atributos
ORGANIZATION e ACCESS MODE.

"I" (Indexado) - Arquivo para acesso indexado. O acesso sequencial também será
possível.

Para arquivos com ORGANIZACAO = "I", serão geradas, na library, as seguintes declarações :

ORGANIZATION IS INDEXED;

ACCESSMODE IS DYNAMIC;

RECORD KEY IS sxx01-atributo, onde:

"s" é o sub-sistema do arquivo;

"xx" é o identificador do arquivo;

"01" indica que a chave é um atributo do registro "01" do arquivo;

"atributo" é o mnemônico do atributo indicado em CAMPO CHAVE.

"D" (Direto) - Arquivo para acesso direto ou randômico. O acesso sequencial também
será possível.

Para arquivos com ORGANIZACAO = "D", serão geradas, na library, as seguintes
declarações :

ACCESS IS RANDOM;

ACTUAL KEY IS KEY-sssAxx, onde "ssAxx" é o código do arquivo.

KEYEDIOII (S/N) - Indicador de definição de arquivo indexado que utiliza o método de acesso KEYEDIOII.

Caso o arquivo possua este método de acesso, a sua library será gerada com as declarações exigidas pelas rotinas do referido método

Os valores possíveis são os seguintes:

"S" - Para definir a utilização do método de acesso KEYEDIOII para um arquivo indexado;

"N" - Para não definir o método de acesso KEYEDIOII para um arquivo indexado.

 CAMPO CHAVE - Código do atributo ou campo que será chave do registro.

Este atributo/campo não pode ser definido com sufixo na estrutura do arquivo.

Para arquivos com ORGANIZACAO = "I", deve-se informar o atributo do registro "01" do arquivo que será utilizado como chave. A partir deste campo, será gerada a cláusula RECORD KEY.

 CAMPO CHAVE SECUNDARIA - Código do atributo ou campo que será chave secundária do registro.

Informação válida para arquivos indexados.

Este atributo/campo não pode ser definido com sufixo na estrutura do arquivo.

 DUPLICATES - Indicador de chave secundária que permite duplicates.Os valores possíveis são os seguintes:

"S" - Para permitir duplicates;

"N" - Para não permitir duplicates.

 LIBRARIES - Indicador de quais libraries do arquivo devem ser geradas. Os valores possíveis são os seguintes:

"C" (Completa) - Para que sejam geradas todas as libraries do arquivo, quais sejam:

ss/s/AL/xx - A ser inserida na FD - File Description;

ss/s/AL/xxW - A ser inserida na Working-Storage Section; e

ss/s/AL/xxS - A ser inserida na SD - Sort Description - a fim de ser utilizada pelo
comando de SORT.

"W" (Work) - Para que sejam geradas as seguintes libraries do arquivo:

ss/s/AL/xx - A ser inserida na FD - File Description; e

ss/s/AL/xxW - A ser inserida na Working-Storage Section.

"S" (Sort) - Para que sejam geradas as seguintes libraries do arquivo:

ss/s/AL/xx - A ser inserida na FD - File Description, e

ss/s/AL/xxS - A ser inserida na SD - Sort Description - a fim de ser utilizada pelo
comando de SORT.

Caso este indicador não seja informado, será gerada apenas a library de FD - ss/s/AL/xx - para arquivos do tipo cartão ("C" no campo DISPOSITIVO FISICO).

A library de sort não será gerada, caso o arquivo não seja sequencial.

 GERA FILE STATUS - Indicador de tratamento do atributo filestatus.

Os valores possíveis são os seguintes:

"S" - Para geração do comando write com um display do filestatus, caso haja algum problema na gravação do arquivo;

"N" - Para não haver o tratamento do filestatus.

Se a definição do arquivo for alterada para tratar filestatus, a sua library deve ser gerada novamente para efetivar a alteração. O programa também deve ser gerado novamente, pois o campo filestatus pertence à área de work do programa.

 WORK SEM REDEFINICAO - Indicador de registro de área de work sem redefinição para o registro "00" e, também, sem o filler final que complementa o registro para o tamanho definido para o arquivo.

Arquivos já existentes, continuarão a ser gerados com a redefinição.

Os valores possíveis são os seguintes:

"S" - Para geração sem redefinição;

"N" - Para geração com redefinição.

 REGISTRO:

 TAMANHO - Tamanho máximo dos registros pertencentes ao arquivo.

 OBRIGATORIO - Indicador de que o tamanho informado no campo TAMANHO
tem de ser respeitado no momento da geração da library pelo SADS.

Os valores possíveis são os seguintes:

"S" - Para que a library seja gerada com o tamanho definido no campo
TAMANHO;

"N" - Para que, no momento da geração da library, seja calculado um tamanho
otimizado para o registro do arquivo.

 ATRIBUTOS:

 AREAS - Quantidade de áreas do arquivo.

 AREASIZE - Quantidade de registros em cada área.

 FATOR DE BLOCO - Fator de bloco do arquivo, ou seja, número de registros que irão compor um bloco de I/O.

Se o fator de bloco for informado, o gerador de library não fará o cálculo do fator ideal para o bloco. Portanto, ao informar um fator de bloco, deverá ser observado um número otimizado, para que não haja problemas de perfomance quando da leitura ou gravação do arquivo.

 POPULACAO / MAXSUBFILES - População estimada para o arquivo, ou seja, previsão da quantidade máxima de registros que o arquivo irá armazenar.

Esta informação servirá de base para o cálculo de um fator de bloco otimizado para o arquivo.

Deve-se optar entre informar POPULACAO ou AREAS e AREASIZE para um mesmo arquivo.

Para arquivo "P" ("Port File"), deve-se informar neste campo o número máximo de arquivos aos quais eles podem se conectar (MAXSUBFILES).

 INCORPORAR NOME EXTERNO ATRIBUTOS - Indicador de geração de library, considerando o código externo do atributo, conforme sua definição no repositório de dados.

Os valores possíveis são os seguintes:

"S" - Para geração de library utilizada pelos programas SADS e de uma library específica, que se diferencia da primeira através do sufixo "E", onde os atributos são declarados com o código externo;

"N" - Para geração apenas da library utilizada pelos programas SADS.

 OPERACAO - Operação a ser efetuada em relação à definição do arquivo.

Os valores possíveis são os seguintes:

"INC" (Inclusão);

"ALT" (Alteração);

"CON" (Consulta);

"EXC" (Exclusão).

Cadastramento de estrutura de arquivo

Estruturas de arquivos compreendem os atributos e campos que compõem seus registros e suas características dentro da estrutura.

Deverão ser especificadas a estrutura de cada um dos tipos de registro de um arquivo.

As estruturas dos registros são utilizadas na geração da library do arquivo, a ser incorporada em todos os programas que fizerem acesso a ele ou que o atualizarem.

Para efetuar o cadastramento da estrutura de um arquivo, é necessário que o mesmo tenha sido definido no repositório de dados do SADS.

No contexto de arquivo, entenda-se atributo como sendo atributo propriamente dito, campo ou sinônimo.

CAMPOS DA TELA

 ARQUIVO - Código do arquivo no padrão SADS (sssAxx).

 REGISTRO - Número do registro.

 NOME - Descrição resumida do registro.

O nome tem o objetivo de identificar e explicar resumidamente a que se refere o registro.

 SEQ - Número sequencial que indica qual a posição do atributo dentro da estrutura do registro do arquivo.

O primeiro número é "0010" e os demais, acrescidos de 10.

Para incluir um atributo entre dois outros já incluídos, é necessário preencher uma linha que esteja em branco e alterar o número sequencial desta linha para um número entre os já informados. O SADS reorganizará automaticamente a estrutura. Na próxima operação, os atributos já estarão na nova ordem.

 ATRIBUTO - Mnemônico do atributo pertencente à estrutura do registro do arquivo.

 SUF - Sufixo do atributo.

O sufixo é utilizado para diferenciar um mesmo atributo que aparece mais de uma vez na mesma estrutura.

Neste caso, para diferenciar suas ocorrências, deve-se cadastrar sufixos diferentes para cada um deles.

 NIVEL - Nível de estruturação dos atributos (análogo ao nível do Cobol).

O nível inicial é o 10. Se for necessário criar agrupamentos de atributos, deve-se usar múltiplos de 5 (15, 20 e assim por diante), de forma semelhante ao Cobol.

 NDEF - Nível de definição do atributo.

Os valores possíveis são os seguintes:

"S" (Sistema) - Para atributo definido no sistema informado no campo SIS ou pertencente ao mesmo sistema do arquivo;

"G" (Geral) - Para atributo definido no repositório de dados do SADS para utilização genérica nos sistemas aplicativos.

"R" (Redefinição) - Para atributo utilizado na redefinição de um item definido com sequência imediatamente anterior.

Atributos com NDEF = "R" têm obrigatoriamente de ser itens grupo.

Caso nada seja informado em NDEF, será assumido que o atributo é um item grupo.

SIS - Código do sistema no qual o atributo está definido.

Não é necessário informar este campo quando o atributo for genérico do SADS (NDEF = "G") ou estiver definido no mesmo sistema do arquivo em questão.

 COMP - Indicador de atributo compactado.

Os valores possíveis são os seguintes:

"S" - (Numérico compactado);

"B" - (Binary).

 FORM - Formato do atributo.

Os valores possíveis são os seguintes:

"A" (Alfabético);

"N" (Numérico);

"D" (Display - alfanumérico).

Se o formato não for informado, será assumida a definição de formato do atributo contida no repositório de dados.

Este campo não pode ser informado caso tenha sido parametrizado na tabela SBPM5000 que os atributos/campos não podem ter seus formatos redefinidos nas estruturas nas quais são incluídos.

Maiores detalhes a respeito desta parametrização, deve-se consultar a Seção V, "Parametrizações do SADS", item "Atributo", subitem "Redefinição em estruturas".

 TAM - Tamanho do atributo.

Se o tamanho não for informado, será assumido o tamanho definido para o atributo no repositório de dados.

Este campo não pode ser informado caso tenha sido parametrizado na tabela SBPM5000 que os atributos/campos não podem ter seus tamanhos redefinidos nas estruturas nas quais são incluídos.

Maiores detalhes a respeito desta parametrização, deve-se consultar a Seção V, "Parametrizações do SADS", item "Atributo", subitem "Redefinição em estruturas".

 DEC - Número de casas decimais para o atributo númerico.

Se não for informado, será assumida a quantidade de decimais definida para o atributo no repositório de dados. Se um atributo com decimais no repositório de dados tiver sua quantidade de decimais redefinidas para "00", o atributo será gerado sem decimais.

 SIN - Indicador de atributo numérico sinalizado.

Os valores possíveis são os seguintes:

"S" - Para atributo sinalizado;

"N" - Para atributo não sinalizado.

Se não for informado, será assumida a definição de sinal para o atributo contida no repositório de dados.

 OCCURS - Número de ocorrências do atributo dentro da estrutura.

O campo OCCURS é semelhante a cláusula "OCCURS" das estruturas do Cobol e é usado para definir tabelas e itens de grupo repetitivos.

 CONTINUA (S/N) - Indicador de continuidade de operação.

As possibilidades para o preenchimento deste campo são as seguintes:

"S" - Nova tela será aberta para prosseguir informando mais atributos para a estrutura do registro em cadastramento;

"N" - Para encerrar o cadastramento desta estrutura do arquivo.

 OPERACAO - Operação a ser efetuada em relação à definição do arquivo.

Os valores possíveis são os seguintes:

"INC" (Inclusão);

"ALT" (Alteração);

"CON" (Consulta);

"EXC" (Exclusão).

Geração de library de arquivo

A library é gerada a partir da estrutura de dados de todos os registros do arquivo. Portanto, para gerá-la, é necessário que o arquivo e as estruturas de seus registros estejam cadastrados no repositório de dados do SADS.

A library contém várias informações, sendo referenciada em pontos distintos do programa da seguinte forma:

O seguimento da sequência 020 até 900, onde está declarada a identificação do arquivo, e o OBJECT-COMPUTER, é chamado na CONFIGURATION SECTION da ENVIRONMENT DIVISION.

O seguimento da sequência 100000 até 199000, onde está declarado o SELECT do arquivo, é chamado no parágrafo FILE-CONTROL da ENVIRONMENT DIVISION.

O seguimento da sequência 200000 até 299000, onde está declarado a FD do arquivo, é chamado no parágrafo FILE SECTION da DATA DIVISION. Nesta parte da library, também está declarado o tamanho do registro.

O seguimento da sequência 300000 até 399000, onde estão declaradas as estruturas dos registros do arquivo, também é chamado na FILE SECTION da DATA DIVISION.

O seguimento da sequência 400000 até 499000, onde estão declaradas novamente as estruturas dos registros do arquivo, é chamado na WORKING-STORAGE SECTION da DATA DIVISION. Estas estrututras funcionam como áres auxiliares para manipulação dos registros do arquivo.

Os prefixos dos atributos da library de um arquivo têm a seguinte formação:

CAMPOS DE FD: Terceira letra do sistema ("Y", no exemplo) + o sequencial do arquivo ("51", no exemplo) + o número do registro ("01" e "02", no exemplo) + a letra "F" (identifica campos de FD) + "-" (constante) + o mnemônico do atributo ou campo.

CAMPOS DE WORK : Terceira letra do sistema ("Y", no exemplo) + o sequencial do arquivo ("51", no exemplo) + o número do registro ("01" e "02", no exemplo) + "-" (constante) + o mnemônico do atributo ou campo.

Os prefixos dos atributos de um arquivo, diferentemente de outros elementos do SADS, não possuem a letra "A" que identifica um arquivo. Porém, os campos da estrutura de FD são diferenciados através da letra "F", no final do prefixo.

Libraries de arquivo são geradas no usercode definido para o ambiente escolhido, sendo possível gerá-la sem usercode (em *) para o ambiente de produção. A especificação de onde a library deverá residir encontra-se descrita na tabela SBPM5002, que pode ser consultada na Seção V, "Parametrizações do SADS", neste manual.

A transação que executa a geração de library de arquivo não está associada a nenhuma tela de processamento. Portanto, para efetuar a sua geração, basta transmitir os argumentos na tela menu de operação "GERLIBARQ", cujos campos são explicados a seguir:

CAMPOS DA TELA

 Arquivo - Código do arquivo no padrão SADS (sssAxx).

 Ambiente - Ambiente para o qual a library será gerada:

Podem ser informados os seguintes ambientes:

"T" (Teste);

"P" (Produção).

Caso o ambiente não seja informado, será assumido o ambiente default do usuário. Detalhes sobre o cadastramento do ambiente default do usuário podem ser obtidos na seção VI, "Sistema de Segurança", item "Usuários e habilitações".

Quando uma library é gerada em ambiente de teste, o acesso à mesma será feito somente nas compilações para este ambiente. Porém, se a geração for executada para produção, ficando residente sem usercode (dependendo do cadastramento do seu usercode de análise na tabela SBPM5002), o acesso à mesma será feito nas compilações para ambos os ambientes.

 Usercode - Usercode de análise.

O usercode a ser informado deve estar habilitado ao usuário, como elemento, em sua lista de elementos "SEGUSER", no Sistema de Segurança. Detalhes sobre este cadastramento estão descritos na Seção VI, "Sistema de Segurança", item "Habilitações de usuários SADS".

Caso o usercode não seja informado, será assumido o usercode default do usuário. Detalhes sobre o cadastramento do usercode default do usuário podem ser obtidos na seção VI, "Sistema de Segurança", item "Usuários e habilitações".

Para os usercodes de análise, são definidos os ambientes (usercode e disco) onde ficarão residentes as libraries de teste e produção, sendo possível que as libraries de produção fiquem sem usercode. Estas definições estão na tabela SBPM5002, cuja manutenção é feita pelo responsável pelo SADS na sua instalação. Esta tabela encontra-se descrita na Seção V, "Parametrizações do SADS", neste manual.

Liberação de library de arquivo para produção

A liberação da library consiste em copiá-la do usercode de teste para o de produção a ele relacionado. Deste modo, esta library estará disponível para ser incorporada aos programas que forem compilados para produção.

Para liberar uma library para produção, é necessário que esta tenha sido gerada inicialmente para o ambiente de teste.

A liberação da library de arquivo para produção não está relacionada a nenhuma tela de processamento. Para efetuá-la, basta transmitir os argumentos do menu de operação "LIBLIBARQ", cujos campos são explicados a seguir:

CAMPOS DA TELA

 Arquivo - Código do arquivo nos padrões SADS (sssAxx).

 Usercode - Usercode de análise relacionado.

O usercode a ser informado deve estar habilitado ao usuário, como elemento, em sua lista de elementos "SEGUSER", no Sistema de Segurança. Detalhes sobre este cadastramento estão descritos na Seção VI, "Sistema de Segurança", item "Habilitações de usuários SADS".
