Referências cruzadas

O SADS possui um repositório de dados ativo, ou seja, que se mantém atualizado a qualquer fase do desenvolvimento dos sistemas.

Desta forma, para auxiliar nas manutenções e desenvolvimento dos sistemas, o SADS oferece uma grande variedade de referências cruzadas, que exibem os relacionamentos existentes entre os diversos elementos cadastrados em seu repositório de dados, quais sejam:

Atributo x estruturas;

Atributo x programas;

Atributo x sinônimos;

Domínio x atributos;

Elementos x elemento;

Set de acesso x programas.

Atributo x estruturas

Esta consulta é utilizada a fim de obter a lista de todas as estruturas que contêm determinado atributo ou campo.

Caso o atributo a ser pesquisado pertença à estrutura do header padrão de datasets da instalação, nesta consulta também serão exibidos todos os datasets que possuem tal header padrão.

Pode ser informado um código de uma estrutura inicial como parâmetro. Neste caso, a consulta será parcial, informando apenas os elementos relacionados ao atributo a partir daquele especificado.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados", neste manual.

Argumentos da consulta

Para que esta consulta seja utilizada, alguns argumentos devem ser informados na tela REFATREST, quais sejam:

Sistema - Código do sistema do atributo ou campo, no padrão SADS (sss).

Atributo - Mnemônico do atributo ou campo.

  Estrutura - Código da estrutura, a partir da qual a consulta será apresentada (opcional).

Informações exibidas na consulta

ATRIBUTO - Código e nome do atributo ou campo.

Serão informados para cada estrutura onde o atributo ou campo estiver cadastrado:

ESTRUTURA - Código da estrutura.

NOME - Nome da estrutura.

No rodapé da última página da consulta será apresentado o total de estruturas onde o atributo ou campo estiver cadastrado.

Atributo x programas

Esta consulta é utilizada a fim de obter a lista dos programas que trabalham ou utilizam um determinado atributo ou campo.

Será apresentada uma tela de consulta com o atributo ou campo e seu nome no cabeçalho, e a lista de programas que o utilizam.

Para cada programa serão exibidas as estruturas que utilizam o atributo ou campo.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados", neste manual.

Argumentos da consulta

Para que esta consulta seja utilizada, alguns argumentos devem ser informados na tela REFATRPRO, quais sejam:

Sistema - Código do sistema do atributo ou campo, no padrão SADS (sss).

Atributo - Mnemônico do atributo ou campo.

Informações exibidas na consulta

ATRIBUTO - Código e nome do atributo ou campo.

Serão informados para cada programa que utilizar o atributo ou campo.

PROGRAMA - Código do programa, no padrão SADS.

O SADS marca com um "*" os programas que não envolvem movimentação de dados com o atributo ou campo, ou seja, este atributo/campo somente aparece declarado nas estruturas que estão cadastradas no programa.

Porém, se o atributo/campo pertencer à área de work cadastrada para o programa no RD, independentemente de haver ou não movimentação para o mesmo, não será marcado com "*".

NOME - Nome do programa.

ELEMENTOS - Códigos das estruturas que utilizam o atributo ou campo internamente ao programa.

No rodapé da última página da consulta será apresentado o total de programas onde o atributo ou campo está sendo utilizado.

Atributo x sinônimos

Esta consulta é utilizada a fim de obter a lista de todos os sinônimos de um atributo ou campo.

Pode-se definir para todo atributo ou campo vários sinônimos. Deste modo, estes sinônimos herdam todas as suas características, exceto a sintaxe, a tabela e a sub-rotina.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados", neste manual.

Argumentos da consulta

Para que esta consulta seja utilizada, alguns argumentos devem ser informados na tela REFATRSIN, quais sejam:

Sistema - Código do sistema do atributo ou campo que possui sinônimos, no padrão SADS (sss).

Atributo - Mnemônico do atributo ou campo que possui sinônimos.

Informações exibidas na consulta

SISTEMA - Código do sistema do campo ou atributo.

NOME - Nome do campo ou atributo.

CAMPO/ATRIBUTO - Código do campo ou atributo.

FORMATO - Formato do campo ou atributo.

TAMANHO - Tamanho do campo ou atributo.

DECIMAIS - Número de casas decimais que o campo ou atributo possui.

DOMINIO - Sistema e mnemônico do domínio, caso o atributo ou campo o possua.

SINONIMOS:

Serão informados para cada sinônimo do atributo ou campo:

SISTEMA - Código do sistema do sinônimo.

ATRIBUTO - Código do sinônimo.

NOME - Nome do sinônimo.

No rodapé da última página da consulta será apresentado o total de sinônimos do atributo ou campo.

Domínio x atributos

Esta consulta é utilizada a fim de obter a lista de todos os atributos ou campos que pertencem a um determinado domínio.

Domínio é um conjunto de características que podem estar associados aos atributos ou campos.

É apresentada uma tela de consulta paginada com as características do domínio seguida da lista de atributos e campos que possuem este domínio.

A tela de processamento é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados", neste manual.

Argumentos da consulta

Para que esta consulta seja utilizada, alguns argumentos devem ser informados na tela REFDOMATR, quais sejam:

Sistema - Código do sistema do domínio, no padrão SADS.

Dominio - Código do domínio.

Informações exibidas na consulta

SISTEMA - Código do sistema do domínio.

NOME - Nome do domínio.

DOMINIO - Código do domínio.

FORMATO - Formato do domínio.

TAMANHO - Tamanho do domínio.

DECIMAIS - Número de casas decimais do domínio.

ATRIBUTOS:

Serão informados para cada atributo ou campo relacionado ao domínio:

SISTEMA - Código do sistema do atributo ou campo relacionado ao domínio.

ATRIBUTO - Código do atributo ou campo relacionado ao domínio.

NOME - Nome do atributo ou campo relacionado ao domínio.

No rodapé da última página da consulta será apresentado o total de atributos e campos relacionados ao domínio especificado.

Elementos x elemento

Esta consulta é utilizada a fim de obter a lista de todos os elementos relacionados a um determinado elemento.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados", neste manual.

Argumentos da consulta

Para que esta consulta seja utilizada, alguns argumentos devem ser informados na tela REFELEELE, quais sejam:

Elemento-1 - Código de um dos elementos pertencentes à lista Elemento-1 (opcional).

A lista Elemento-1 é composta pelos seguintes elementos:

Banco;

Base de dados;

Função;

Dataset;

Host;

Programa;

Entidade e

Consistência.

Elemento-2 - Código de um dos elementos pertencentes à lista Elemento-2 (opcional).

A lista Elemento-2 é composta pelos seguintes elementos:

Banco;

Dataset;

Sub-rotina;

Remap;

Menu;

Job;

Arquivo;

Tela;

Estrutura de trabalho;

Relatório;

Tabela;

Arquivo geral e

Função.

Quando apenas um elemento for informado na consulta- elemento-1 ou elemento-2, todos os elementos da outra lista que estiverem relacionados com o elemento especificado serão apresentados na consulta.

Quando os dois elementos forem informados - elemento-1 e elemento-2, o elemento-2 será considerado como argumento e o elemento-1 será considerado como parâmetro inicial da consulta a ser apresentada. Neste caso então,todos os elementos pertencentes à lista Elemento-1, a partir do elemento-1 informado, que estiverem relacionados com o elemento-2 especificado serão apresentados na consulta.

Por exemplo:

 Para consultar o banco de dados físico, os hosts e os programas relacionados a um determinado dataset, deve-se informar o código do dataset no campo Elemento-2.

 Para consultar os programas, a partir do programa DCEN11, relacionados a um determinado dataset, deve-se informar o código do dataset no campo Elemento-2 e o código do programa (DCEN11) no campo Elemento-1.

Para consultar os menus, datasets, telas, arquivos, sub-rotinas e demais elementos da lista 2 que estão cadastrados como recursos de um programa, deve-se informar o código do programa no campo Elemento-1.

Para consultar todos os datasets ou tabelas gerados a partir de uma entidade, deve-se informar o código da entidade no campo Elemento-1.

Informações exibidas na consulta

ELEMENTO - Código do elemento que está sendo consultado, no padrão SADS.

NOME - Nome do elemento que está sendo consultado.

Serão apresentados para cada elemento relacionado ao elemento informado:

ELEMENTOS - Código do elemento relacionado ao elemento informado.

NOME - Nome do elemento relacionado.

COMPLEMENTO - Informações complementares do relacionamento existente entre os dois elementos, quais sejam:

Menu - Cadastramento da transação no host;

Library - Tipo da library:

W - library de work;
P - library de procedure;
C - library de área conversacional.

Dataset - Banco de dados físico;

Arquivo:

I - Input;
O - Output.

etc.

No rodapé da última página da consulta será apresentado o total de elementos relacionados ao elemento especificado.

Set de acesso x programas

Esta consulta é utilizada para obter a lista de todas os programas que utilizam determinado set de acesso.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados", neste manual.

Argumento da consulta

Para que esta consulta seja utilizada, um argumento deve ser informado na tela REFSETPRO:

Set acesso - Código do set de acesso, no padrão SADS (sssBxxx).

Informações exibidas na consulta

SET DE ACESSO - Código e nome do set de acesso.

Serão informados para cada programa onde o set de acesso estiver sendo utilizado:

PROGRAMA - Código do programa.

NOME - Nome do programa.

No rodapé da última página da consulta, será apresentado o total de programas onde o set de acesso estiver sendo utilizado.
