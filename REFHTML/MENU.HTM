Menu

Conceituação

Para o usuário final, um menu representa uma tela que exibe determinadas opções, operações ou escolhas que, ao serem solicitadas, ativam transações ou outros menus.

Um menu está sempre vinculado a um sistema, que está identificado em seu código.

Para o SADS, existem três tipos de menus: menus de escolha e de operação, utilizados apenas pela navegação interpretativa, e menus de opção, utilizados para o processamento das transações. Os menus de opção têm de ser definidos no repositório de dados do SADS.

A seguir, será apresentada uma explicação detalhada de cada um destes tipos de menus:

Menu de escolha

Tela que contém rotas de navegação, sendo que cada rota corresponde a um destino que pode ser outro menu de escolha ou um menu de operação. Portanto, um menu de escolha é utilizado apenas para navegação entre menus, não sendo possível solicitar operações específicas a partir dele.

Um menu de escolha não possui cadastramento no repositório de dados do SADS, mas apenas o desenho de sua tela para apresentação durante o processo de navegação.

Menus de escolha não são gerenciados por hosts, e sim pelo módulo de navegação do SADS.

Menu de operação

Tela que contém os dados básicos a serem informados para processamento das transações e de onde as operações são solicitadas.

Um menu de operação não necessita ser cadastrado no repositório de dados do SADS, mas apenas seu desenho e sua estrutura.

Menus de operação não são gerenciados por hosts e sim pelo módulo de navegação do SADS.

Menu de opção

Telas que são tratadas pelos hosts e a partir das quais os programas são disparados para processamento de transações. Estes menus são os únicos efetivamente processados pelos programas aplicativos e não são tratados pelo módulo de navegação do SADS.

A especificação de um menu de opção é composta:

a) De uma tela e sua estrutura;

b) Da definição no repositório de dados, onde serão especificadas as características do processamento a ser executado pelo host para cada uma das transações.

Cada menu de opção pode tratar as opções de 01 a 93, sendo que as opções de 94 a 99 são reservadas para procedimentos gerais do SADS e serão descritas no tópico "Definição de menu", posteriormente.

Para cada opção escolhida deverão ser definidos o programa executor, ou a transação a ser disparada, e as funções genéricas que o host deverá processar, tais como: segurança no acesso aos dados, utilização da área conversacional, disparo da consistência geral, prototipação da transação, etc.

Nos menus gerenciados por hosts primários deve-se definir todas as opções que serão tratadas por eles, inclusive as que serão repassadas a hosts secundários. Nos menus gerenciados por hosts secundários deve-se definir apenas as opções que serão tratadas por ele e submenus de onde as opções serão solicitadas, caso existam.

Devido ao fato de os menus de escolha e de operação serem utilizados apenas na navegação, eles serão detalhadamente explicados no item "Navegação", na Seção III, "Funções de Apoio", neste manual. Desta forma, para este item, qualquer referência ao termo menu deve-se entender "menu de opção".

Padronização

O padrão de codificação adotado pelo SADS para a identificação de menus é o seguinte:

"sssUxn" onde:

"sss" é o código alfabético que identifica o sistema;

"U" é a constante de menu e

"xn" é o identificador alfanumérico do menu.

Se o menu pertencer a um host primário, seu identificador alfanumérico poderá variar de 01 a 09; se pertencer a um host secundário, seu identificador poderá variar de L1 a L9.

Cada host pode tratar, no máximo, cinco menus, sendo que os pares de menus 01 e 06, 02 e 07, 03 e 08, 04 e 09 não podem pertencer ao mesmo host.

Cadastramento

Para efetuar o cadastramento do menu, é necessário que o sistema para o qual o menu está sendo definido esteja cadastrado no repositório de dados do SADS. É necessário, também, que o host que irá gerenciar o menu já esteja cadastrado.

Roteiro

O cadastramento de menu é efetuada através de uma única função, qual seja:

-Definição de menu.

Definição de menu

CAMPOS DA TELA

Menu - Código do menu no padrão SADS (sssUxn).

Nome - Nome do menu, ou seja, sua descrição resumida.

Host - Código do host que irá tratar o menu que está sendo definido no padrão SADS (sssNnn).

Se o identificador alfanumérico do menu variar de 01 a 09, deve-se informar um host primário; se o seu identificador variar de L1 a L9, deve-se informar um host secundário (o identificador alfanumérico de um menu compreende os dois últimos caracteres de seu código).

Cada host pode tratar, no máximo, cinco menus, sendo que os pares de menus 01 e 06, 02 e 07, 03 e 08, 04 e 09 não podem pertencer ao mesmo host.

Para cada uma das opções relativas ao menu devem ser definidos os campos a seguir.

Opcao - Número da opção que poderá ser solicitada do menu.

As opções válidas são de 01 a 93, sendo que as opções de 94 a 99 são reservadas ao SADS e não poderão ser utilizadas para processamentos específicos do sistema.

Todas as opções que poderão ser solicitadas de um menu deverão ser especificadas em seu cadastramento, inclusive aquelas cujos programas executores serão agregados a um host secundário. O programa ou transação responsável por tratar a opção será determinado no campo PROG/TRAN.

As opções de 94 a 99, reservadas ao SADS, podem ser disparadas de qualquer um dos menus dos sistemas desenvolvidos em SADS, não sendo necessário especificá-las na definição destes menus.

Estas opções, denominadas opções genéricas, são tratadas diretamente pelos hosts e, ao serem disparadas, executam funções específicas, tais como:

- Opção 94 - Retorna o menu hierarquicamente superior da navegação executada através do SOS01;

- Opção 95 - Integração com o SPO;

- Opção 96 - Acesso ao menu com detalhamento de opções;

- Opção 97 - Ativação da tela de explicação dos argumentos do menu;

- Opção 98 - Ativação do logon;

- Opção 99 - Ativação do logoff.

A seguir, todas estas opções são explicadas separadamente, com mais detalhes.

Opção 94 - Retorno ao menu hierarquicamente superior da navegação executada através do SOS01.

A navegação via host não possibilita a navegação entre menus e, consequentemente, entre sistemas. O SOS01 propicia esta navegação, auxiliando o usuário final a encontrar o código correto do menu de opção a ser solicitado para a operação desejada.

No menu SOS01 são apresentados os menus disponíveis, podendo ser de todos os sistemas. Deste modo, caso queira se passar de um sistema para o SOS01, deve-se informar opção 94 em qualquer menu de opção.

Opção 95 - Integração com o Sistema de Controle de Processamento (SPO).

Esta opção tem o objetivo de apresentar os serviços cadastrados no SPO que estiverem autorizados ao usuário que a solicitou. O usuário pode, a partir do seu próprio menu, solicitar a preparação de algum dos serviços apresentados.

Opção 96 - Acesso ao menu com detalhamento de opções.

Quando um sistema é utilizado por um número reduzido de usuários e estes já conhecem amplamente todas as transações dos menus, é dispensável que as telas de menu lhes sejam completamente apresentadas. Basta que os campos de argumento e opção estejam disponíveis para solicitar a operação desejada.

Considerando que os menus são as telas mais utilizadas em um sistema, pois é através deles que as operações do sistema são iniciadas, o ganho de tráfego de caracteres na linha seria enorme se fosse possível trabalhar sempre com menus resumidos.

Para isso o SADS oferece a opção 96, um recurso que possibilita a simplificação da tela menu, a fim de reduzir o tráfego da linha, sem, no entanto, perder o acesso ao seu desenho completo.

Para isso, o analista deverá construir o desenho resumido do menu para ser utilizado pelos usuários, conservando o desenho completo do mesmo, que, através da opção 96, pode ser consultado sempre que for necessário.

Passos para simplificação da tela menu:

a) Duplicação da tela menu de código sssTnn00 para o novo código sssTnn96;

b) Simplificação do desenho da tela menu de código sssTnn00 para conter apenas o trancode, data/hora (caso o desenho completo já contiver), campos de argumentos, de opção e de transmissão;

c) Certificação de que o desenho da tela menu de código sssT0096 contém a descrição de todas as suas opções;

d) Se o sistema for cooperativo, deve-se gerar a library da tela de código sssTnn00;

e) Liberação dos desenhos de ambas as telas para o ambiente onde o sistema será executado: teste e/ou produção.

Opção 97 - Ativação da tela de explicação dos argumentos do menu.

Através desta opção o usuário pode obter a explicação sobre o preenchimento dos argumentos necessários para disparar uma determinada transação a partir de um menu ou submenu de opção.

Quando o usuário tiver dúvidas a respeito do preenchimento dos argumentos, ele deve informar o número da opção que lhe gerou dúvidas no início do primeiro campo disponível da tela menu e informar 97 no campo OPCAO. Neste momento, o host lhe apresentará a tela de explicação dos argumentos referente à opção informada.

Para que este recurso oferecido pelo SADS possa ser utilizado, alguns procedimentos devem ter sido efetuados, quais sejam:

a) Cadastramento do desenho da tela de explicação dos argumentos da opção no repositório de dados.

Esta tela deve possuir o código sssEnnmm, onde "nn" representa o código do menu e "mm", o número da opção. Por exemplo, a tela explicativa da opção 03 do menu ABC01 deve possuir o código ABCE0103.

b) Liberação do desenho da tela de explicação para o ambiente onde o sistema será executado: teste ou produção.

Telas de explicação de argumentos não são definidas no repositório de dados do SADS, portanto, suas estruturas não podem ser cadastradas e suas libraries não podem ser geradas, pois não são necessárias.

Esta opção fornece, também, descrições de telas e funções, dependendo da situação em que a opção e a função de help for requisitada.

Opção 98 - Ativação do logon.

Esta opção é utilizada quando o usuário deseja efetuar o logon no ambiente SADS.

Quando esta opção é disparada de um menu de opção de um sistema qualquer desenvolvido através do SADS, o host deste sistema desvia o controle para o Sistema de Segurança, que, por sua vez, apresenta uma tela onde o usuário deve informar seu número e senha, dentre outros dados.

Caso já exista um usuário trabalhando na estação, é efetuado o seu logoff e o logon do novo usuário.

Após o logon, será exibida uma tela, mensagem ou contexto, ou será executado um programa, de acordo com os defaults de encadeamento cadastrados para o usuário, ou de acordo com a parametrização adotada para a instalação e cadastrada na tabela SBPM6004.

Detalhes sobre defaults de respostas após o logon/logoff do usuário estão descritos na Seção VI, "Sistema de Segurança", item "Usuários e habilitações", neste manual.

Detalhes sobre parametrização da resposta a ser enviada após o logon/logoff  estão descritos na Seção III, "Parametrizações do SADS", neste manual.

Opção 99 - Ativação do logoff.

Esta opção é utilizada quando o usuário deseja efetuar o logoff no ambiente SADS.

Após o logoff, será exibida uma tela, mensagem ou contexto, ou será executado um programa, de acordo com os defaults de encadeamento cadastrados para o usuário, ou de acordo com a parametrização adotada para a instalação e cadastrada na tabela SBPM6004.

Detalhes sobre defaults de respostas após o logon/logoff do usuário estão descritos na Seção VI, "Sistema de Segurança", item "Usuários e habilitações", neste manual.

Detalhes sobre parametrização da resposta a ser enviada após o logon/logoff  estão descritos na Seção III, "Parametrizações do SADS", neste manual.

H.Sec - Indicador de desvio de controle para o host secundário.

Pode assumir os seguintes valores:

"S" - Se a opção for tratada por um host secundário;

"N" - Se a opção for tratada pelo host primário.

Todas as transações de um sistema, quando disparadas, ativam primeiramente o host primário, responsável pelos seus respectivos menus. Porém, se a transação disparada possuir o campo H.SEC = "S", o host primário irá desviar seu controle para o host secundário específico. Neste caso, um programa agregado ao host secundário, definido no campo PROG/TRAN, é que irá processar esta transação.

O objetivo do host secundário é processar as transações mais demoradas, para aliviar o processamento do host primário e evitar enfileiramentos. Pode-se conseguir, desta forma, melhor performance do sistema.

Prog/Tran - Código do programa ou transação que irá tratar a opção.

Este campo poderá ser preenchido com:

- O código do programa, quando a opção disparar a execução de um programa específico. Pode ser informado um código de programa mesmo que a transação esteja em prototipação;

- O código da transação, quando a opção disparar a execução de uma transação já implementada em outro host, diferente daquele que trata o menu. Neste caso, quando a opção for requisitada, o host se encarregará de passar os argumentos informados no menu para o programa executor da transação informada no campo PROG/TRAN.

Quando for informado o código de uma transação, deve-se informar "T" no campo A/C/E/M/T.

- O código do host, quando a opção for um submenu ou a transação estiver em prototipação. Pode-se informar o código do host também para transações que serão processadas pelo host secundário;

- O código do host secundário, quando, no cadastramento do menu primário, a opção for executada por um programa que será agregado ao host secundário. Neste caso, pode-se informar também o código do host primário.

ACV - Indicador de utilização de área conversacional.

Pode assumir os seguintes valores:

"S" - Se o programa que executa a opção utilizar área conversacional;

"N" - Se o programa que executa a opção não utilizar área conversacional.

A área conversacional tem como função armazenar e transferir dados entre as transações efetuadas em uma mesma estação.

Toda transação de atualização deve utilizar obrigatoriamente a área conversacional.

O host limpará a área conversacional toda vez que for disparada uma opção que não utilizar área conversacional. Portanto, mesmo em programas de consulta, para conservar as informações armazenadas nesta área, deve-se declarar a sua utilização.

Acao - Indicador de utilização do campo ACAO na tela de processamento tratada pela opção.

Pode assumir os seguintes valores:

"S" - Se a tela de processamento tratada pela opção utilizar o campo ACAO.

"N" - Se a tela de processamento tratada pela opção não utilizar o campo ACAO.

Para obter maiores detalhes em como utilizar o campo ACAO em telas de processamento, deve-se consultar o item "Tela", subitem "Cadastramento", nesta seção.

A/C/E/M/T - Tipo do programa ou transação a ser disparada pela opção.

Os valores possíveis são os seguintes:

"A" - Para que a opção dispare a execução de um programa de atualização(Neste caso, deve-se informar o código do programa em PROG/TRAN).

"C" - Para que a opção dispare a execução de um programa de consulta(Neste caso, deve-se informar o código do programa em PROG/TRAN).

"E" - Para que a opção dispare a execução de um programa cuja lógica seja de consulta encadeada ou de seleção(Neste caso, deve-se informar o código do programa em PROG/TRAN).

"M" - Para que a opção ative o host para apresentação da tela de um submenu(Neste caso, deve-se informar o código do host que irá tratar as opções do submenu em PROG/TRAN e o identificador numérico do submenu a ser apresentado no campo TELA. O código do submenu será composto de: código do menu + a opção).

"T" - Para disparar uma transação já implementada em outro host(Neste caso, deve-se informar qual transação será disparada em PROG/TRAN. Caso a transação solicitada seja processada, no outro host, através de um submenu, deve-se informar o número do submenu de destino no campo TELA).

Exemplo:

Se a transação a ser disparada pertencer ao menu ADF03, submenu 60, e se referir à opção 63, deve-se informar ADF0363 no campo PROG/TRAN (que será o código da transação disparada), e 60 (submenu) no campo TELA.

Segur - Indicador de execução de testes de segurança quando a opção for disparada.

Os valores possíveis são os seguintes:

"S" - Para que o host execute os testes de habilitação do Sistema de Segurança, para esta opção;

"N" - Para que a opção seja utilizada sem verificação de habilitação no Sistema de Segurança.

Através da tabela SBPM5000, o responsável pelo suporte SADS pode definir que somente usuários autorizados podem eliminar testes de segurança que tenham sido cadastrados para serem disparados antes da execução das transações. Desta forma, a informação do valor "N" nos campos "Segur", "Consistencia Menu" e "Consistencia Transacao" somente poderá ser feita por usuários devidamente autorizados.

Detalhes sobre esta parametrização podem ser obtidos na Seção V, "Parametrizações do SADS", item "Segurança", neste manual.

Caso todas as opções do menu exijam segurança, este menu só será apresentado para usuários habilitados. Caso ele seja solicitado e o usuário não tenha efetuado o logon, será apresentada a tela para a execução do logon e, posteriormente, será apresentado o menu.

Consistencia - Indicador de disparo da consistência geral pelo host.

A consistência geral de uma opção poderá ser disparada pelo host nas seguintes situações:

1- Quando a opção for solicitada do menu - Neste caso, a consistência geral validará os argumentos da tela menu informados no momento do disparo da opção;

2- Antes de o host entregar a mensagem vinda da tela de processamento da opção para seu programa executor - Neste caso, a consistência geral validará os dados informados pelo usuário nesta tela de processamento.

Cumpre ressaltar que programas de atualização paginada não devem definir o disparo da consistência geral pelo host, visto que o Paginador de Entrada é que se encarrega de dispará-la.

Os preenchimento dos dois próximos campos da tela de cadastramento de menu determinam se haverá ou não disparo da consistência geral da opção pelo host no menu e em sua tela de processamento.

Menu - Indicador de disparo da consistência geral a partir do menu.

Pode assumir os seguintes valores:

"S" - A consistência geral do menu será disparada pelo host quando esta opção for solicitada do menu;

"N" - A consistência geral do menu não será disparada pelo host quando esta opção for solicitada do menu.

Através da tabela SBPM5000, o responsável pelo suporte SADS pode definir que somente usuários autorizados podem eliminar testes de segurança que tenham sido cadastrados para serem disparados antes da execução das transações. Desta forma, a informação do valor "N" nos campos "Segur", "Consistencia Menu" e "Consistencia Transacao" somente poderá ser feita por usuários devidamente autorizados.

Detalhes sobre esta parametrização podem ser obtidos na Seção V, "Parametrizações do SADS", item "Segurança", neste manual.

Transacao - Indicador de disparo da consistência geral a partir da tela de processamento da transação.

Pode assumir os seguintes valores:

"S" - A consistência geral da tela de processamento será disparada pelo host quando da sua transmissão;

"N" - A consistência geral não será disparada pelo host quando da transmissão da tela de processamento da transação.

Através da tabela SBPM5000, o responsável pelo suporte SADS pode definir que somente usuários autorizados podem eliminar testes de segurança que tenham sido cadastrados para serem disparados antes da execução das transações. Desta forma, a informação do valor "N" nos campos "Segur", "Consistencia Menu" e "Consistencia Transacao" somente poderá ser feita por usuários devidamente autorizados.

Detalhes sobre esta parametrização podem ser obtidos na Seção V, "Parametrizações do SADS", item "Segurança", neste manual.

O ideal é que as consistências sejam disparadas pelo host pois, neste caso, evita-se que programas sejam carregados na memória somente para detectar erros de consistência.

Protot - Indicador de prototipação da transação relativa à opção.

Pode assumir os seguintes valores:

"S" - Se a transação está em prototipação. Neste caso, o código do host ou do futuro
programa que executará a opção deve ser informado no campo PROG/TRAN;

"N" - Se a transação não está em prototipação. Neste caso, o código do programa ou da
transação que irá tratar a opção deve ser informado no campo PROG/TRAN.

Não é possível informar "S" neste campo para submenus e para transações externas.

A prototipação permite que o usuário tenha uma visão de como será o funcionamento do sistema antes dos programas executores das opções e da base de dados do mesmo serem construídos.

Quando for informado que haverá prototipação, nenhum programa chamado ou transação será disparada. Neste caso, o host apenas se encarregará de efetuar funções como:

- Testar se o usuário está habilitado a trabalhar com a opção requisitada;

- Exibir a tela de processamento definida para a opção solicitada;

- Validar os dados de entrada;

- Interagir com o help on-line do SADS;

- Executar a navegação dos menus desejada.

Tela - Código numérico que identifica a tela.

A partir do identificador numérico informado neste campo, o código da tela será composto da seguinte forma:

"sssTnnmm" onde:

"sss" é o código alfabético que identifica o sistema;

"T" é a constante de tela;

"nn" é o identificador numérico do menu e

"mm" é o identificador numérico da tela.

É aconselhável que a tela possua o mesmo número da opção do menu ao qual se encontra relacionada, pois o processo de geração de programas on-line pressupõe esta compatibilidade.

Este número é utilizado para identificar o código da tela nas seguintes situações:

- Quando a opção estiver em prototipação - PROTOT = "S". Neste caso, o número informado identificará a tela que será apresentada ao usuário quando a opção for disparada do menu para uma prévia visualização do sistema;

- Quando tiver sido definido um programa executor específico da opção - PROG/TRAN = <<sssNnn>>. Neste caso, o código numérico identificará a tela de processamento do programa executor da opção. Esta tela será lida pelo host antes de desviar o processamento para o programa executor da opção, evitando que o programa tenha que estabelecer um link físico com a sub-rotina de acesso a telas. Portanto, é aconselhável identificar o número da tela para uma melhor performance do sistema;

- Quando tiver sido determinado que uma transação já definida em outro host será disparada através da opção. Neste caso, este número identificará o código do submenu de destino, relativo à transação a ser disparada.

Exemplo:

Se a transação for disparada pela opção 63 do menu ADF03, submenu 60, deve-se informar "ADF0363" no campo PROG/TRAN, que é o código da transação disparada, e "60" (submenu) no campo TELA.

O número de tela não deverá ser informado quando a transação destino for executada a partir do menu principal.

Continua (S/N) - Indicador de continuidade de operação.

As possibilidades de preenchimento deste campo são as seguintes:

"S" - Nova tela será exibida para prosseguir informando as demais opções do menu,
quando uma tela não for suficiente para a informação de todas elas;

"N"- Última tela para a informação das opções do menu.

Operacao - Operação a ser efetuada em relação ao cadastramento do menu.

As operações possíveis são:

"INC" (Inclusão);

"ALT" (Alteração);

"EXC" (Exclusão);

"CON" (Consulta).
