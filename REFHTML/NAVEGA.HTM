Navegação

Conceituação

Navegação é o processo de caminhar pelos menus dos sistemas e, através deles, solicitar transações.

O SADS tem à disposição duas formas de navegação para os sistemas aplicativos:

Navegação via host e

Navegação interpretativa.

Navegação via host

A navegação via host é aquela em que o usuário solicita as transações diretamente a partir dos menus de opção, informando neles o número da opção e os argumentos referentes à transação que ele deseja executar.

Nesta navegação só é permitido o deslocamento entre submenus de um mesmo menu, caso existam. Também só é permitida a solicitação, de um menu de opção, daquelas transações que foram definidas como pertencentes a ele no repositório de dados do SADS. Não é possível, pois, através deste tipo de navegação, navegar livremente entre diferentes menus.

Deste modo, para se solicitar uma transação é obrigatório ter conhecimento, de antemão, do código do menu no qual a mesma está definida. Somente a partir deste menu é que a transação poderá ser solicitada.

A transação SOS01 propicia a navegação entre os menus dos sistemas de forma hierarquizada, auxiliando o usuário final a encontrar o código correto da transação a ser disparada, para utilização de um determinado sistema.

O SOS01 controla a navegação entre os níveis I, II, III e IV. O nível V já é controlado pelo host do sistema. A partir de algum menu (nível V) ou submenu (nível VI) de um sistema, para retornar ao menu de nível IV e passar o controle da navegação para o SOS01, deve-se utilizar a opção 94.

O SOS01 aceita também, para fazer o encadeamento, que seja informado o nome de um contexto já existente, informando ou não a palavra "CONTEXTO" anteriormente.

Navegação interpretativa

A navegação interpretativa é o processo que permite caminhar pelas funções disponíveis de um ou vários sistemas, através de um deslocamento amigável entre seus menus de escolha, menus de operação e telas de processamento.

Estes menus foram projetados em conformidade com o padrão SAA - System Application Architecture, que, através de seu módulo CUA - Common User Access, define padrões para a interface do usuário com o software.

Menus de escolha são aqueles que definem as rotas de navegação disponíveis, sendo que cada rota corresponde a um destino que pode ser outro menu de escolha ou um menu de operação.

Um menu de escolha é utilizado, portanto, apenas para navegação entre menus, não sendo possível solicitar operações específicas a partir dele.

Menus de operação são aqueles que definem as operações e os dados básicos a serem informados para solicitação de processamento das transações.

A diferença entre os menus de operação e os de escolha está no corpo de cada um: enquanto os menus de escolha exibem rotas de navegação e um campo para escolha da rota a ser seguida, os de operação apresentam os campos específicos associados às transações que podem ser solicitadas a partir do menu.

Menus de operação, portanto, são as únicas telas, na navegação interpretativa, a partir das quais pode-se solicitar a execução de uma operação.

Deste modo, o acesso a uma tela de processamento de uma operação é feito a partir do preenchimento dos campos do menu de operação a ela correspondente.

Na navegação interpretativa os menus de escolha agrupam transações operacionalmente relacionadas com o objetivo de criar visões para grupos de usuários que executam funções similares. Estas funções, por sua vez, podem ter sido definidas em diferentes menus de opção e em diferentes sistemas.

A navegação interpretativa é controlada pelo módulo de navegação em uma camada superior à navegação via host.

Assim, a navegação interpretativa funciona como uma máscara para o usuário final, que, após permitir a ele um livre deslocamento entre os diversos menus, desvia o controle para a navegação via host, de forma transparente para o usuário, para ser efetivada.

Internamente, os sistemas continuam trabalhando da forma convencional, através de menus de opção, que são controlados diretamente pelo host.

O módulo de navegação do SADS faz a conversão dos dados entre os layouts do menu de operação e do menu de opção automaticamente, toda vez que uma operação é disparada. Esta transferência só é possível devido ao fato de existirem regras de conversão de layout cadastradas.

As regras de conversão são definidas apenas para os menus de operação, pois apenas esses menus disparam transações vinculadas aos menus de opção.

Concluindo, a navegação interpretativa possui uma série de vantagens que sugerem, por isso, a sua utilização nos sistemas de cunho operacional e, principalmente, nos sistemas gerenciais.

Suas principais características são:

Possibilidade de criação de vários contextos de navegação - agrupamentos de telas menu - totalmente independentes.

Com isso, torna-se possível obter visões diferenciadas das transações que compõem um sistema de acordo com a classe de usuários que irão utilizá-las.

Possibilidade de agrupamento de sistemas ou funções de sistemas diferentes em um único contexto de navegação, possibilitando aos usuários utilizar todas as transações para as quais esteja habilitado como se fosse um único sistema.

Alteração nos menus, das rotas de deslocamentos entre os menus e das regras de conversão de layout sem necessidade de gerações e recompilações de programas envolvidos na navegação.

Independência do processo de navegação em relação aos programas aplicativos, possibilitando sua transposição para processamento cooperativo em estações de trabalho.

Possibilidade de utilização de ajuda on-line (help), em tempo de execução.

Elementos da navegação interpretativa

No processo de navegação interpretativa estão envolvidos os seguintes elementos:

Menus de escolha;

Menus de operação;

Telas de processamento;

Rotas de navegação e

Regras de conversão de layout.

Cada um destes elementos será detalhadamente explicado a seguir, para melhor entendimento do mecanismo da navegação interpretativa.

Menus de escolha

Menus de escolha são aqueles onde estão apresentadas palavras-chave possíveis de serem escolhidas. Estas palavras do corpo do menu identificam as rotas que podem ser selecionadas a partir dele. Cada rota corresponde a um destino que pode ser outro menu de escolha ou um menu de operação.

As telas dos menus de escolha não pertencem a um contexto específico, ou seja, podem ser referenciadas em vários contextos simultaneamente. No entanto, as palavras-chave apresentadas definem rotas de navegação e as rotas são definidas por contexto. Ou seja, apesar de as palavras-chave disponíveis em uma tela menu de escolha serem as mesmas para qualquer contexto que a utilize, as rotas podem definir destinos diferentes em contextos diferentes para uma mesma palavra-chave.

Um menu de escolha é utilizado, portanto, apenas para navegação entre menus, não sendo possível solicitar operações específicas a partir dele.

As escolhas disponíveis são indicadas no corpo do menu por palavras-chave, que são indicadores ou mesmo segmentos das palavras que representam as rotas desejadas. No contexto SADS estes indicadores estão representados em letras maiúsculas.

Para cada menu de escolha devem ser cadastradas no repositório de dados as rotas de navegação que partem dele, ou seja, qual tela de destino deverá ser apresentada para cada palavra-chave possível de ser escolhida no menu.

O menu de escolha é dividido em três áreas: barra de ação, corpo do menu e área de mensagem.

Cada um destes elementos será explicado a seguir.

Barra de ação

A barra de ação é a primeira área do menu e é composta por:

Título do menu: Código alfabético do menu, composto de três, seis ou nove caracteres;

Descrição do título;

Data e hora em que o menu foi recebido no terminal;

Campo ACAO: Campo onde é possível executar um dos seguintes comandos:

"Inicio <nome do contexto> <título do menu >" - Comando para apresentar o menu inicial do contexto especificado, ou o menu especificado do referido contexto;

A ação "Início" sem especificação do nome do contexto e título do menu, apresenta o menu inicial do contexto corrente.

Caso um nome de contexto tenha sido especificado, este passará a ser o contexto corrente.

A informação do título do menu exige que o nome do contexto tenha sido especificado.

"Anterior" - Comando para exibir o último menu apresentado com os campos preenchidos

Este comando não irá exibir as telas repetidas, caso uma mesma tela seja utilizada mais de uma vez. Apenas uma única amostragem das últimas telas processadas será exibida.

Quando for informada a ação "anterior", e a tela anterior for uma tela de processamento, será apresentado o menu de operação que a tiver originado, pois telas de processamento não são gravadas no arquivo de controle da navegação.

"Origem" - Comando para retornar ao menu de nível imediatamente superior na hierarquia de menus;

Quando for informada a ação "origem" em uma tela de processamento, será apresentado o menu de operação que tiver originado esta transação.

"Desvio <título do menu> <nome do contexto>" - Comando para desviar para o menu correspondente ao título informado do contexto corrente ou do especificado;

Caso um nome de contexto tenha sido especificado, este passará a ser o contexto corrente.

A informação do nome do contexto exige que o título do menu tenha sido especificado.

As telas de processamento não possuem título, e por isto não é possível executar um desvio para elas.

"Restaura" ou "Refresh" - Comando para restaurar o menu corrente com todos os campos preenchidos, caso a transação tenha sido executada.

Nesta ação, serão restaurados até o limite de 700 bytes dos campos preenchidos.

Quando se executa a ação "restaura" ou "refresh" em uma tela de processamento com um campo de ação, será apresentado o menu de operação que a tiver originado.

"Fim" - Comando para encerrar a sessão do SADS, efetuando um logoff na estação;

"Comando <contexto>:<título do menu>> <argumentos da transação>" - Comando que corresponde à transmissão da tela menu com o título informado, definida no contexto determinado, com os argumentos especificados.

O código do contexto pode não ser informado e, neste caso, o título do menu deve estar definido no contexto correntemente logado.

Os argumentos da transação podem não ser informados, possibilitando o disparo de um menu de operação sem argumentos.

Este título de tela informado deve ser correspondente a um menu de operação, pois somente de menus de operação pode-se solicitar uma transação qualquer.

Exemplo: "C PEDIDOS:CADCLI ALT001"

"Contexto <nome do contexto> <título do menu>" - Comando para se efetuar desviar para o contexto especificado ou para o menu especificado do referido contexto ;

A informação do título do menu exige que o nome do contexto tenha sido especificado.

Caso não se saiba o nome do contexto que se deseja utilizar, pode-se solicitar a ação "D CONTEXTO" a fim de que seja apresentada a tela de seleção de contextos disponíveis (Tela STKT0004).

"Usuario" - Comando para fornecer as seguintes informações sobre o usuário e a sessão:

Usuário: número, nome e código do usuário, sigla e área da empresa, padrões de impressora e de usercode;

Sessão: número e nome da estação, data e horário do início da sessão e ambiente do logon.

Para executar uma ação, deve-se informar a palavra referente à ação que se quer executar ou apenas o conjunto de caracteres que identifica o comando. Porém, para utilizar a ação CONTEXTO, deve-se informar pelo menos "CON" para diferenciá-la da ação COMANDO.

Como exemplo, as três informações descritas a seguir irão disparar a mesma operação de navegação:

"D DEFENT";

"DES DEFENT";

"DESVIO DEFENT".

Exemplos de comandos disparados no campo ACAO para o contexto SADS:

ORIGEM: No menu DEFENT (Definição de Entidade), se o campo ACAO for preenchido com "O" (Origem), a navegação apresentará o menu de escolha ENT (Entidade), que corresponde à origem do menu DEFENT;

DESVIO: Se o campo ACAO for preenchido com "D INCUSU", a navegação apresentará o menu de título INCUSU, que corresponde ao menu de operação de inclusão de usuários do Sistema de Segurança;

COMANDO: Se o campo ACAO for preenchido com "C DEFENT INCABCECAT", a navegação apresentará a tela de processamento para inclusão da entidade ABCECAT, pois foi disparado o menu de operação DEFENT com os argumentos "INCABCECAT" (Operação = "INC", Entidade = "ABCECAT").

As ações INICIO, DESVIO, COMANDO, RESTAURA (REFRESH), CONTEXTO e USUARIO podem também ser executadas de uma tela desformatada ou vazia.

Também é possível efetuar outras operações no campo ACAO, além daquelas disponíveis no SADS, tais como: "?ON CANDE", "?WRU", outros trancodes, etc.

O campo ACAO deve estar vazio para que qualquer informação seja transmitida do campo ESCOLHA.

Corpo do menu

O corpo do menu é a sua segunda área, onde está definido o conjunto de rotas de navegação válidas e o campo ESCOLHA.

Cada rota de navegação corresponde a um destino que pode ser outro menu de escolha ou um menu de operação.

A hierarquia de menus deve ser construída a partir de uma ordem lógica de utilização, de forma a facilitar o seu uso.

O campo ESCOLHA permite a navegação por esses menus e o seu preenchimento é feito utilizando-se a identificação da rota a ser seguida. No contexto SADS foram utilizadas letras maiúsculas - em geral, as três letras iniciais das palavras - para representar as rotas.

Como exemplo, para selecionar o menu de Utilitários, a partir da tela inicial do SADS, informe "UTI" no campo ESCOLHA.

Também é possível fazer escolhas múltiplas nos menus de escolha. Para isto deve-se digitá-las separadas por um caracter especial no campo ESCOLHA. Por exemplo: no menu inicial do SADS, pode-se digitar "TEL DES" no campo escolha e será apresentado o menu "DESTEL".

Área de mensagem

A área de mensagem é o último espaço do menu, que corresponde às duas linhas finais da tela.

Esse é o local em que serão apresentadas mensagens que orientarão a utilização dos sistemas: mensagens de erro e mensagens de confirmação de operação.

Menus de operação

Menu de operação é aquele que define as operações e os dados básicos a serem informados para solicitação de processamento das transações.

Os menus de operação são divididos em três áreas: barra de ação, corpo do menu e área de mensagem.

A diferença entre os menus de operação e de escolha está no corpo de cada um: enquanto os menus de escolha exibem rotas de navegação e um campo para escolha da rota a ser seguida, os de operação apresentam os campos específicos associados às transações que podem ser solicitadas a partir deles.

Menus de operação, portanto, são as únicas telas a partir das quais pode-se solicitar a execução de uma operação.

Menus de operação, a exemplo dos de escolha, não pertencem a um contexto específico e, portanto, podem também ser utilizados em vários contextos simultaneamente. Como as regras de conversão de layout são definidas para um determinado menu de operação, elas também não são definidas para um contexto específico.

O módulo de navegação do SADS faz a conversão dos dados transmitidos nos menus de operação para o layout do menu de opção automaticamente, toda vez que uma operação é disparada. Esta transferência só é possível devido ao fato de existirem regras de conversão de layout cadastradas.

Não é necessário que a estação esteja logada em um contexto para que, ao ser transmitido um menu de operação, seja executada sua regra de conversão.

As regras de conversão são definidas apenas para os menus de operação, pois apenas esses menus disparam transações vinculadas aos menus de opção.

Telas de processamento

Tela de processamento é uma tela utilizada pelo programa que processa uma transação, permitindo a execução das operações de inclusão, alteração, exclusão, consulta, geração, liberação e outras relativas aos elementos manipulados pelos programas.

Existem transações que podem ser executadas diretamente a partir de um menu de operação, sem que sejam apresentadas telas de processamento associadas. Como exemplo destas transações, no SADS tem-se as liberações de consistências, ressequenciamento de estruturas etc.

Em uma tela de processamento, pode-se tomar duas decisões:

Retornar a um menu, sem executar o processamento da transação, considerando-se os seguintes casos:

Se a tela de processamento utilizar o campo de ação, o retorno será através deste campo;

Caso o campo de ação não seja utilizado, o retorno será ao menu de escolha hierarquicamente superior ao menu de operação que tiver originado a transação.

Neste caso, a tela deve ser transmitida com o cursor no início do seu primeiro campo, após o código de transação, ou caso a tela possua o campo CONFIRME, este deve ser preenchido com "N";

Se o terminal estiver programado para transmitir a tela inteira, deve ser utilizada a tecla "ETX" para transmitir somente o trancode.

Processar a operação solicitada.

As telas de processamento são de quatro tipos:

Atualização Simples: são telas com características básicas dos elementos, como definições ou descrições.

Atualização Paginada: são telas que permitem atualização de itens em uma ou mais páginas, com facilidade de deslocamento entre elas, sem obedecer a uma sequência lógica.

Os campos específicos para atualização paginada são:

PAGINA - Número da página corrente.

PROXIMA PAGINA - Número da próxima página que se deseja visualizar.

Pode assumir os seguintes valores:

"nn" - O número da página desejada;
"--" - Para acesso à página anterior;
"UL" - Para acesso à última página;
"99" - Para encerramento da paginação e efetivação do processamento da
transação.

Se este campo não for informado, será apresentada a próxima página na sequência que foi programada.

A atualização de páginas somente será efetuada quando for transmitido o valor "99" no campo PROXIMA PAGINA.

Consulta Simples: são telas com características básicas dos elementos, como definições ou descrições.

Consulta Paginada: são telas que permitem que os itens sejam apresentados em uma ou mais páginas, com a facilidade de deslocamento entre elas, sem obedecer a uma sequência lógica.

Os campos específicos para consulta paginada são:

PAGINA - Número da página corrente.

PROXIMA PAGINA - Número da próxima página que se deseja visualizar.

Pode assumir os seguintes valores:

"nn" - O número da página desejada;
"--" - Para acesso à página anterior;
"UL" - Para acesso à última página;
"99" - Para encerramento da paginação.

Maiores detalhes a respeito de telas de processamento estão descritos na Seção II, "Elementos do repositório de dados do SADS", item "Tela", neste manual.

Rotas de navegação

Uma rota é definida por um menu de escolha, uma palavra-chave e um destino.

O destino pode ser outro menu de escolha ou um menu de operação.

As rotas de navegação são sempre definidas a partir de um menu de escolha, sendo que este tipo de menu é utilizado apenas para navegação entre menus, não sendo possível solicitar operações específicas a partir deles.

As escolhas disponíveis são indicadas no corpo do menu por palavras-chave que, no contexto SADS, são segmentos das palavras que representam as rota desejadas, em letras maiúsculas.

Regras de conversão de layout

Regras de conversão de layout são mapeamentos existentes entre os campos do menu de operação e os campos do menu de opção correspondente.

A navegação, com seus menus de escolha e operação, funciona como uma máscara para o usuário final. Assim, internamente, os sistemas continuam trabalhando da forma convencional, através de menus de opção.

Para que uma transação seja realmente disparada, os campos de seu menu de opção correspondente devem ser preenchidos com os argumentos informados pelo usuário.

Sendo assim, toda vez que uma operação for solicitada a partir de menu de operação, os dados nele informados deverão ser passados para os campos correspondentes no menu de opção, para que a transação seja efetivamente processada.

O módulo de navegação do SADS fará a transferência automática de dados entre o menu de operação e o menu de opção a ele relacionado. Porém, esta transferência só é possível devido ao fato de existirem regras de conversão cadastradas para cada menu de operação da navegação.

As regras de conversão são definidas apenas para os menus de operação, pois apenas a partir deles as transações podem ser solicitadas.

Não é necessário que a estação esteja logada em um contexto para que, ao ser transmitido um menu de operação, seja executada sua regra de conversão.

Cadastramento

Um sistema aplicativo pode ter sua navegação projetada seguindo o mesmo padrão SAA adotado pelo SADS. O desenvolvimento do sistema, porém, tem de ser efetuado de acordo com a navegação via host.

Para um sistema utilizar a navegação interpretativa, seus menus de escolha e de operação devem ser desenhados e devem ser feitos os cadastramentos das rotas, para possibilitar o deslocamento entre eles, e das regras de conversão de layout.

Um sistema aplicativo já existente não precisa de adaptação para utilização da navegação interpretativa. Basta que ela seja criada, pois os dois tipos de navegação funcionam independentemente e apenas a executada pelo host é obrigatória.

A construção de um novo sistema exige que a navegação via host, com menus de opção, seja construída. Se o sistema, porém, for operado apenas através da navegação interpretativa, devem ser executados os seguintes procedimentos:

Definição de um único menu - menu de opção - contendo todas as opções possíveis de serem executadas pelo sistema.

O processo de cadastramento de menu está descrito na Seção II, "Elementos do repositório de dados do SADS", no item "Menu".

Cadastramento do desenho do menu de opção.

Não é necessário desenhar as constantes, bastando desenhar todos os argumentos a serem utilizados pelos programas chamados e o campo de opção.

Cadastramento da estrutura do menu de opção.

Geração e liberação de sua library.

Cadastramento do desenho e estrutura de todas as telas de processamento tratadas pelos programas chamados.

Geração e liberação da library destas telas de processamento.

Após terem sido executados estes procedimentos, pode-se, então, definir a sua navegação através de menus de escolha, menus de operação, regras e rotas de navegação.

Roteiro

O processo de definição da navegação consiste em cadastrar todos os elementos nela envolvidos, através da execução consecutiva dos seguintes passos:

Especificação dos menus de operação e escolha a serem utilizados no contexto.

Definição de um contexto de navegação para o sistema aplicativo ou criação de novas rotas em um contexto já existente.

Contexto de navegação é o nome dado ao agrupamento de menus de escolha e de operação originários de um ou mais sistemas e que delimitam uma visão global para o usuário.

Para definir um determinado contexto, com seus menus, rotas e regras, o usuário deve ter habilitação de elemento no Sistema de Segurança para tal contexto. O nome da lista de elementos a qual o usuário deverá ser habilitado é a "CONTEXT". Esta lista deverá conter, como elementos, o código de todos os contextos com os quais o usuário irá trabalhar, com o devido tipo de habilitação (só consultar, consultar e atualizar ou não trabalhar). Para maiores detalhes sobre o cadastramento de habilitação de contextos, deve-se consultar a Seção VI, "Sistema de Segurança", item "Habilitação de usuários SADS", neste manual.

Cadastramento dos títulos dos menus desenhados, associando-os aos seus códigos de tela.

Não deve haver títulos de telas idênticos num mesmo contexto, para que seja possível o deslocamento do usuário através dos nomes delas.

Cadastramento das rotas de navegação para cada menu de escolha, isto é, cadastramento das possibilidades de deslocamentos a partir das palavras-chave nele apresentadas.

Cadastramento e liberação das regras de conversão de layout para cada menu de operação, para converter os dados informados no menu para o formato que os programas foram preparados para processar. A liberação das regras pode ser feita uma a uma ou, de uma só vez, pode ser feita a liberação de todas as regras cadastradas em um contexto especificado.

Após cadastrados os elementos relativos à navegação devem ser efetuados certos procedimentos de preparação para utilização da mesma.

A seguir, cada um destes passos será explicado detalhadamente, assim como o procedimento para utilização do contexto definido.

Especificação dos menus

Menus de operação e de escolha não são obrigatórios para o funcionamento do sistema e somente devem ser especificados caso contextos de navegação venham a ser criados para o sistema.

Já os menus de opção e as telas de processamento, independentemente da utilização ou não da navegação, devem ter suas características especificadas no repositório de dados a fim de permitir o desenvolvimento de um determinado sistema.

Neste tópico, que diz respeito à navegação, somente serão explicadas as especificações dos menus de escolha e de operação. As especificações dos demais elementos - menus de opção e telas de processamento - estão descritas na Seção II, "Elementos do repositório de dados do SADS", itens "Menu" e "Tela", respectivamente.

Para os menus de escolha deverão ser especificados:

Desenho de menu;

Definição de menu;

Liberação de desenho de tela menu;

Cadastramento de título de tela menu;

Cadastramento de rotas de navegação.

Para os menus de operação deverão ser especificados:

Desenho de menu;

Definição de menu;

Cadastramento de estrutura de tela menu;

Cadastramento e liberação de consistência de tela menu, se necessário;

Liberação de desenho de tela menu;

Cadastramento de título de tela menu;

Cadastramento de regras de conversão de layout;

Liberação de regras de conversão de layout.

Menus de escolha e de operação não possuem libraries. Não é necessário, portanto, efetuar a geração das mesmas.

A seguir, serão apresentados os passos a serem obedecidos para se definir cada um destes menus e os padrões utilizados para a navegação do SADS.

Desenho de menu

O desenho destes menus deve obedecer às seguintes regras:

O trancode deve ser desenhado como um campo protegido, transmissível e invisível, a fim de que seja transparente ao usuário final.

No desenho do menu, o título deve ser colocado na primeira linha, seguido de sua descrição.

Sugere-se que, tal qual na navegação do SADS, o mesmo seja definido em caracteres brilhantes.

Exemplo: CADFOR - Cadastro de Fornecedores.

Para utilização amigável do sistema, deve-se escolher um título adequado para o menu e de fácil memorização pelo usuário.

Sugere-se que este título seja formado em conjunto de três caracteres, respeitando o limite de nove, sendo que a regra de formação do título deve ser informada ao usuário para facilitar a utilização do sistema.

Exemplo: Nome do menu - "Desenho de Tela";

Título do menu - "DESTEL".

Na segunda linha deve ser desenhado o campo de ação, com 69 posições, e na terceira, as ações que podem ser informadas neste campo.

No início da terceira linha deverá ser definido um caractere de sublinhado, a fim de separar a barra de ação do corpo do menu.

O corpo do menu deve ser desenhado abaixo da barra de ação.

Este varia de acordo com o tipo do menu, escolha ou operação.

Nos próximos tópicos, serão detalhados os procedimentos e regras necessários ao correto desenho de cada um destes menus.

Em seguida ao conteúdo definido no corpo do menu, deve-se definir um campo de transmissão.

Caso se queira seguir a padronização adotada nos menus de navegação do SADS, deve-se desenhar uma linha com apenas o caractere de sublinhado na primeira posição, para separar o corpo do menu da área de mensagem.

Menu de escolha

Padrões a serem utilizados no desenho dos menus de escolha, que são diferentes dos menus de operação:

No corpo do menu devem ser desenhadas as rotas que podem ser seguidas.

Cada uma das rotas definirá um deslocamento a partir deste menu. Em cada rota, deve-se indicar em caracteres maiúsculos as palavras-chave que correspondem às escolhas que podem ser selecionadas, a fim de efetuar tais deslocamentos.

Deve-se distribuir as funções nos menus de escolha conforme sua utilização pelo usuário, e não conforme menus e submenus de opção, sobre os quais os sistemas foram desenvolvidos.

Deve-se desenhar, abaixo do corpo do menu, um campo de entrada, para preenchimento da escolha, com 66 posições, que irá anteceder o campo de transmissão.

Sugere-se que seja dado o nome de ESCOLHA a este campo, para padronizar com o SADS e torná-lo condizente com o tipo do menu: menu de escolha.

Menu de operação

Padrões utilizados pelo SADS no desenho dos menus de operação, que são diferentes dos menus de escolha:

Deve-se desenhar somente os campos de argumentos que são necessários ao disparo da transação, da seguinte forma:

Um campo para informação da operação, caso deste menu possam ser disparadas mais de uma operação;

Os campos obrigatórios devem ser desenhados primeiramente, seguidos pelos opcionais;

Para os campos opcionais é desejável que sejam descritas quais as ocasiões em que devem ser preenchidos;

Para os campos que podem assumir vários valores, é conveniente indicar, na frente dos mesmos, quais são os valores possíveis de serem informados.

Existe um limite máximo de 90 campos de input por menu de operação.

Cadastramento de estrutura de tela menu

Apenas os menus de operação exigem o cadastramento de estrutura da tela.

Cadastramento e liberação de consistência de tela menu

Não é possível efetuar o cadastramento ou geração da consistência dos menus de escolha. Porém, caso haja necessidade, para menus de operação sua geração ou cadastramento pode ser efetuada normalmente. Estes procedimentos estão descritos na Seção II, "Elementos do repositório de dados do SADS", item "Consistência", tópico "Cadastramento".

Liberação de desenho de tela menu

Os desenhos dos menus devem ser liberados para um ambiente - teste (T) ou produção (P) - de acordo com a necessidade de utilização da navegação.

A liberação do desenho das telas dos menus de escolha ou de operação é semelhante à liberação de uma tela qualquer do sistema e está explicada na Seção II, "Elementos do repositório de dados do SADS", item "Tela", tópico "Liberação de desenho de tela".

Definição de contexto

Contexto é um agrupamento de menus de escolha e de operação que define uma visão das aplicações disponíveis para os usuários, podendo disparar transações de um ou mais sistemas.

As transações de um contexto são operacionalmente relacionadas e se destinam a um grupo de usuários que efetuam funções similares.

Após ter concluído o desenvolvimento de um sistema aplicativo, pode-se determinar então o contexto em que o mesmo será inserido.

As transações de um sistema aplicativo podem ser incluídas em um ou mais contextos já existentes e, também, pode-se criar um novo contexto para agrupar suas transações.

A adaptação de contextos já existentes para inclusão de novas transações deve ser feita através das seguintes funções:

Alteração dos desenhos dos menus de escolha incluindo as escolhas referentes as novas funções;

Definição das rotas das novas escolhas incluídas e

Definição das regras de conversão dos novos menus de operação.

Caso as funções do novo sistema não sejam necessárias em contextos já existentes, pode-se criar um contexto próprio, cuja definição será explicada a seguir.

Após o cadastramento dos elementos de um contexto, para utilizá-lo, deve-se informar no campo ACAO de um menu de qualquer contexto o seguinte comando: "Inicio ;<contexto>", onde <contexto> é o nome do contexto ao qual se quer executar o acesso. Pode-se, também, de uma tela desformatada transmitir "CONTEXTO <contexto>".

Para definir e alterar qualquer elemento de um determinado contexto, o usuário deve ter habilitação de elemento no Sistema de Segurança para tal contexto. O nome da lista de elementos a qual o usuário deverá ser habilitado é a "CONTEXT". Esta lista deverá conter, como elementos, o código de todos os contextos com os quais o usuário irá trabalhar, com o devido tipo de habilitação (só consultar, consultar e atualizar ou não trabalhar). Para maiores detalhes sobre o cadastramento desta habilitação de contextos, deve-se consultar a Seção VI, "Sistema de Segurança", item "Habilitação de usuários SADS", neste manual.

Definir um contexto é cadastrar um código, dar-lhe um nome, indicar a tela menu inicial e definir se será exigida segurança na sua utilização.

A definição do contexto deve ser efetuada antes dos cadastramentos dos títulos de telas, das rotas de navegação e das regras de conversão de layout, por ser um pré-requisito para tais definições.

CAMPOS DA TELA

TITULO DO CONTEXTO - Código do contexto.

O código do contexto é livre, devendo ter no máximo 09 caracteres.

NOME DO CONTEXTO - Descrição resumida do contexto.

TRANSACAO INICIAL - Código da tela inicial do contexto, no formato padrão de transação do SADS:

Os dois primeiros caracteres do identificador numérico da transação deve ser "00" ou um múltiplo de "10".

CONTEXTO COM SEGURANCA (S/N) - Indicador de exigência de segurança do SADS para utilização da navegação do contexto.

Pode assumir os seguintes valores:

"S" - Para obrigar o teste de habilitação do usuário que tentar utilizar a navegação do contexto.

Neste caso é exigido que o usuário efetue o logon no SADS.

"N" - Para não testar a habilitação do usuário que tentar utilizar a navegação do contexto.

Neste caso não é necessário que o usuário efetue o logon no SADS para navegar no contexto.

CONFIRME - Indicador de confirmação da operação sobre o contexto.

Pode assumir os seguintes valores:

"S" - Para confirmar o processamento da operação sobre o contexto;

"N" - Para desistir do processamento da operação sobre o contexto.

OPERACAO - Operação realizada sobre o contexto.

Pode assumir os seguintes valores:

"INC" (Inclusão);

"ALT" (Alteração);

"EXC" (Exclusão);

"CON" (Consulta).

A seguir, é apresentado um exemplo de definição de contexto de navegação:

Na operação de exclusão, será excluída a definição do contexto e, se houver, os títulos das telas menus e as rotas de navegação nele cadastrados; ou seja, na exclusão de um contexto, todos os demais cadastramentos a ele relacionados serão excluídos, exceto as regras de conversão de layout.

O contexto SADS já cadastrado não pode ser alterado ou excluído em hipótese alguma.

A inclusão de um contexto cria um código interno para o repositório de dados do SADS. Este código é formado por duas letras. O primeiro contexto incluído na instalação terá o código "AA", o próximo será o "AB" e assim sucessivamente.

Conhecer o código interno do contexto é importante para as operações de extração e carga de contexto, uma vez que um contexto pode englobar mais de um sistema. Não basta, portanto, extrair um sistema, como se faz convencionalmente.

A operação "LIS", do menu CONCONNAV (Consultas relativas a contextos de navegação), exibe a lista dos contextos cadastrados e apresenta o código interno de cada um deles. As rotinas de extração e carga de contexto de navegação, bem como as consultas relativas a contextos, estão descritas na Seção III, "Funções de apoio", neste manual.

Cadastramento de título de tela menu

Cadastrar títulos de telas menu é definir, para cada menu de escolha e de operação do contexto, os códigos pelos quais eles poderão ser identificados na ação de desvio ou na ação de comando.

O título da tela a ser cadastrado para um menu de escolha ou de operação deve ser o mesmo que se encontra identificado em seu desenho.

Uma mesma tela menu pode assumir títulos diferentes em contextos diferentes. Este, porém, não é um bom procedimento, uma vez que é importante que o título da tela apareça no seu desenho.

Deve ser escolhido um título adequado para o menu, de modo que seja fácil a sua memorização pelo usuário, para uma utilização mais rápida do sistema.

Sugere-se que este título seja formado em conjunto de três caracteres, com o máximo de nove, sendo que a regra de formação do título deve ser informada ao usuário para facilitar a utilização do sistema.

A seguir serão apresentados alguns exemplos de formação de títulos de menus:

Nome do menu - "Desenho de Tela";

Título do menu - "DESTEL".

Nome do menu -"Liberação de Desenho de Tela"

Título do menu - "LIBDESTEL".

Para cadastrar títulos de telas menu, é necessário que o contexto já esteja cadastrado no repositório de dados do SADS.

Cumpre ressaltar que, quando se solicita a operação de exclusão, todos os títulos referentes ao contexto informado serão excluídos. Para excluir apenas um título, deve-se então informar operação alteração (ALT) e limpar a linha referente ao título que se deseja excluir.

CAMPOS DA TELA

CONTEXTO - Código do contexto.

TRANSACAO - Código da transação à qual será associado o título.

Este código de transação tem de ser informado conforme a seguinte padronização do SADS:

Os dois primeiros caracteres do identificador numérico da transação têm de ser 00 ou múltiplo de 10.

TITULO DA TELA - Título com o qual o menu será conhecido, para utilização nas acões Desvio e Comando.

Sugere-se que este título seja formado em conjunto de três caracteres, com o máximo de nove, sendo que a regra de formação do título deve ser informada ao usuário para facilitar a utilização do sistema.

CONTINUA(S/N) - Indicador de continuidade de operação.

Pode assumir os seguintes valores:

"S" - Para prosseguir no cadastramento de títulos de tela para o contexto;

"N" - Para encerrar o cadastramento de títulos de tela para o contexto.

OPERACAO - Operação realizada sobre os títulos de telas.

Pode assumir os seguintes valores:

"INC" (Inclusão);

"ALT" (Alteração);

"EXC" (Exclusão);

"CON" (Consulta).

Cadastramento de rotas de navegação

Definir rotas de navegação é estabelecer quais palavras-chaves podem ser selecionadas em um determinado menu de escolha e que outro menu deverá ser apresentado quando a palavra-chave for escolhida.

Uma rota é, pois, um deslocamento que se pode efetuar a partir de uma escolha selecionada no menu.

Cada rota determina um destino que pode ser outro menu de escolha ou um menu de operação.

Um menu de destino só pode estar relacionado a um menu de origem, ou seja, não pode haver mais de uma rota em menus de origem diferentes com destino a um mesmo menu.

As rotas de navegação são sempre definidas a partir de um menu de escolha, sendo que este tipo de menu é utilizado apenas para navegação entre menus, não sendo possível disparar operações específicas a partir deles.

As escolhas disponíveis são indicadas no corpo do menu pelas palavras-chave, que são partes das palavras que representam as rotas desejadas, em letras maiúsculas.

Para definir as rotas de navegação de uma determinada tela menu, é necessário que seu título esteja cadastrado no repositório de dados do SADS para o contexto em questão.

É importante salientar que, se desejável que a ação "Origem" seja executada corretamente a partir da tela inicial de um contexto qualquer, retornando como origem a própria tela inicial, é necessário cadastrar a seguinte rota no contexto:

Título origem = Título da tela inicial do contexto;

Palavra-chave = Qualquer palavra, diferente apenas daquelas já cadastradas para esta tela inicial;

Título destino = Título da tela inicial do contexto.

Cumpre ressaltar que, quando se solicita a operação de exclusão, todas as rotas referentes à tela origem informada serão excluídas. Para excluir apenas uma rota, deve-se então informar operação alteração (ALT) e limpar a linha referente à rota que se deseja excluir.

CAMPOS DA TELA

CONTEXTO - Código do contexto.

TITULO ORIGEM - Título do menu de escolha de origem.

PALAVRA CHAVE - Código da palavra-chave.

Ao ser informada a palavra-chave no campo ESCOLHA da tela origem, será apresentado o menu correspondente ao título destino.

TITULO DESTINO - Título do menu de destino.

Um menu de destino só pode estar relacionado a um menu de origem, ou seja, não pode haver mais de uma rota em menus de origem diferentes com destino a um mesmo menu.

CONTINUA(S/N) - Indicador de continuidade de operação.

Pode assumir os seguintes valores:

"S" - Para prosseguir no cadastramento de rotas de navegação para o menu de escolha informado no menu;

"N" - Para encerrar o cadastramento de rotas de navegação para o menu de escolha informado no menu.

OPERACAO - Operação realizada sobre as rotas de navegação.

Pode assumir os seguintes valores:

"INC" (Inclusão);

"ALT" (Alteração);

"EXC" (Exclusão);

"CON" (Consulta).

Cadastramento de regras de conversão de layout

Regras de conversão de layout são mapeamentos que possibilitam a conversão dos dados informados nos menus de operação para o formato de um menu ou submenu de opção, a partir do qual uma transação pode efetivamente ser ativada.

A navegação, com seus menus de escolha e de operação, funciona como uma máscara para o usuário final. Internamente, os sistemas aplicativos trabalham da forma convencional, ou seja, através de menus de opção.

Sendo assim, toda vez que um menu de operação é disparado, os dados nele informados devem ser colocados na disposição dos campos do menu de opção, para que o programa que processa a transação possa executá-la.

A conversão dos dados entre os layouts dos menus de operação e dos menus de opção é executada automaticamente, toda vez que uma operação é disparada de um menu de operação. Porém, esta transferência só é possível devido ao fato de existirem regras de conversão cadastradas para cada menu de operação.

As regras de conversão são definidas apenas para os menus de operação, pois apenas esses menus disparam as transações vinculadas aos menus de opção.

Como as regras de conversão estão vinculadas às telas dos menus de operação, ao excluir um contexto que utilize estes menus, as regras não serão excluídas. Somente quando a tela do menu de operação tiver sua definição excluída do repositório de dados é que as regras relativas a ele serão também excluídas.

O tamanho total permitido para os atributos de entrada de um menu de operação, exceto o campo TLAACAO, é de 700 bytes. Caso este tamanho não seja respeitado, não será possível liberar as regras do mesmo.

Para cadastrar regras de conversão de layout, é necessário que a tela de menu de operação, sua estrutura e seu título estejam cadastrados no repositório de dados do SADS.

CAMPOS DA TELA

CODIGO DA TELA - Código da tela do menu de operação.

CODIGO DO MENU DESTINO - Código do menu ou submenu de opção correspondente ao menu de operação.

O código deste menu ou submenu deve ser informado no seguinte padrão:

Como exemplo, suponha que o menu de operação INCPED - Inclusão de Pedido - dispare uma transação do menu de opção PEDU01, submenu 20. O código de tela deste submenu é, pois, PEDU0120.

Neste caso, na definição desta rota, deverá ser informado "PEDU0120" no campo TRAN. RELACIONADA, para indicar que a transação será disparada através do submenu PED01.20.

SEQUENCIA - Número para posicionamento do teste na estrutura.

O primeiro número é "0010" e os demais, acrescidos de 10. Para incluir um teste entre dois outros já incluídos, é necessário preencher uma linha que esteja em branco e alterar o número desta linha para um número entre os já informados. O SADS reorganizará automaticamente a estrutura. Na próxima operação os testes já estarão na nova ordem.

SE/MV - Operação a ser efetuada com um campo do menu de operação.

Pode assumir os seguintes valores:

"SE" (Teste condicional SE) - Para testar se o conteúdo do campo do menu de operação é igual a uma constante;

"MV" (Mover) - Para mover uma constante ou o conteúdo de um campo do menu de operação para um campo do menu de opção.

TIPO - Tipo do campo informado em OPERANDO-1.

O tipo deve ser informado para o comando "MV".

Pode assumir o seguinte valor:

"K" - Quando o OPERANDO-1 for um valor constante.

Quando o tipo do operando não for informado, indica que o operando é um campo da estrutura do menu de operação que será movido.

OPERANDO-1 - Valor constante ou código de um atributo/campo da estrutura do menu de operação.

SUFIXO - Sufixo do OPERANDO-1.

Este sufixo deve ser especificado quando o campo informado no OPERANDO-1 aparecer mais de uma vez na estrutura do menu de operação e, por isso, for necessário diferenciá-los.

OPERADOR - Operador relacional a ser utilizado no teste condicional (opcional).

Este campo só deve ser informado quando for utilizado o teste condicional SE no campo SE/MV.

Este operador definirá o relacionamento entre o OPERANDO-1 e o OPERANDO-2.

Pode assumir os seguintes valores:

"EQ" (Equal) - Igual a;

"NE" (Not equal) - Diferente de;

"GT" (Greater than) - Maior que;

"GE" (Greater than or equal) - Maior que ou igual a;

"LT" (Less than) - Menor que;

"LE" (Less than or equal) - Menor que ou igual a.

OPERANDO-2 - Constante ou código de um atributo/campo da estrutura do menu de opção (opcional).

Pode assumir os seguintes valores:

<Uma constante> - Para comandos do tipo SE;

<Um atributo/campo de entrada do menu de opção> - Para comandos do tipo MV.

SUFIXO - Sufixo do OPERANDO-2.

O sufixo é utilizado para diferenciar um atributo/campo, quando aparecer mais de uma vez na estrutura do menu de opção.

CONTINUA(S/N) - Indicador de continuidade de operação.

Pode assumir os seguintes valores:

"S" - Para prosseguir no cadastramento das regras de conversão de layout para o menu de operação;

"N" - Para encerrar o cadastramento das regras de conversão de layout para o menu de operação.

OPERACAO - Operação realizada sobre as regras de conversão.

Pode assumir os seguintes valores:

"INC" (Inclusão);

"ALT" (Alteração);

"EXC" (Exclusão).

"CON" (Consulta);

Liberação de regras de conversão de layout

Liberar as regras de conversão de layout é torná-las disponíveis para uso nos ambientes de teste ou produção.

Para liberar as regras para teste, é necessário que as mesmas estejam cadastradas no repositório de dados do SADS, tenham sido incluídas ou modificadas, tenham sofrido alterações em suas estruturas e que o tamanho total dos atributos de entrada do menu de operação, exceto o campo TLAACAO, não seja superior a 700 bytes.

A liberação para produção só é possível para regras previamente liberadas para teste, o que torna a regra de conversão disponível para o ambiente de produção da mesma máquina onde ela foi liberada.

Após a liberação das regras de conversão de layout, está terminado o cadastramento da navegação SADS.

Caso o processamento dos sistemas em produção se dê em um computador diferente daquele onde foram cadastradas as regras, deve-se efetuar os seguintes procedimentos para utilização das regras já liberadas no outro computador:

Deve-se executar o job SD/D/JB/70 na máquina de desenvolvimento,

Copiar o arquivo SS/C/AF/71 para a máquina de produção e,

Na máquina de produção, executar o job SD/D/JB/72.

Estes jobs processam a extração e carga das regras de conversão de layout liberadas para produção, telas e consistências. Portanto, outros arquivos serão gerados e devem ser copiados para a máquina de produção, para garantir a completeza do processo.

A liberação de regras de conversão de layout não está relacionada a nenhuma tela de processamento. Deste modo, para executá-la, basta transmitir o menu de operação LIBNAV, informando nele os seguintes campos:

CAMPOS DA TELA

Contexto - Título do contexto para o qual serão liberadas as regras de conversão.

Titulo da tela - Título do menu de operação para o qual serão liberadas as regras de conversão.

Ambiente - Ambiente para o qual as regras serão liberadas.

Pode assumir os seguintes valores:

"T" - Para liberar as regras para o ambiente de teste;

"P" - Para liberar as regras para o ambiente de produção.

Será apresentada uma mensagem confirmando a liberação das regras.

Liberação de família de regras de conversão de layout

Liberar família de regras de conversão de layout é tornar todas as regras de um determinado contexto disponíveis para uso no ambiente especificado - teste ou produção. Esta operação elimina a necessidade de se ter que liberar, uma a uma, as regras de um determinado contexto.

Para liberar as regras para teste, é necessário que as mesmas estejam cadastradas no repositório de dados do SADS, tenham sido incluídas ou modificadas, tenham sofrido alterações em suas estruturas e que o tamanho total dos atributos de entrada do menu de operação, exceto o campo TLAACAO, não seja superior a 700 bytes.

A liberação de família de regras para produção só é possível para regras previamente liberadas para teste, o que torna a regra de conversão disponível para o ambiente de produção da mesma máquina onde ela foi liberada. Caso o processamento dos sistemas em produção se dê em um computador diferente daquele onde foram cadastradas as regras, deve-se executar alguns jobs de transferência, conforme descrito no tópico anterior , "Liberação de regras de conversão de layout".

Após todas regras de conversão de layout estarem liberadas, está terminado o cadastramento da navegação SADS.

Qualquer erro ocorrido na liberação de família de regras será armazenado, podendo ser verificado através da consulta automaticamente encadeada após a liberação. Deste modo, algumas regras do contexto especificado serão liberadas corretamente e as demais, que possuírem erro, não serão liberadas e terão a descrição do errro gravada para possibilitar a sua correção posterior. A consulta aos erros ocorridos poderá ser também disparada manualmente, a partir do menu de operação ERRLIB (Erros na liberação de família de regras).

A liberação de família de regras de conversão de layout não está relacionada a nenhuma tela de processamento. Deste modo, para executá-la, basta transmitir o menu de operação FAMNAV, informando nele os seguintes campos:

CAMPOS DA TELA

Contexto - Nome do contexto para o qual serão liberadas as regras de conversão.

Ambiente - Ambiente para o qual as regras serão liberadas.

Pode assumir os seguintes valores:

"T" - Para liberar as regras para o ambiente de teste;

"P" - Para liberar as regras para o ambiente de produção.

Será apresentada uma mensagem confirmando a liberação das regras.

Procedimentos de preparação para uso da navegação

Uma vez efetuados os cadastramentos relativos à navegação - menus de escolha, menus de operação, rotas e regras -, é necessário efetuar alguns procedimentos a fim de tornar a navegação disponível para uso, quais sejam:

Deve-se solicitar ao administrador do SADS da instalação o cadastramento, no Utility do COMS, dos trancodes que fazem parte do contexto. Esta carga pode ser feita utilizando-se do arquivo gerado via menu GERCOMHOS. Para maiores detalhes de como gerar o arquivo para carga no COMS, deve-se consultar a Seção III, "Funções de Apoio", neste manual.

O parâmetro Module Function Index, no Utility do COMS, dos trancodes relativos aos menus de escolha e de operação deve ser definido com o valor 10 ou 20, dependendo de estarem definidos para o ambiente de teste ou produção, respectivamente.

O host que tratará estes trancodes pode ser qualquer dos hosts do sistema que tratam os menus de opção. Não é conveniente relacionar os trancodes da navegação a hosts secundários.

Adequar a opção de compilação "SADSDESENV" aos programas chamados.

A navegação interpretativa do SADS não funciona com a opção de compilação SADSDESENV. Se for desejável construir a navegação específica para um sistema, seus programas devem ser compilados sem utilizar esta opção. Nesse caso, o ambiente de trabalho deve ser diferenciado pelo parâmetro Module Function Index do trancode no COMS, conforme explicado anteriormente.

Definir, se desejável, a tela inicial do contexto como sendo a tela a ser enviada ao usuário após o logon e/ou logoff.

Caso tenha sido cadastrada uma resposta após o logon, através de uma parametrização de resposta após o logon cadastrada especificamente para o usuário, em seu cadastro no Sistema de Segurança, esta será então a resposta a ser encadeada.

Após a criação de um contexto e a construção de sua navegação, o mesmo pode ser utilizado pelo administrador de segurança como item válido a ser cadastrado na tabela SBPM6004 - (Tabela de Controle de Resposta após Logon/Logoff). Nesta tabela são definidos os programas que serão executados ou as telas que serão apresentadas após o logon e logoff das estações.

Nesta tabela, a indicação "CONTEXTO <nome do contexto>" pode ser informada para as estações que se deseja que apareça a tela inicial do contexto após o logon e/ou logoff. Caso seja indicada apenas a palavra "CONTEXTO", será apresentada uma tela para o usuário escolher que contexto ele deseja utilizar.

Se a tabela possuir um registro para estação default, este deve ser alterado, caso se queira que para as estações não declaradas na tabela também seja apresentada a tela inicial do contexto.

A tela de escolha de contexto apresentada pelo SADS, após o logon, é a STKT0004. O administrador de segurança deverá alterar o desenho desta tela indicando todos os contextos da instalação.

Se a palavra "CONTEXTO" for informada na tabela SBPM6004 seguida do nome de um contexto, será apresentada na estação especificada a tela inicial do contexto informado.

Para maiores detalhes de como cadastrar a tabela SBPM6004, deve-se consultar a Seção V, "Parametrizações do SADS", item "Segurança", subitem "Resposta após <W0>logon/logoff".

Utilização do contexto definido

Caso o usuário já esteja trabalhando no SADS em um determinado contexto e deseja trocar de contexto, deve-se informar no campo ACAO o comando "Inicio <contexto>", onde <contexto> é o nome do contexto que se deseja utilizar.

Para entrar em um contexto, pode-se fazê-lo, também, através da transmissão do comando "CONTEXTO <nome do contexto>" de uma tela desformatada.

Caso o nome do contexto que se deseja utilizar não seja conhecido, pode-se solicitar a ação "D CONTEXTO" a fim de que seja apresentada a tela de seleção de contextos disponíveis (Tela STKT0004).

Funções de apoio

Específicas da navegação

Consulta erros na liberação de família de regras

Esta consulta apresentará uma lista com todos os erros ocorridos na liberação de família de regras de navegação de um contexto para um determinado ambiente.

Para efetuar esta consulta, é necessário que o contexto já tenha sido cadastrado no repositório de dados e que, após uma liberação de família de regras deste contexto, tenha ocorrido pelo menos um erro na liberação.

Argumentos da consulta

Para disparar esta consulta, os seguintes argumentos devem ser informados a partir da tela "ERRLIB":

Contexto - Código do contexto para o qual as regras de navegação foram liberadas.

Informações apresentadas na consulta

Na consulta "ERRLIB" (Erros na liberação de família de regras ), será apresentada a tela de processamento contendo as seguintes informações:

Contexto - Título e nome do contexto.

Titulo - Título da tela cuja regra não foi liberada devido a um erro ocorrido.

Código e descrição do erro ocorrido na tela com o título anterior.

Lista de contextos

Esta consulta apresentará uma lista de todos os contextos cadastrados no repositório de dados.

Argumento da consulta

Para disparar esta consulta, o seguinte argumento deve ser informado a partir da tela "CONCONNAV".

Tipo da consulta - "LIS", para disparar a consulta à lista de contextos cadastrados.

Informações apresentadas na consulta

Na consulta "LIS" - Lista de contextos cadastrados, será apresentada uma tela de processamento contendo as seguintes informações:

CONTEXTO - Título do contexto.

NOME - Nome do contexto.

TR. INICIAL - Código da transação inicial.

NOME INTERNO - Nome interno atribuído pelo SADS ao contexto.

Relação de títulos de telas

Esta consulta apresentará uma lista com os títulos de telas de menu de escolha e de operação cadastrados para um determinado contexto.

Para efetuar esta consulta é necessário que o contexto já tenha sido cadastrado no repositório de dados e exista pelo menos um título de tela para ele cadastrado.

Argumentos da consulta

Para disparar esta consulta, os seguintes argumentos devem ser informados a partir da tela CONCONNAV:

Tipo da consulta - "TIT" , para indicar que será efetuada a consulta aos títulos de telas cadastrados para o contexto.

Contexto - Código do contexto para os quais os títulos de telas foram cadastrados.

Informações apresentadas na consulta

Na consulta "TIT" - Relação de títulos de telas, será apresentada uma tela de processamento contendo as seguintes informações:

CONTEXTO - Título e nome do contexto.

TRANSACAO - Código da transação correspondente ao título.

TITULO - Títulos de telas ligados ao contexto.

Quando um título não estiver relacionado a nenhuma rota, aparecerá precedido de "*".

Relação de rotas de navegação

Esta consulta apresentará uma lista com as rotas de navegação cadastradas em um determinado contexto cadastrado no repositório de dados.

Para efetuar esta consulta, é necessário que o contexto já tenha sido cadastrado no repositório de dados e que exista pelo menos uma rota de navegação cadastrada para ele.

Argumentos da consulta

Para disparar esta consulta, os seguintes argumentos devem ser informados a partir da tela "CONCONNAV":

Tipo da consulta - "ROT", para indicar que será efetuada consulta às rotas de navegação cadastradas para o contexto.

Contexto - Código do contexto para o qual as rotas de navegação foram cadastradas.

Informações apresentadas na consulta

Na consulta "ROT" (Relação de rotas de navegação), será apresentada a tela de processamento contendo as seguintes informações:

CONTEXTO - Título e nome do contexto.

ORIGEM - Título do menu de origem.

PALAVRA CHAVE - Palavra-chave para apresentação do menu de destino, quando esta for informada no menu de origem.

DESTINO - Título do menu de destino.

TRANSACAO CHAMADA - Código da transação disparada, quando o menu informado no campo DESTINO for um menu de operação, ou seja, quando o menu informado no campo DESTINO disparar uma transação;

Este campo não será informado quando o menu apresentado no campo DESTINO for um menu de escolha.

Regras a serem liberadas para um ambiente

Esta consulta apresentará uma lista com as regras de navegação cadastradas em um determinado contexto que necessitam ser liberadas para um determinado ambiente em que ainda não estão disponíveis.

Para efetuar esta consulta, é necessário que o contexto já tenha sido cadastrado no repositório de dados e que exista pelo menos uma regra de navegação cadastrada nele.

Argumentos da consulta

Para disparar esta consulta, os seguintes argumentos devem ser informados a partir da tela "CONCONNAV":

Tipo da consulta - "LIB", para indicar que será efetuada consulta às regras de navegação que deverão ser liberadas para um determinado ambiente.

Contexto - Código do contexto para o qual as regras de navegação foram cadastradas.

Ambiente - Código do ambiente para o qual as regras deverão ser liberadas.

Informações apresentadas na consulta

Na consulta "LIB" (Regras a serem liberadas para um ambiente), será apresentada a tela de processamento contendo as seguintes informações:

Contexto - Título e nome do contexto.

Titulo/Transacao - Título da tela cuja regra deverá ser liberada.

Nome - Nome da tela cuja regra deverá ser liberada.

Programas relacionados ao menu de operação

Esta consulta apresentará uma lista com os programas relacionados aos menus existentes em regras de um determinado contexto cadastrado no repositório de dados.

Para efetuar esta consulta, é necessário que o contexto já tenha sido cadastrado no repositório de dados e que exista pelo menos uma regra de navegação cadastrada para ele.

Argumentos da consulta

Para disparar esta consulta, os seguintes argumentos devem ser informados a partir da tela "CONCONNAV".

Tipo da consulta - "PRO", para indicar que será efetuada consulta aos programas relacionados aos menus de operação das regras cadastradas para o contexto.

Contexto - Código do contexto para o qual as regras de navegação foram cadastradas.

Menu de operacao - Título do menu de operação, para o qual serão enviadas as informações. Caso não seja informado, serão considerados todos os menus de operação utilizados em regras do contexto.

Informações apresentadas na consulta

Na consulta "PRO" (Programas relacionados ao menu de operação), será apresentada a tela de processamento contendo as seguintes informações:

Contexto - Título e nome do contexto que contém os menus de operação.

Menu de Operacao - Título do menu de operação.

Transacao - Transação relacionada ao menu de operação.

Menu de Opcao - Código do menu de opção relacionado ao menu de operação.

Opcao - Opção utilizada nas regras.

Programa - Código do programa relacionado à regra utilizada.

Help através de SPFY

Pode-se obter acesso à descrição de uma tela de menu de escolha ou de operação através da tecla SPFY. Para isto, é necessário que a descrição do menu de escolha ou de operação tenha sido previamente cadastrada no repositório de dados do SADS.

Ressequenciamento de regras de conversão

Esta função permite efetuar o ressequenciamento das regras de conversão de navegação cadastradas para um determinado menu de operação.

Para efetuar o ressequenciamento, é necessário que as regras de navegação já estejam cadastradas.

Argumentos

Para disparar esta função, os seguintes argumentos devem ser informados a partir da tela "REGNAV":

 Operação - "RES", para indicar que será ressequenciada as regras de navegação.

 Contexto - Código do contexto no qual as regras de navegação foram cadastradas.

 Titulo da tela origem Código do menu de operação para o qual as regras de navegação estão cadastradas.
