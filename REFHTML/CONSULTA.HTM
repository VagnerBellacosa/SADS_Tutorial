Consultas gerais

O SADS possui um repositório de dados ativo, ou seja, que se mantém atualizado a qualquer fase do desenvolvimento dos sistemas.

Desta forma, para auxiliar nas manutenções e desenvolvimento dos sistemas, o SADS oferece uma grande variedade de consultas às informações cadastradas em seu repositório de dados, quais sejam:

Atributo nos sistemas;

Elementos por sistema;

Definição e validades de atributo;

Estrutura de dados;

Estrutura detalhada de dataset;

Descrição de elemento;

Fonte de programa/library;

Erros de compilação de programa;

Consultas relativas a contexto;

Impressão de consultas paginadas executadas;

Módulos de programas;

Arquivos de erros no usercode;

Elementos pendentes de geração/liberação para produção.

A seguir, cada uma destas consultas encontra-se detalhadamente explicada.

Consulta atributo nos sistemas

Um mnemônico de atributo, campo, sinônimo ou domínio pode estar cadastrado em vários sistemas, tendo em cada um deles características próprias.

Esta consulta é utilizada com objetivo de obter a relação de todos os sistemas em que determinado mnemônico está cadastrado, seu nome e sua descrição em cada um deles.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados".

Argumentos da consulta

Para que esta consulta seja utilizada, no menu de operação CONATRSIS deve ser informado o seguinte argumento:

 Atributo - Mnemônico do atributo, campo, sinônimo ou domínio; no padrão SADS.

A padronização utilizada pelo SADS para os mnemônicos dos atributos, campos, sinônimos ou domínios está explicada detalhadamente na Seção II, "Elementos do repositório de dados do SADS", nos itens "Atributo", "Campo", "Sinônimo" e "Domínio", respectivamente.

Informações exibidas na consulta

 SISTEMA - Código do sistema no qual o mnemônico está cadastrado.

 ATRIBUTO - Mnemônico do elemento.

 NOME - Descrição resumida do elemento.

Se o elemento consultado pertencer a um domínio, serão exibidos:

 DOMINIO - SISTEMA - ATRIBUTO - Código do sistema e mnemônico do domínio ao qual o elemento pertence.

Se o elemento consultado estiver relacionado a um sinônimo, serão exibidos:

 SINONIMO - SISTEMA - ATRIBUTO - Código do sistema e mnemônico do atributo ou campo do qual o elemento é sinônimo.

 DESCRICAO - Primeira página da descrição cadastrada para o mnemônico no sistema indicado na tela.

Consulta elementos por sistema

Esta consulta é utilizada para obter a relação de todos os elementos de um determinado tipo cadastrados em um sistema.

Podem ser consultados, por exemplo, os seguintes grupos de elementos: entidades, atributos, programas, relatórios etc.

A tela de processamento apresentada na consulta é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados".

Argumentos da consulta

Para que esta consulta seja utilizada, no menu de operação CONELESIS, deve ser informado o seguinte argumento:

 Elemento - Tipo do elemento ou o código completo do elemento, no padrão SADS. Se for informado o código completo do elemento, será obtida a relação de todos os elementos do tipo informado, a partir deste código.

A padronização utilizada pelo SADS para diferenciar os diversos tipos de elementos é "ssstt" onde:

 sss - Identificador alfabético do tipo do elemento

 tt - código alfabético que identifica o sistema

Informações exibidas na consulta

 SISTEMA - Código e nome do sistema.

<<<<Elemento>>>> - Código do elemento.

Dependendo do tipo do elemento a ser consultado, será apresentado, anteriormente ao seu código, um caractere para identificá-lo e/ou diferenciá-lo.

Os caracteres que precedem os elementos em cada consulta são:

 Consulta às entidades:

"S" - Se o elemento for uma chave secundária;

 Consulta às telas:

"E" - Se o elemento for uma tela de explicação;

 Consulta a programas on-line e programas batch:

"*" - Se o programa pertencer a uma versão anterior à 3.0 do SADS;

 Consulta a atributos:

"S" - Se o elemento for um sinônimo;

"D" - Se o elemento for um domínio;

"C" - Se o elemento for um campo.

 NOME - Nome do elemento.

 ULTIMA LIBERACAO/CODIGO EXTERNO/ SITUACAO/GERACAO - Informação dependente do tipo do elemento a ser consultado.

O cabeçalho deste campo é variável: na consulta aos programas batch e/ou on-line, é apresentado o campo ULTIMA LIBERACAO/GERACAO; na consulta aos atributos, o campo CODIGO EXTERNO; e nas demais consultas, o campo SITUACAO.

Nas consultas a programas batch e on-line por sistema, o SADS informa, no campo ULTIMA LIBERACAO, a data da última liberação do programa para produção ou não informa nada. Caso o programa ainda não tenha sido liberado, no campo GERACAO, é informado se o programa é gerado 100% pelo SADS.

Nas consultas aos atributos, o SADS informa no campo CODIGO EXTERNO o código externo cadastrado para o atributo no repositório de dados, caso ele o possua.

Nas consultas a arquivos convencionais, estruturas DMS, entidades, registros de uso geral e relatórios, o campo SITUACAO pode apresentar as seguintes informações:

 "CADASTR. INCOMPLETO" - Quando o número de registros estabelecido na definição do elemento for maior que o número de estruturas de registros já cadastrados;

 "A GERAR LIBRARY" - Quando o elemento ainda não teve sua library gerada ou então seu cadastramento atual está incompatível com a library existente;

 "LIBRARY EM TESTE" - Quando a library no ambiente de teste estiver compatível com o cadastramento do elemento no repositório de dados do SADS;

 "LIBRARY EM PRODUCAO" - Quando a library no ambiente de produção estiver compatível com o cadastramento do elemento no repositório de dados SADS.

Nas consultas às consistências gerais, o campo SITUACAO pode conter as seguintes informações:

 "A LIBERAR PARA TESTE" - Quando a consistência cadastrada no repositório de dados do SADS ainda não estiver disponível para o ambiente de testes;

 "A LIBERAR PARA PRODUCAO" - Quando a consistência cadastrada no repositório de dados do SADS ainda não estiver disponível para o ambiente de produção.

Nas consultas às tabelas, o campo SITUACAO pode conter as seguintes informações:

 "CADASTR. INCOMPLETO" - Quando apenas parte da tabela estiver definida no repositório de dados;

 "A GERAR LIBRARY" - Quando a tabela ainda não tiver sua library gerada ou seu cadastramento atual estiver incompatível com a library existente;

 "LIBRARY EM TESTE" - Quando a library no ambiente de teste estiver compatível com o cadastramento do elemento no repositório de dados do SADS;

 "LIBRARY EM PRODUCAO" - Quando a library no ambiente de produção estiver compatível com o cadastramento do elemento no repositório de dados SADS.

 "PARAMET.NAO CADAST." - Quando a tabela ainda não tiver seus parâmetros cadastrados no repositório de dados do SADS.

Sempre que o cadastramento do elemento estiver completo e correto, este campo aparecerá em branco.

No rodapé da última página é exibido o total de elementos, que apresenta a quantidade de elementos consultados.

Consulta definição e validades de atributo

Esta consulta é utilizada a fim de obter a relação de todas as definições cadastradas para um mnemônico em um determinado sistema.

O mnemônico pode ser de um atributo, campo, sinônimo ou domínio.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados".

Argumentos da consulta

Para que esta consulta seja utilizada, alguns argumentos devem ser informados no menu de operação CONDEFVAL, quais sejam:

 Sistema - Código do sistema do atributo, campo, sinônimo ou domínio; no padrão SADS.

 Atributo - Mnemônico do atributo, campo, sinônimo ou domínio.

A padronização utilizada pelo SADS para os mnemônicos dos atributos, campos, sinônimos e domínios está explicada detalhadamente na Seção II, "Elementos do repositório de dados do SADS", nos itens "Atributo", "Campo", "Sinônimo" e "Domínio", respectivamente.

Informações exibidas na consulta

 SISTEMA - Código do sistema do elemento.

 CODIGO - Mnemônico do elemento.

 NOME - Descrição resumida do elemento.

 FORMATO - Formato do elemento.

 SINTAXE - Sintaxe do elemento.

 TAMANHO - Tamanho do elemento.

Se o elemento consultado pertencer a um domínio, será apresentado:

 DOMINIO - Sistema e código do domínio do elemento.

Se o elemento consultado estiver relacionado a um sinônimo, será apresentado:

 SINONIMO DE - Sistema e mnemônico do elemento do qual ele é sinônimo.

Após consultar esta tela, tem-se as seguintes possibilidades:

 Se o elemento tiver descrição cadastrada, será apresentada a primeira tela de sua descrição;

 Se o elemento for uma variável definida, será apresentada uma tela com suas validades;

 Se o elemento for uma variável gerada, será apresentada uma tela com a definição de sua primeira fórmula cadastrada.

 Se existirem validades ou fórmula cadastradas para o elemento, estas serão exibidas logo após a apresentação de sua descrição.

Consulta estrutura de dados

Esta consulta é utilizada para obter a estrutura de dados de um elemento cadastrado no repositório do SADS.

Este elemento pode ser:

uma consistência geral;

uma entidade;

um registro de arquivo;

um registro de dataset;

um set de acesso a um dataset;

um registro de uso geral;

um registro de área de trabalho;

um registro da área de comunicação com sub-rotina;

um registro da área de work de um programa ou sub-rotina;

um registro da área conversacional;

uma linha de um relatório;

uma tabela;

uma tela.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados".

Argumento da consulta

Para que esta consulta seja utilizada, deve ser informado no menu de operação CONESTDAD o seguinte argumento:

 Estrutura - Código do elemento, no padrão SADS.

A padronização utilizada pelo SADS para os códigos de elementos, incluindo suas estruturas, é:

 "ssstxxxxxx" onde:

 "sss" é o código alfabético que identifica o sistema;

 "t" é o identificador do tipo do elemento e

 "xxxxxx" é o identificador do elemento e estrutura.

Exemplos:

"FACB05" - Dataset FACB05;

"FACB0500" - Dataset FACB05, registro 00;

"FACA5002" - Arquivo FACA50, registro 02;

"FACT0110" - Tela FACT0110.

O padrão para a estrutura de dados de uma linha de relatório deve ser informado conforme indicado abaixo:

 "sssRxnnnn" onde:

 "sss" é o código alfabético que identifica o sistema;

 "R" é a constante de elemento relatório e

 "xnn nn" é o identificador do relatório, um espaço em branco em branco e número da linha.

O identificador do relatório pode ser composto por dois ou três caracteres. Quando for composto por dois caracteres, o argumento deverá ter dois espaços em branco entre o identificador do relatório e o número da linha; caso contrário, apenas um espaço em branco, conforme os exemplos a seguir:

"FACR01 01" - Relatório FACR01, linha 01;

"FACR021 01" - Relatório FACR021, linha 01.

Informações exibidas na consulta

Nesta consulta são apresentados, no cabeçalho da tela, o tipo, o código e o nome do elemento consultado. A seguir, são exibidos os dados referentes a cada um dos atributos ou campos, que são ocorrências que compõem a estrutura especificada.

Os dados de cada atributo ou campo da estrutura são:

 NIVEL - Nível de estruturação do atributo ou campo.

 ATRIBUTO - Mnemônico do atributo ou campo.

 TAM - Tamanho do atributo ou campo na estrutura de dados.

 DEC - Número de casas decimais do atributo ou campo na estrutura de dados.

 SIN - Indicador de sinalização do atributo ou campo na estrutura de dados.

 FOR - Formato do atributo ou campo na estrutura de dados.

 PAD - Teste padrão do atributo ou campo na estrutura de dados.

 NOME - Nome do atributo ou campo.

Na consulta a datasets , quando forem utilizados nomes externos, serão apresentados os códigos externos dos atributos.

No rodapé da última página é exibido o total de atributos cadastrados na estrutura consultada.

Consulta estrutura detalhada de dataset

Esta consulta é utilizada para obter a estrutura de dados de um dataset cadastrado no repositório do SADS.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados".

Argumento da consulta

Para que esta consulta seja utilizada, no menu de operação CONESTDET, devem ser informados os seguintes argumentos:

 Dataset - Código do dataset no padrão SADS (sssBxx)

 Registro - Número do registro correspondente ao dataset solicitado. Se não informado, será apresentada a estrutura principal do dataset.

Informações exibidas na consulta

Nesta consulta são apresentados, no cabeçalho da tela, o código, o número do registro e o nome do dataset consultado. A seguir, são exibidos os dados referentes a cada um dos atributos ou campos, que são ocorrências que compõem a estrutura do dataset.

Os dados de cada atributo ou campo da estrutura são:

 Niv. - Nível de estruturação do atributo ou campo.

 Atributo - Mnemônico do atributo ou campo.

 Nome Externo - Código externo do atributo ou campo.

 Frm - Formato do atributo ou campo na estrutura de dados do dataset.

 Tam/Dec - Tamanho e número de casas decimais do atributo ou campo na estrutura de dados do dataset.

 Nome - Nome do atributo ou campo na estrutura de dados do dataset.

Os atributos que tiverem sido definidos como variável definida, terão suas validades apresentadas nesta consulta.

No rodapé da última página é exibido o total de atributos cadastrados na estrutura consultada.

Consulta descrição de elemento

Esta consulta é utilizada para obter a descrição de um elemento cadastrada no repositório de dados do SADS.

Este elemento pode ser um sistema, uma tela, um relatório, um arquivo, um registro de arquivo, um programa ou qualquer outro elemento que possua cadastramento no repositório de dados do SADS.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados".

Argumento da consulta

Para que esta consulta seja utilizada, no menu de operação CONDSCELE deve ser informado o seguinte argumento:

 Elemento - Código do elemento, no padrão SADS.

A padronização utilizada pelo SADS para elementos é:

 "ssstxxxxxx" onde:

 "sss" é o código alfabético que identifica o sistema;

 "t" é o tipo do elemento e

 "xxxxxx" é o diferenciador para elementos de mesmo tipo.

Exemplos:

"FACR01" - Relatório FACR01;

"FACR021" - Relatório FACR021;

"FACA5002" - Arquivo FACA50, registro 02;

"FACT0110" - Tela FACT0110.

Informações exibidas na consulta

 ELEMENTO - Código do elemento.

 NOME - Nome do elemento.

 DESCRICAO - Descrição do elemento.

Consulta fonte de programa/library

Esta consulta é utilizada a fim de se obter acesso ao código fonte de um programa ou de uma library qualquer: de tela, de área conversacional, de tabela, de DASDL etc.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados".

Argumentos da consulta

Para que esta consulta seja utilizada, alguns argumentos devem ser informados no menu de operação CONFON, quais sejam:

 Programa/library - Código do programa ou library, no padrão SADS.

A padronização utilizada para o fonte destes elementos é:

Fontes de programas on-line e batch, sub-rotinas e DASDL de banco:

"sssXEnnT" onde

"sss" é o código alfabético que identifica o sistema;

"X" é tipo do elemento ("N", "P", "S", ou "B", respectivamente;

"E" é a contante da fonte;

"nn" é o identificador do elemento e

"T" é a constante de DASDL de banco de dados em teste (opc.).

 Libraries de área de comunicação com sub-rotina, de área de work, de tela, de relatório, de registro de uso geral, de base de dados do sistema, de dataset, de parâmetros físicos de dataset, de tabela e de arquivo:

 "sssXLmmnn" onde:

"sss" é o código do sistema;

"X" é o tipo do elemento ("S", "P", "T", "R", "G", "B", "M", e "A", respectivamente);

"L" é a contante de library;

"mm" é o número sequencial e

"nn" é o número sequencial.

 Banco dados - Código do banco de dados que possui o dataset para o qual foram especificados os parâmetros físicos a serem consultados (sssBxx).

 Usercode - Usercode de análise do fonte a ser consultado.

 Ambiente - Código do ambiente do fonte (opcional).

 Label/sequencia - Label/seqüência do programa/library onde deve iniciar-se a consulta (opcional).

Se este campo não for informado, o SADS apresentará a consulta a partir do início do programa/library, exibindo todo o seu conteúdo.

Pode-se informar, por exemplo, o label PROCEDURE ou um número de seqüência do fonte, como "100032".

Informações exibidas na consulta

 CODIGO/NOME - Código e nome do programa ou library.

Serão exibidas telas com o código fonte do programa ou library.

 MARKID - Identificação do programa. Neste campo são identificadas as linhas de programa que foram geradas pelo SADS ou o nome do módulo em que elas estão cadastradas. Este campo aparece em branco para as linhas de programa que foram alteradas no Cande e para as linhas de libraries.

Consulta erros de compilação de programa

Esta consulta é utilizada para obter o arquivo de erros de compilação de um determinado programa ou sub-rotina.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados".

Argumentos da consulta

Para que esta consulta seja utilizada, alguns argumentos devem ser informados no menu de operação CONERRPRO, quais sejam:

 Programa - Código do programa ou sub-rotina, no padrão SADS.

A padronização utilizada pelo SADS para programas on-line e batch e sub-rotinas é:

 "ssstFnn" onde:

 "sss" é o código alfabético que identifica o sistema;

 "t" é o tipo do elemento ("N", "P" ou "S", respectivamente);

 "F" é a constante de fonte e

 "nn" é o identificador do elemento.

ou

 "ssstnn" onde:

 "sss" é o código que identifica o sistema;

 "t" é o tipo do elemento ("N", "P" ou "S", respectivamente);

 "nn" é o identificador do elemento.

Usercode - Usercode de análise do arquivo de erros a ser consultado.

Informações exibidas na consulta

No cabeçalho da tela serão apresentados o código e o nome do programa ou sub-rotina, seguidos da lista de erros de compilação do(a) mesmo(a).

Consultas relativas a contexto

Esta consulta é utilizada para obter a lista de contextos cadastrados no SADS, os títulos e as rotas cadastradas para um determinado contexto.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados".

Argumentos da consulta

Para que esta consulta seja utilizada, alguns argumentos devem ser informados no menu de operação CONRELCON, quais sejam:

 Tipo da consulta - Identificador do tipo da consulta, que pode ser:

 "LIS" - Para consultar a lista de contextos cadastrados no repositório de dados do SADS;

 "TIT" - Para consultar os títulos de telas cadastrados para um contexto;

 "ROT" - Para consultar as rotas de navegação cadastradas para um contexto.

 Contexto - Título do contexto (opcional).

Este campo deve ser informado apenas para as consultas "TIT" e "ROT".

Informações exibidas na consulta

Na consulta "LIS", será apresentada a tela de processamento do tipo consulta paginada contendo as seguintes informações:

 CONTEXTO - Título do contexto.

 NOME - Nome do contexto.

 TR.INICIAL - Código da transação inicial.

 NOME INTERNO - Nome interno atribuído pelo SADS ao contexto.

No rodapé da última página, será exibido o total de contextos com a quantidade de contextos consultados.

Na consulta "TIT", será apresentada a tela de processamento do tipo consulta paginada contendo as seguintes informações:

 CONTEXTO - Título e nome do contexto.

 TRANSACAO - Código da transação correspondente ao título.

 TITULO - Títulos de telas cadastrados no contexto.

Quando um título não estiver relacionado a nenhuma rota, aparecerá precedido de um "*".

Na consulta "ROT", será apresentada a tela de processamento do tipo consulta paginada contendo as seguintes informações:

 CONTEXTO - Título e nome do contexto.

 ORIGEM - Título da tela origem.

 PALAVRA CHAVE - Escolha que pode ser selecionada na tela origem.

 DESTINO - Título da tela obtida quando na tela informada no campo ORIGEM se faz a escolha informada no campo PALAVRA CHAVE;

 TRANSACAO CHAMADA - Código da transação chamada.

Esta informação pode conter:

 "<<Código da transação disparada>>" - Quando a tela informada no campo DESTINO disparar uma transação;

 <<brancos>> - Quando a tela informada no campo DESTINO for um menu de escolha.

Impressão de consultas paginadas executadas

Esta função é utilizada para imprimir a última consulta paginada efetuada na estação, no dia corrente.

Argumentos

Para que esta função seja utilizada, alguns argumentos devem ser informados no menu de operação IMPCONPAG, quais sejam:

 Primeira pagina - Número da primeira página consultada a ser impressa.

 Ultima pagina - Número da última página consultada a ser impressa.

Pode assumir os seguintes valores:

 "nn" - Para limitar a última página a ser impressa, onde "nn" é um número de uma página existente na consulta;

 "99" - Para imprimir até o fim da consulta, sem limitar o número de páginas.

 Impressora - Código da impressora.

Pode assumir os seguintes valores:

 <<Código da impressora remota>>" - Para listagem da consulta em impressora remota (esta impressora deve estar cadastrada na rede de terminais);

 "PRINTER" - Para listagem da consulta em impressora central, no formulário de 132 posições;

 "PRINxx" - Para listagem da consulta em impressora central e em formulário especial, onde "xx" é um formid definido para a sua instalação.

Caso este campo não seja informado, será assumida a impressora default cadastrada para o usuário no Sistema de Segurança. Caso esta não exista cadastrada, então é obrigatório informar o código da impressora no momento do disparo da impressão.

As impressoras remotas podem ser cadastradas com um formid correspondente, na tabela SBPM5026. Sempre que for disparada uma impressão pelo SADS, esta tabela é consultada e, caso a impressora esteja nela cadastrada, o relatório será impresso com o seu formid correspondente. A tabela SBPM5026 está descrita na Seção V, "Parametrizações do SADS", neste manual.

A impressão será imediatamente iniciada, sem apresentação de tela de processamento e, ao seu término, será apresentada mensagem confirmando a impressão da consulta.

Consulta módulos de programas

Esta consulta é utilizada para obter a lista de módulos de um programa, de uma grupo de programas ou de todos os programas de um sistema.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados".

Argumentos da consulta

Para que esta consulta seja utilizada, alguns argumentos devem ser informados no menu de operação CONMODPRO, quais sejam:

 Sistema - Código do sistema dos programas cujos módulos serão consultados;

 Identificador (N/P) - Identificador do tipo dos programas cujos módulos serão consultados. (Opcional).

Pode assumir os seguintes valores:

 "N" - Para consultar os módulos de todos os programas on-line do sistema; ou

 "P" - Para consultar os módulos de todos os programas batch do sistema.

Numero do programa - Identificador do programa cujos módulos serão consultados. (Opcional)

Informações exibidas na consulta

As informações a serem exibidas irá depender dos argumentos especificados, da seguinte forma:

 Se informado apenas o sistema (obrigatório), serão exibidos os módulos de todos os programas do sistema;

 Se informados o sistema e o identificador, serão exibidos os módulos de todos os programas do sistema que são do tipo especificado - on-line ou batch;

 Se informados o sistema, identificador e número do programa, serão exibidos apenas os módulos do programa especificado.

Para cada módulo apresentado, a consulta exibe as seguintes informações:

 PROGRAM/MOD - Códigos do programa e do módulo;

 NOME - Nome do módulo;

 ULTIMA ATUALIZACAO - Data da última atualização do programa.

Arquivos de erros no usercode

Esta consulta é utilizada a fim de obter a lista de todos os arquivos de erros de compilação/binder existente em um determinado usercode.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados".

Argumentos da consulta

Para que esta consulta seja utilizada, alguns argumentos devem ser informados no menu de operação CONARQERR, quais sejam:

 Usercode - Usercode no qual os arquivos de erros serão pesquisados;

 Sistema - Código do sistema cujos programas possuem arquivos de erros a serem consultados; (Opcional)

 Tipo - Identificador do tipo dos programas cujos arquivos de erros serão consultados. (Opcional).

Pode assumir os seguintes valores:

 "NF" - Para consultar os arquivos de erros de compilação dos programas on-line do sistema;

 "PF" - Para consultar os arquivos de erros de compilação dos programas batch do sistema;

 "SF" - Para consultar os arquivos de erros de compilação das sub-rotinas do sistema; ou

 "NC" - Para consultar os arquivos de erros de binder dos hosts do sistema.

Informações exibidas na consulta

As informações a serem exibidas irá depender dos argumentos especificados, da seguinte forma:

 Se informado apenas o usercode (obrigatório), serão exibidos todos os arquivos de erros nele residentes;

 Se informado o usercode e o sistema, serão exibidos todos os arquivos de erros de programas e sub-rotinas do sistema residentes no usercode especificado;

 Se informados o usercode, o sistema e o identificador, serão exibidos os arquivos de erros de todos os programas do sistema que são do tipo especificado - on-line ou batch e estiverem residente no usercode especificado.

Para cada arquivo de erros encontrado, a consulta exibe as seguintes informações:

 CODIGO DO ARQUIVO - Código do arquivo de erro;

 DATA DE CRIACAO - Data em que o arquivo foi criado;

 HORA DE CRIACAO - Hora em que o arquivo foi criado.

Elementos pendentes de geração/liberação para produção

Esta consulta é utilizada a fim de obter a lista de todos os elementos de um sistema que ainda não foram gerados ou liberados para produção.

A tela de processamento apresentada é do tipo consulta paginada e está descrita na Seção I, "Requisitos e conceitos para utilização do SADS", no item "Paginador de saída de dados".

Argumentos da consulta

Para que esta consulta seja utilizada, o seguinte argumento deve ser informado no menu de operação CONELEPEN:

 Usercode - Usercode no qual os arquivos de erros serão pesquisados;

 Elemento - Código do sistema cujos elementos com pendência de geração e/ou liberação serão consultados.

Caso desejável, pode-se informar o código de um elemento a partir do qual todos os demais elementos do sistema serão pesquisados quanto à pendência de geração e/ou liberação.

Informações exibidas na consulta

Para cada elemento com pendência de geração e/ou liberação, a consulta exibe as seguintes informações:

 Elemento - Código do elemento;

 Nome - Nome do elemento;

 Situacao - Situação de geração/liberação do respectivo elemento.

O campo SITUACAO pode apresentar as seguintes informações:

 "CADASTR. INCOMPLETO" - Quando o número de registros estabelecido na definição do elemento for maior que o número de estruturas de registros já cadastrados;

 "A GERAR LIBRARY" - Quando o elemento ainda não teve sua library gerada ou então seu cadastramento atual está incompatível com a library existente;

 "LIBRARY EM TESTE" - Quando a library no ambiente de teste estiver compatível com o cadastramento do elemento no repositório de dados do SADS;

 "LIBRARY EM PRODUCAO" - Quando a library no ambiente de produção estiver compatível com o cadastramento do elemento no repositório de dados SADS.

Nas consultas às consistências gerais, o campo SITUACAO pode conter as seguintes informações:

 "A LIBERAR PARA TESTE" - Quando a consistência cadastrada no repositório de dados do SADS ainda não estiver disponível para o ambiente de testes;

 "A LIBERAR PARA PRODUCAO" - Quando a consistência cadastrada no repositório de dados do SADS ainda não estiver disponível para o ambiente de produção.

Nas consultas às tabelas, o campo SITUACAO pode conter as seguintes informações:

 "CADASTR. INCOMPLETO" - Quando apenas parte da tabela estiver definida no repositório de dados;

 "A GERAR LIBRARY" - Quando a tabela ainda não tiver sua library gerada ou seu cadastramento atual estiver incompatível com a library existente;

 "LIBRARY EM TESTE" - Quando a library no ambiente de teste estiver compatível com o cadastramento do elemento no repositório de dados do SADS;

 "LIBRARY EM PRODUCAO" - Quando a library no ambiente de produção estiver compatível com o cadastramento do elemento no repositório de dados SADS.

 "PARAMET.NAO CADAST." - Quando a tabela ainda não tiver seus parâmetros cadastrados no repositório de dados do SADS.
