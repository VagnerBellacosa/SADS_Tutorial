Glossário

Ajuda on-line (help) - Função que o SADS oferece, em tempo de execução, para o esclarecimento de dúvidas e erros. A ajuda on-line está presente em todas as fases do desenvolvimento de sistemas, com a finalidade de agilizar o uso da ferramenta.

Análise de dados - Conjunto de técnicas, com abordagens próprias, que possibilitam a identificação e modelagem das informações e dos dados envolvidos no projeto de um sistema.

Os elementos da análise de dados englobam todos aqueles definidos quando da elaboração do Diagrama de Entidades e Relacionamentos Estendido (DER-E) - entidades, relacionamentos, hierarquias de generalização, atributos, sinônimos e domínios.

Análise estruturada ou análise funcional - Conjunto de técnicas que, com a utilização de ferramentas apropriadas - ferramentas gráficas e textos formais -, tem o objetivo de gerar a especificação funcional do sistema, composta de módulos que traduzem a realidade com uma visão essencialmente lógica, de um modo formal, não ambíguo e estruturado.

A análise estruturada fornece uma opção eficiente para o desenvolvimento de sistemas a partir do estudo das funções e processos envolvidos no mesmo.

Os elementos da análise funcional englobam todos aqueles definidos quando da elaboração do Diagrama Hierárquico-Funcional e do Diagrama de Fluxo de Dados - função/processo, fluxo de dados, entidade externa e depósito de dados.

Aplicativo (Aplicação) - Sistema que pode ser desenvolvido por usuários SADS com a finalidade de controlar um conjunto de dados e procedimentos. Também referenciado neste manual como sendo sistema aplicativo.

Área conversacional - Conjunto ou estrutura de atributos/campos definido em nível de sistema aplicativo com o objetivo de armazenar e transferir dados entre as transações de uma estação.

Área de comunicação da sub-rotina - Conjunto de atributos/campos que será utilizado para transferência de informações entre a sub-rotina e os programas que farão chamada à mesma. Esta área deve conter os parâmetros de entrada, fornecidos pelos programas, e os parâmetros de saída, fornecidos pela sub-rotina.

Área de trabalho (área de work) - Conjunto de atributos/campos que poderão ser utilizados na working-storage section de sub-rotinas, programas host, programas on-line e batch. Nos programas on-line, as áreas de trabalho podem assumir outras características, ou seja, podem ser utilizadas como área conversacional e área global, isto se a mesma library for utilizada no host no qual será feito o binder do programa.

Área global de comunicação - Área comum entre os programas on-line e o programa host a eles associado, que pode ser utilizada para transferência de informações entre os mesmos.

Área global de comunicação - Área comum entre os programas aplicativos e o Sistema de Segurança, onde as informações do usuário e suas habilitações ficam disponíveis para serem utilizadas pelos aplicativos.

Arquivo (convencional) - Arquivo é um conjunto de dados armazenados, geralmente de forma temporária, e que serão processados pelo sistema.

Atributo - Dado elementar que caracteriza, identifica, classifica ou qualifica uma entidade. No repositório de dados do SADS, o atributo é identificado pelo seu sistema e código (mnemônico).

Atributos globais - Atributos definidos como pertencentes ao SADS e que, por isso mesmo, podem ser utilizados por quaisquer sistemas aplicativos. Para utilizá-los, basta indicar que seu nível de definição é "G" (NDEF = "G").

Atributo de vídeo - Atributo utilizado para determinar a aparência do campo no desenho da tela. Ele é definido através de alguns comandos de teclado.

Atributo provisório - Atributo cadastrado na estrutura da entidade (com NDEF = "P") sem ter, porém, suas características definidas no repositório de dados.

Banco de dados - Conjunto de dados que se relacionam e que estão distribuídos em um ou mais arquivos, aos quais são permitidos acessos simultâneos por diferentes aplicações.

Banco de dados físico - Para o SADS, banco de dados físico é um banco composto por datasets.

Banco de dados lógico - Para o SADS, banco de dados lógico é um banco composto por remaps.

Base de dados - A base de dados compreende os bancos de dados e datasets a serem utilizados pelos hosts, programas batch e sub-rotinas de um sistema.

Uma base de dados pode ser genérica para todo o sistema, possuindo até dois bancos de dados, ou específica por elemento(s) do sistema, com até dez bancos de dados. Desta forma, os elementos de um sistema - hosts, programas batch e sub-rotinas - podem utilizar diferentes bases de dados, que podem ser específicas para cada um deles ou específicas para um grupo deles.

Binder - Processo através do qual diferentes objetos de programas são agregados a um programa host com a finalidade de formar um único objeto executável. Os programas on-line desenvolvidos no padrão SADS não funcionam isoladamente e, portanto, para se tornarem executáveis, devem ser agregados aos seus respectivos hosts através do binder.

Bloqueio de usuário - Impedimento a que o usuário efetue logon para abrir uma sessão de trabalho no SADS e, consequentemente, processe qualquer aplicativo dentro de uma sessão SADS.

Browser - O browser é um cliente para extração de informação em um servidor Web. Tipicamente, um browser será um programa, em um computador pessoal, que acessará, via Internet ou Intranet, um servidor (isto é, um programa que atende a demanda de clientes remotos) contendo informações de interesse amplo.

Campo - Dado elementar que caracteriza, identifica e qualifica uma estrutura qualquer, exceto a entidade. No repositório de dados do SADS, o campo é identificado pelo seu sistema e código.

Campo de transmissão - Último campo disponível da tela, utilizado para posicionamento do cursor no momento de envio das informações contidas na tela para processamento pelo sistema aplicativo. Normalmente, os sistemas aplicativos utilizam o atributo global "TLAESTAC" para denominar este campo na estrutura da tela.

Chave de acesso - Estrutura composta de atributos/campos que têm a função de permitir a rápida recuperação de registros em um determinado conjunto de dados e representar relacionamentos entre estes registros. As chaves de acesso aos datasets são comumente conhecidas como sets, subsets e access. Em se tratando de entidades e tabelas, também podem ser denominadas de chaves secundárias; quando não forem as principais chaves de acesso aos mesmos, passam a ser denominadas chaves secundárias.

Código externo - Código com o qual o elemento foi definido externamente ao SADS. O código externo pode ser utilizado quando houver integração entre o ET-SADS e o SADS, da seguinte forma: o código externo do elemento no SADS representa o código do elemento no ET-SADS. O código externo, para o SADS, é meramente documental.

Comandos de módulo - Macroinstruções no processo de cadastramento dos módulos de um programa. Os comandos de módulo, padronizados pelo SADS, têm o objetivo de fatorar o trabalho de complementação da lógica do programa e, por conseguinte, diminuir o tempo de construção do mesmo.

Consistência - Consistência geral, genérica ou simplesmente consistência é um módulo do SADS que executa a validação da entrada de dados para telas e tabelas de um sistema aplicativo de forma interativa e em tempo de execução. As consistências são de diferentes naturezas e são efetuadas a partir de parâmetros previamente cadastrados, eliminando a necessidade de suas definições e codificações em programas de um sistema aplicativo.

Contexto - Conjunto de menus e telas de processamento que engloba funções afins, ou um conjunto de telas quaisquer que o analista deseja agrupadas, que podem fazer parte de um ou mais sistemas.

COOPERATOR - Ferramenta desenvolvida pela MSA-Infor com o objetivo de permitir o compartilhamento dos recursos do micro e do mainframe. Trata-se de uma divisão de tarefas onde parte do processamento é feito com os recursos utilizados do micro e parte pelos do mainframe. O COOPERATOR foi desenvolvido, pois, para atender à necessidade de economia de recursos de mainframe, de manutenção das linhas de comunicação disponíveis e de obtenção de um melhor aproveitamento de recursos de hardware utilizados.

DASDL (Data and Structure Definition Language) - Linguagem para construção de banco de dados em DMSII.

Dataset - Coleção de registros de dados armazenados em um arquivo, manipulada pelo sistema gerenciador de banco de dados DMSII. O dataset é similar a um arquivo convencional; a diferença consiste na organização dos dados e nas formas de acesso.

Defaults globais de banco de dados - Valores assumidos pelos atributos em todas as estruturas do banco de dados que não declararem explicitamente algum valor para o mesmo.

Depósito de dados - Elemento do Diagrama de Fluxo de Dados que representa o local físico de armazenamento de informações do sistema.

Disco primário - Disco onde ficarão residentes os arquivos aos quais o sistema poderá executar o acesso e os que ele irá gerar.

Disco secundário - Disco no qual o sistema encontrará os arquivos necessários para execução e que não estão residentes no disco primário.

Domínio - Conjunto de características gerais a um grupo de atributos/campos e que geralmente engloba os valores ou intervalo de valores que estes poderão assumir. No repositório de dados do SADS, o domínio é identificado pelo seu sistema e código.

Elemento - Qualquer tipo de informação que pode ter seu acesso autorizado ou restringido ao usuário em uma aplicação, através do Sistema de Segurança. Pode ser hierarquizado e se referir a um conjunto de elementos ou se referir a um elemento específico, de acordo com a aplicação; podendo ser uma conta contábil para o sistema de contabilidade, um código de produto para um sistema de produção. Para o SADS, pode ser o código de uma tabela ou o código de um menu, etc.

Elementos do COMS - Informações básicas utilizadas pelo COMS para gerenciamento de transações on-line.

Entidade - Tudo aquilo que pode ser percebido no ambiente e do qual desejamos obter informações; tudo aquilo que faz parte da realidade, podendo ser de natureza concreta, como, por exemplo, objetos (equipamento, material) ou pessoas (fornecedor, funcionário), ou de natureza abstrata (curso, profissão).

Entidade derivada de relacionamento - Entidade resultante da associação de duas ou mais entidades no modelo conceitual de dados e que contém informações decorrentes desta associação.

Entidade externa - Entidade proveniente de fora dos limites de um sistema, podendo ser uma pessoa, um departamento, um sistema automatizado, uma organização externa e similares, desde que estes se comuniquem com o sistema, enviando e/ou recebendo dados.

Entidade meramente conceitual - Entidade que aparece apenas no modelo conceitual do sistema, não sendo transposta para o modelo lógico ou físico do mesmo. Deste modo, ela é apenas documental.

Estação - Periférico ou terminal destinado à utilização pelo usuário final. Através de uma estação o usuário envia mensagens para processamento no computador central ou dele recebe mensagem.

Estação habilitada - Terminal da instalação que pode ser utilizado por um usuário para executar o acesso a um aplicativo.

Estado da tela/tabela - Elemento responsável por identificar para qual ambiente e, consequentemente, para quais programas aplicativos a tela ou tabela estará disponível para utilização. Existem quatro estados distintos - "D" (desenvolvimento), "T" (teste), "P" (produção) e "M" (manutenção).

Estrutura - Conjunto de atributos/campos que compõem e caracterizam o elemento.

A estrutura de uma entidade é composta por todos os atributos que a caracterizam, identificam ou classificam.

A estrutura de uma tela compreende os atributos/campos que compõem a parte variável da mesma, ou seja, os campos de input e output.

A estrutura de uma consistência compreende os atributos/campos que são captados do usuário pelo programa aplicativo, ou seja, somente os campos de input.

A estrutura de um arquivo, dataset, set e tabela compreende os atributos/campos que compõem um registro de ocorrência dos mesmos.

A estrutura de um área de work de programa, área conversacional e área de comunicação com sub-rotina compreende os atributos/campos que serão referenciados pelos programas para utilização dos mesmos.

ET-SADS - Ferramenta Case desenvolvida pela MSA-Infor que cobre toda a fase de análise de um sistema. Automatiza e estimula a utilização de metodologias estruturadas da Engenharia da Informação.

Fluxo de dados - Caminho pelo qual as informações fluem no sistema, retratando as interfaces entre os componentes do DFD. Os sentidos possíveis são: entre processos, entre depósitos de dados e processos e entre entidades externas e processos.

Formato - Indicador do tipo do conteúdo do atributo/campo, que pode ser: "A" (alfabético), "B" (boolean), "D" (display), "N" (numérico), "R" (real) e "F" (field).

Fórmula - Sequência de operações matemáticas para se alcançar o valor do atributo/campo. A fórmula de um atributo/campo é identificada pelo sistema e mnemônico do atributo/campo acrescido de um sequencial que representa o número da fórmula do atributo/campo, que pode possuir mais de uma fórmula.

Função ou processo - Conjunto de procedimentos que atua na transformação dos dados, no Diagrama de Fluxo de Dados e no Diagrama Hierárquico-funcional.

Header padrão de dataset - Conjunto de atributos/campos definidos na tabela SBPM8014 e que opcionalmente podem ser incorporados ao header do dataset, na geração de sua DASDL (Data and Structure Definition Language).

Hierarquia de generalização - Hierarquia de generalização, ou simplesmente generalização, é uma abstração que permite representar uma classe de objetos similares como um objeto genérico. O objeto genérico é denominado superentidade (entidade "pai") e os objetos similares são denominados subentidades (entidades "filhas"). A entidade genérica possui informações comuns a todas as subentidades e estas, apenas suas informações específicas.

Host - Programa responsável pelo gerenciamento de transações; executa funções genéricas do sistema, recebendo e distribuindo as transações para os programas, cujos objetos lhe são agregados através do processo de binder.

Host físico - Host que processa as transações após o desenvolvimento e implantação do sistema. Neste caso, o host gerencia transações que realmente fazem acesso às bases de dados do sistema e não apenas simulam sua operação.

Host lógico - Host utilizado na etapa de prototipação do sistema. Ele gerencia um sistema on-line quando este ainda não possui programas e a base de dados ainda não foi construída. A partir de menus, telas e regras previamente cadastradas, simula a execução do sistema, dando a idéia de como será sua operação após o desenvolvimento.

Host primário - Host encarregado de controlar e disparar todas as transações pertencentes aos menus do sistema a ele relacionado. É aconselhável que se definam como agregadas diretamente ao host primário apenas as transações menos pesadas do sistema, ou seja, aquelas transações que não consomem muito tempo de processamento. Sendo assim, as demais transações ficariam agregadas ao host secundário, que, por sua vez, seria disparado pelo host primário.

As transações gerenciadas diretamente pelo host primário estão associadas aos menus de código "sssUnn".

Host secundário - Host disparado pelo primário com o objetivo de gerenciar as transações mais pesadas do sistema, ou seja, aquelas que consomem maior tempo de processamento. Sendo assim, é aliviado o processamento do host primário, evitando-se enfileiramentos, de modo a garantir um bom tempo de resposta ao usuário final.

As transações gerenciadas pelo host secundário estão associadas aos menus de código "sssULn".

HTML - Acrônimo de Hypertext Markup Language, é a linguagem padrão para escrever páginas de documentos Web, que possibilita preparar documentos com gráficos e links para outros documentos, para visualização em browsers.

Input - Palavra em inglês que, traduzida para o português, significa "entrada".

Integração ET-SADS x SADS - Processo que permite a complementação entre as fases de um projeto de sistema desenvolvidas no ET-SADS com aquelas desenvolvidas no SADS. Após a elaboração do modelo conceitual no ET-SADS, transferem-se os dados para o mainframe e a implementação dos processos do sistema é feita utilizando-se o SADS.

Interface - Relação ou ligação entre dois módulos de software.

Job - Conjunto de comandos em linguagem WFL, responsável por disparar a execução de programas batch. O job é também conhecido como rotina batch.

Library - Arquivo fonte Cobol contendo os atributos/campos que compõem a estrutura de um elemento. A library de um elemento é gerada completamente pelo SADS, podendo ser incorporada aos programas aplicativos que o utilizam. Como exceção, a library de declaração de banco contém também comandos de open e de close do banco.

Internet -

1. Com inicial maiúscula, significa a "rede das redes", originalmente criada nos EUA, que se tornou uma associação mundial de redes interligadas, que utilizam protocolos da família TCP/IP. A Internet provê transferência de arquivos, login remoto, correio eletrônico, news e outros serviços;

2. Com inicial minúscula, genericamente, significa uma coleção de redes locais e/ou de longa distancia, interligadas por pontes, roteadores e/ou gateways.

A Internet utiliza um modelo de computação denominado Cliente/Servidor, que permite o uso eficiente dos recursos disponíveis em cada máquina.

As aplicações deste modelo reduzem o tráfego na rede através da otimização na troca de dados entre clientes e servidores. O servidor cuida do armazenamento e da alteração de grandes coleções de dados, controla o acesso à informação e processa pedidos, enquanto o cliente (um PC, por exemplo) se encarrega de fazer o que for necessário para que a aparência e a interação com a informação seja a melhor possível.

Intranet - É o termo descritivo usado para a implentação das tecnologias da Internet dentro de uma corporação, ao invés de uma conexão externa com a Internet. Essa implementação proporciona, de forma tranparente, a disponibilização de imensos recursos informativos de uma organização para todos os seus computadores, com um custo e tempo mínimos de implementação.

Linha modelo - Linha de relatório cujo desenho já está cadastrado no repositório de dados e que, por isso mesmo, será utilizado como referência para o cadastramento de uma segunda linha qualquer do mesmo relatório.

Link - Conjunto de procedimentos executados pelo sistema operacional (MCP), com o objetivo de permitir que um programa aplicativo e as sub-rotinas por ele chamadas compartilhem fisicamente a área definida como de comunicação.

Lógica de programa - Especificação mais detalhada do tipo de processamento executado por um programa. Existe uma grande possibilidade de cadastramento de lógicas de programas para cada um dos tipos de lógica existentes. Cada uma dessas lógicas irá consistir em uma sublógica dentro de seu tipo.

Logoff - Encerramento da sessão de trabalho do SADS. Ao finalizar o seu trabalho, todo usuário deve encerrar a sessão SADS através do procedimento do logoff, por questões de segurança. Isto obrigará o próximo usuário que for trabalhar na mesma estação a efetuar um novo logon.

Logon - Habilitação para uso do SADS. Para iniciar uma sessão de trabalho no SADS, é necessário cumprir o procedimento de habilitação. Uma vez cumprido este procedimento, o usuário estará habilitado a utilizar os aplicativos para os quais foi autorizado.

Máscara de edição - A máscara ou picture de edição define a forma de edição do atributo/campo, nos mesmos padrões adotados na linguagem Cobol.

Menu - Para o usuário final, um menu representa uma tela que exibe determinadas operações ou escolhas que, ao serem selecionadas, ativam transações ou outros menus. Para o cadastramento no repositório do SADS, um menu representa um agrupamento de opções ou transações onde serão definidas as características dos processamentos a serem executados pelo host a cada opção (transação); daí esse menu ser denominado menu de opção. Existem quatro tipos de classificações para os menus, de acordo com o SADS - menu de escolha, de operação, de opção e secundário.

Menu de escolha - Tela que define as rotas de navegação, sendo que cada rota corresponde a um destino que pode ser outro menu de escolha ou um menu de operação. Portanto, um menu de escolha é utilizado apenas para navegação entre menus, não sendo possível solicitar operações específicas a partir dele.

Menu de opção - Agrupamento de opções ou transações onde serão definidas as características dos processamentos a serem executados pelo host a cada opção (transação). Sendo assim, um menu de opção é definido através de sua tela e através de seu cadastramento no repositório de dados do SADS. A tela do menu irá apresentar as opções disponíveis para o usuário e o cadastramento no repositório de dados irá definir as características vinculadas a cada uma destas opções. As telas dos menus de opção não são apresentadas pela navegação do SADS.

Menu de operação - Tela que define as operações e os dados básicos a serem informados para processamento das transações. A cada menu de operação pode estar associada nenhuma, uma ou mais de uma tela de processamento.

Menu secundário - Menu onde estão cadastradas as transações do sistema que estão relacionadas ao host secundário. O menu secundário possui padrão "sssULn".

Módulo de processamento - Conjunto de macroinstruções cadastradas no SADS que têm como objetivo moldar a geração de um programa às necessidades do sistema, incluindo, complementando ou substituindo trechos do fonte que seriam normalmente gerados pelo SADS.

Módulo do SADS - Cada módulo do SADS é uma unidade distinta de gerenciamento de elemento do SADS, com funções nitidamente destinadas ao controle deste elemento, que pode ser composta por um ou mais sistemas. Exemplos de módulos do SADS: Gerenciador de telas, Gerenciador de tabelas, Consistência geral, Controlador de bibliotecas, etc.

Navegação - Processo de deslocamento entre menus do sistema, a fim de facilitar a sua utilização. É realizada através de menus de escolha, menus de operação e telas de processamento, que ficam agrupadas em contextos de navegação.

Nível de acesso geral do usuário - Nível especificado no momento do cadastramento do usuário e que determina o seu poder de acesso ao Sistema de Segurança e demais sistemas desenvolvidos pelo SADS.

Nível de acesso do usuário por aplicativo - Nível especificado no momento de habilitação do aplicativo ao usuário e que determina seu poder de acesso a este aplicativo. Informação disponível na área global de comunicação para ser tratada pelos aplicativos.

Nível de definição (NDEF) - O nível de definição do atributo/campo na estrutura de um elemento especifica em qual sistema, relativamente ao elemento, ele foi definido. Pode assumir os seguintes valores: " " (atributo/campo definido como item grupo na estrutura), "S" (atributo/campo definido no sistema indicado no campo SIS - caso o campo SIS não seja informado, o sistema é o mesmo do elemento) e "P" (atributo/campo não definido no repositório de dados).

Nível de estruturação (NIVEL) - Identificação da hierarquia do atributo/campo dentro da estrutura de um elemento. O nível de estruturação de um atributo/campo é análogo ao nível do Cobol.

Opções globais de banco de dados - Opções responsáveis pelo controle das características das rotinas de acesso ao banco de dados, em tempo de compilação e execução.

Operador aritmético - Operador utilizado para concatenar os operandos de uma expressão matemática. No SADS, é utilizado no cadastramento das fórmulas de atributos/campos que sejam variáveis geradas (sintaxe = "VG") e no cadastramento em nível de estrutura das consistências genéricas. São os seguintes os operadores aritméticos: "+" (adição), "-" (subtração), "*" (multiplicação) e "/" (divisão).

Operador lógico - Operador utilizado para concatenar logicamente as operações. No SADS, é utilizado no cadastramento das validades de atributos/campos que sejam variáveis definidas (sintaxe = "VD") e no cadastramento das validades em nível de estrutura das consistências genéricas. São os seguintes os operadores lógicos: "E" (e - and) e "O" (ou - or).

Operador relacional - Operador utilizado para comparar os operandos. No SADS, é utilizado no cadastramento de variáveis definidas (sintaxe = "VD"), no cadastramento da validação em nível de estrutura das consistências e na definição de condição de subset automático. São os seguintes os operadores relacionais: "LE" (Less or equal - Menor que ou igual a), "LT" (Less than - Menor que), "GE" (Greater or equal - Maior que ou igual a), "GT" (Greater than - Maior que), "EQ" (Equal - Igual a) e "NE" (Not equal - Diferente de).

Ordenação de programa - Especificação de quais e em qual sequência os campos de um arquivo definido como recurso do programa serão utilizados na sua classificação.

Output - Palavra em inglês que, traduzida para o português, significa "saída".

Padrão - Teste padrão ao qual o atributo/campo a ele relacionado será submetido para validação de seu conteúdo, na consistência dos dados de uma tela ou tabela.

Paginador de entrada de dados - Módulo do SADS responsável por agilizar a entrada de dados para os programas on-line que utilizam múltiplas telas.

Paginador de saída de dados (paginador de consulta) - Módulo do SADS responsável por agilizar a programação de sistemas que utilizam consultas de múltiplas páginas em aplicações on-line.

Parâmetros físicos de dataset - Características que o dataset ou suas chaves de acesso assumirão no momento da compilação da DASDL do banco de dados.

Parâmetros globais de banco de dados - Elementos responsáveis pelo controle das características das rotinas de acesso de banco de dados em tempo de compilação e execução.

Perfil modelo - Usuário ao qual podem ser vinculados outros usuários que assumirão suas habilitações. Facilita a manutenção do cadastro de usuários, de maneira a evitar o cadastramento repetitivo.

Processamento Cooperativo - Divisão de tarefas onde parte do processamento é feito com os recurso utilizados do micro e parte pelos de mainframe. V. Cooperator.

Processo ou função - Conjunto de procedimentos que atua na transformação dos dados, no Diagrama de Fluxo de Dados e Diagrama Hierárquico-funcional.

Programa batch - Programa que processa um grande número de informações, em lotes de dados, sem interação direta com o usuário e disparado através de jobs (WFL).

Programa on-line - Programa que processa pequena quantidade de informações, com interação direta com o usuário através de telas. As informações são processadas ao mesmo tempo que o usuário requisita a execução do programa através de transações (transacional) ou através do disparo direto do programa (conversacional).

Prototipação - Fase do projeto lógico de desenvolvimento de sistemas, no qual, a partir de menus, telas e regras previamente cadastradas, o host simula a execução de um sistema, dando a idéia de como será sua operação após seu desenvolvimento, embora ainda não possua programas e a base de dados ainda não tenha sido construída.

Quebra de programa - Especificação de quais campos de um arquivo servirão como marcos na divisão do processamento de um programa em loops.

Recurso de programa - Elemento (datasets, telas, sub-rotinas, etc) que fará parte ou será utilizado no processamento de um programa.

Referências cruzadas - Consultas que permitem a verificação de quais elementos do repositório de dados se relacionam com o argumento de pesquisa.

Registro de uso geral ou dataset de uso geral - Dataset cujos registros são utilizados por vários sistemas para armazenamento de informações de controle. Sendo este dataset de formato variável, cada sistema deverá definir uma máscara para os seus dados nele armazenados.

Relacionamento - Associação feita entre ocorrências de uma ou mais entidades.

Relatório - Conjunto de caracteres ordenados de forma significativa, gerado por programas on-line ou batch para apresentação de dados em formulários de papel via impressoras remotas ou centrais.

Relatório batch - Relatório emitido por programa batch.

Relatório on-line - Relatório emitido por programa on-line.

Relatório remoto - Relatório emitido por programa on-line nas impressoras remotas.

Remap - Visão lógica de um dataset já definido.

Repositório de dados - Local físico onde são armazenadas as características dos dados e suas relações com outros dados, informações estas utilizadas pelos sistemas. O repositório de dados é o módulo básico de infra-estrutura do SADS, que integra e apóia os demais, permitindo o registro de todos os elementos dos Sistemas de Informação, de forma centralizada, com total controle e informação sobre a utilização dos mesmos. Ele não só é utilizado no desenvolvimento de sistemas aplicativos, como também na execução dos mesmos, encarregando-se das funções de help, consistência geral, leitura de telas, etc.

Rota - Conjunto de informações que determinam o caminho para se chegar a uma transação específica a partir da escolha de uma das opções apresentadas em um menu. A rota de navegação define, para determinado menu de escolha, as palavras-chave que podem ser selecionadas e as telas que serão apresentadas após a transmissão de cada uma delas.

SAA (System Application Architecture) - Conjunto de interfaces, convenções e protocolos propostos pela IBM que fornece padrões amigáveis para o desenvolvimento e a utilização de aplicativos em diferentes ambientes e plataformas.

SADS - Sistema de Automação do Desenvolvimento de Sistemas - Conjunto de ferramentas desenvolvido pela MSA-Infor com o objetivo de auxiliar nas diversas fases da Engenharia da Informação, desde o planejamento até a manutenção de sistemas aplicativos em geral.

SADS/SCP - Sistema de Planejamento e Controle de Projetos - Módulo integrante do SADS que oferece os recursos necessários à administração de mão-de-obra alocada ao desenvolvimento e manutenção dos sistemas. Este módulo apóia as gerências na administração e execução de projetos, a partir de um planejamento elaborado, permitindo o acompanhamento individualizado de todas as atividades.

SADS/SPO - Sistema de Controle do Processamento - Módulo do SADS que automatiza as atividades de planejamento e controla a produção do CPD. Aplicável ao ambiente batch, este módulo em si é de natureza on-line, possibilitando que todo ciclo dos serviços seja ativado de forma interativa, desde o registro no sistema até a expedição dos relatórios para os usuários.

SADS/SS ou SBT - Sistema de Segurança - Módulo do SADS que garante a segurança de acesso às informações da empresa, permitindo o uso de transações on-line somente às pessoas habilitadas. O controle é feito através de senhas individuais e o sistema exige que cada usuário se identifique antes de iniciar uma sessão de trabalho.

SADS/STE - Sistema de Estatísticas do Ambiente On-Line - Módulo do SADS encarregado de fornecer um efetivo controle de todas as variáveis envolvidas em ambientes de TP. Ele atua automaticamente na captação de informações geradas pelos COMS e Cande, possibilitando o conhecimento do número de transações e seus respectivos tempos de resposta, sob as mais variadas visões.

Senha - Código individual para identificação do usuário SADS no momento do logon. A senha deve ser de conhecimento restrito ao seu usuário, para que o acesso ao ambiente SADS seja controlado. No Sistema de Segurança do SADS, a senha pode ter até seis caracteres alfanuméricos, com pelo menos uma letra e um número, não podendo, portanto, possuir caracteres especiais.

Senha criptografada - Campo que guarda as mesmas informações contidas na senha do usuário, porém de forma "mascarada", para não permitir seu conhecimento até mesmo pelo administrador de segurança.

Serviço - Job preparado para execução ou em execução. A cada vez que um job que esteja cadastrado no SPO - job fonte - é preparado para execução, ele passa a ser considerado um serviço.

Servidor

1. No modelo cliente/servidor, é o programa responsável pelo atendimento a determinado serviço solicitado por um cliente. Serviços como archie, Gopher, WAIS e WWW são providos por servidores;

2. Referindo-se a equipamento, o servidor é um sistema que provê recursos tais como armazenamento de dados, impressão e acesso dial-up para usuários de uma rede de computadores.

Sessão SADS - Período de trabalho iniciado pela operação de logon do SADS e encerrado pela operação de logoff, que permite ao usuário trabalhar com o SADS ou qualquer aplicativo gerado por ele.

Sinônimo - Atributo/campo definido a partir das características já cadastradas para um outro atributo/campo ao qual ele está relacionado.

Sintaxe - Tipo da variável do atributo/campo. Pode assumir um dos valores: "VD" (variável definida), "VG" (variável gerada), "VE" (variável estruturada) e "VL" (variável livre).

Sistema - Coleção de elementos discretos, interdependentes, que se interagem com o objetivo de atingir uma causa ou meta comum. De maneira genérica, pode ser entendido como um conjunto de processos, dados e estruturas de dados.

Para o SADS, sistema é o elemento de nível mais elevado e sob o qual estarão definidos os demais elementos dos modelos de dados e processos, sejam eles conceituais, lógicos ou físicos.

Sistema aplicativo - Sistema que pode ser desenvolvido por usuários SADS com a finalidade de controlar um conjunto de dados e procedimentos. Também referenciado neste manual como aplicativo ou aplicação.

Sistema cooperativo - Sistema no qual o processamento é dividido entre o computador central e os microcomputadores. Para o SADS, um sistema cooperativo é aquele no qual as máscaras das telas, regras de validação dos dados e navegação das telas são executadas no microcomputador e apenas a execução do processamento da lógica em si é executada no computador central.

Site [instalação]

1. Uma instituição, onde computadores são instalados e operados;

2. Um nó Internet.

Softwares de banco de dados - Rotinas responsáveis pelo gerenciamento lógico e físico do banco de dados, pela recuperação do banco, pelas funções específicas de DMSII e pela recuperação em audit e control file, rebuild ou rollback. Esses softwares são comumente conhecidos como accesroutines, datarecovery, dmsupport, reconstruct e recovery.

Subentidade - Entidade que em uma hierarquia de generalização é subordinada a uma superentidade. Pode também ser denominada entidade "filha", possuindo informações específicas e herdando todas as classificações da superentidade à qual está relacionada.

Submenu - Tipo especial de menu contido hierarquicamente dentro de um outro menu e que possui as mesmas características deste.

Sub-rotina - Conjunto de comandos que podem ser processados por mais de um aplicativo. A sub-rotina tem seu código objeto e fonte independentes dos objetos e fontes dos aplicativos, sendo ativada através do comando "CALL".

Sub-rotinas globais - São sub-rotinas dos aplicativos ou do SADS que são utilizadas pelos programas chamados e que são declaradas na área global de comunicação do host, sendo, por isso mesmo, utilizadas globalmente. Desta forma, são "linkadas" apenas uma vez ao host, somente na sua primeira chamada, economizando os tempos de link nos novos disparos das mesmas pelos programas chamados.

Superentidade - Entidade que em uma hierarquia de generalização possui duas ou mais subentidades a ela subordinadas. Pode também ser denominada de entidade "pai", determinando todas as classificações herdadas pelas subentidades a ela relacionadas.

Tabela - Conjunto composto de uma base de dados, estruturas para acesso aos dados armazenados e uma tela para captação destes dados, sendo todos estes elementos atualizados e controlados pelo SADS. A tabela se apresenta como alternativa para implementação física de alguns tipos de entidades. As funções de criação e manutenção, bem como de armazenamento dos itens de dados, são processadas e gerenciadas por um módulo do SADS.

TCP/ IP - Acrônimo de Transmission Control Protocol/Internet Protocol, é a família de protocolos para a comunicação de dados inter-redes, originalmente proposta para a Advanced Research Products Agency (ARPA). Hoje é um padrão de fato para inter-redes abertas, e seu uso é amplamente difundido dentro e fora dos EUA.

Tela - Conjunto de caracteres ordenados de forma significativa, utilizados em programas on-line transacionais para apresentação e captação de dados em terminais de vídeo ou microcomputador.

Tela de processamento - Tela utilizada pelo programa aplicativo que processa a transação e que permite operações de inclusão, alteração, exclusão e consulta, geração, liberação entre outras operações.

Tipo da lógica de programa - Indicador do tipo de processamento a ser executado pelo programa (inclusão, alteração, consulta, etc) e, por conseguinte, do modelo com o qual ele será gerado. O SADS gera programas segundo tipos de lógicas padronizadas que atendem à maioria das necessidades de aplicativos on-line, batch e sub-rotinas. Quanto mais simples a lógica do programa ou sub-rotina e quanto mais próxima de uma lógica padronizada estiver sua função, maior será a capacidade de geração do SADS.

Trancode - Código de transação. Primeiro campo transmissível de uma tela. É utilizado pelo COMS para identificação do programa que irá tratar aquela tela.

Transação prototipada (ou transação em prototipação) - Transação cadastrada no menu de um sistema porém não relacionada a nenhum programa chamado. Neste caso, quando esta transação for requisitada, o host irá simular a execução de um processamento, dando a idéia de como será sua operação após seu desenvolvimento sem atualização das bases de dados. A prototipação é importante para validação da navegação entre as telas do sistema e da consistência de dados de entrada.

Window - Conceito na arquitetura COMS que habilita um número de ambientes de programas para ser operado independente e simultaneamente a uma estação. Um dos ambientes de programa pode ser visualizado enquanto os outros continuam a operação.

WWW - World Wide Web, ou Web. Meta-rede, baseada em hipertextos, que integra diversos serviços Internet, através de uma interface que possibilita o acesso a informações multimídia.

Unidades autorizadas - Unidades (empresa, estabelecimento e unidade) às quais o usuário pode obter autorização de acesso para uso de sistemas aplicativos.

Unidade de armazenamento - Critério pelo qual os itens são armazenados e considerados na tabela. Ela irá definir em qual nível de unidade de uma instalação os itens armazenados na tabela estarão disponíveis.

Unidade SADS - Código numérico que identifica a partição do repositório de dados que será trabalhada. Na atual versão do SADS não foi implementado ainda o particionamento do repositório; portanto, o código da unidade SADS assumido é igual a 01.

Usercode - Divisão lógica dos arquivos dentro dos discos. Cada usercode se assemelha a um diretório onde residem vários arquivos.

Usercode de análise - Usercode definido para o SADS, na tabela SBPM5000, como principal no desenvolvimento de um sistema. Nele residirão os fontes dos programas aplicativos. Relacionados ao usercode de análise existem os usercodes de teste e produção, onde residirão as libraries e programas objetos respectivamente em teste e produção.

Usuário - Indivíduo habilitado a efetuar transações do SADS e sistemas gerados por ele. Possui um número, código e senha próprios que o identificam para obter acesso a estes sistemas.

Usuário bloqueado - Usuário que está impedido de efetuar logon no ambiente SADS.

Validades em nível de estrutura - Testes aos quais os atributos/campos pertencentes a uma consistência devem ser submetidos, quando a mesma for executada.

Validades - Valores ou intervalo de valores que o atributo ou campo poderão assumir.

Servidor Web - Módulo do SADS que visa disponibilizar os sistemas aplicativos desenvolvidos através do SADS no ambiente World Wide Web.
