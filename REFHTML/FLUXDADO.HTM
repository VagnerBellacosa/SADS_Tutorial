Fluxo de dados

Conceituação

Fluxo de dados é o caminho pelo qual as informações fluem no sistema. Representam, no DFD - Diagrama de Fluxo de Dados, a transferência dos dados entre os elementos do sistema, retratando as interfaces entre os componentes do DFD. Os dados podem fluir entre processos, entre depósitos de dados e processos e entre entidades externas e processos. Um fluxo de dados é composto por um dado ou por uma estrutura de dados.

Os fluxos de dados do DFD podem ser definidos no ET-SADS, que é uma ferramenta UPPER-CASE voltada à aplicação da análise de dados e da análise estruturada e, posteriormente, transferidos para o repositório de dados do SADS. O ET-SADS trabalha na filosofia de estação de trabalho, requerendo uma plataforma IBM-PC ou compatível em ambiente MS-DOS.

As informações relativas à transferência de informações do ET-SADS para o SADS podem ser consultadas no "Guia de Referência do DFD" e no "Guia de Integração com o SADS", manuais que acompanham o ET-SADS.

Padronização

O padrão de codificação para identificar fluxos de dados no repositório de dados do SADS é o seguinte:

"sssZFxxxxx" onde:

"sss" é o código alfabético que identifica o sistema;

"ZF" é a constante de fluxo de dados e

"xxxxx" é o identificador alfanumérico do fluxo de dados.

Utilização

As informações relativas aos fluxos de dados são utilizadas apenas para documentação, pois podem fazer parte dos manuais emitidos pelo SADS, não havendo gerações a partir de suas especificações.

Cadastramento

Para definir um fluxo de dados, é necessário que o sistema já esteja cadastrado no repositório de dados do SADS.

CAMPOS DA TELA

FLUXO DE DADOS - Código do fluxo de dados no padrão SADS (sssZFxxxxx).

CODIGO EXTERNO - Código externo do fluxo de dados.

Este código pode ser informado no caso da instalação adotar um padrão de codificação diferente do padrão SADS ou quando houver integração entre o ET-SADS e o SADS. Neste caso, o código externo do fluxo de dados no SADS conterá o código do fluxo de dados no ET-SADS.

O código externo do fluxo de dados é uma informação puramente documental, ou seja, somente será utilizado para identificar, nos relatórios e manuais, a qual elemento externo ao SADS o fluxo de dados se refere.

NOME - Descrição resumida do fluxo de dados.

O nome tem o objetivo de identificar e explicar, de maneira reduzida, a que se refere o fluxo de dados.

DESCRICAO - Descrição detalhada do fluxo de dados.

Estão disponíveis 99 páginas, com 10 linhas cada, para se efetuar a descrição detalhada do fluxo de dados.

CONTINUA - Indicador de continuidade da operação.

As possibilidades de preenchimento deste campo são as seguintes:

"S" - Nova tela será aberta para prosseguir descrevendo o fluxo de dados;

"N" - Última tela da descrição do fluxo de dados.

OPERACAO - Operação efetuada em relação ao fluxo de dados.

As operações possíveis são as seguintes:

"INC" (Inclusão);

"ALT" (Alteração);

"EXC" (Exclusão);

"CON" (Consulta).
